## Implementation Decisions

### Cross-Project (XPROJ-01)
- **D-01:** Move the Claude Code hook registration from project-level `.claude/settings.local.json` to global `~/.claude/settings.json`. The hook script path is already absolute (`/Users/Q284340/Agentic/coding/src/hooks/knowledge-injection-hook.js`), so it works from any project directory.
- **D-02:** The retrieval service runs on port 3033 from the coding project's Docker stack. It is always available when a `coding` session is active (started via `bin/coding`). No per-project retrieval service needed.
- **D-03:** The hook script stays in `src/hooks/knowledge-injection-hook.js` (single source of truth). All agents reference this same script or its output.

### Agent Adapters (XAGT-01)
- **D-04:** Each agent gets its own thin adapter in `src/hooks/` that calls the retrieval service and injects results using that agent's native mechanism:
  - **Claude Code**: `knowledge-injection-hook.js` (Phase 30, already exists) — UserPromptSubmit hook, stdout JSON with `additionalContext`
  - **OpenCode**: `knowledge-injection-opencode.js` — OpenCode supports custom instructions via `.opencode/instructions.md` context files. The adapter writes a `.opencode/knowledge-context.md` file that OpenCode loads as system context. A pre-launch hook in `config/agents/opencode.sh` triggers the retrieval call.
  - **Copilot**: `knowledge-injection-copilot.js` — GitHub Copilot reads `.github/copilot-instructions.md` for workspace context. The adapter writes/updates this file with retrieved knowledge before each session.
  - **Mastra**: `knowledge-injection-mastra.js` — Mastracode supports custom context via `.mastra/context.md`. The adapter writes this file with retrieved knowledge.
- **D-05:** All adapters share a common retrieval client module `src/hooks/retrieval-client.js` that handles the HTTP call, timeout, fail-open, and response parsing. Each adapter only handles the injection format for its specific agent.
- **D-06:** The `launch-agent-common.sh` script already exports `CODING_HOOKS_CONFIG` and runs agent-specific hooks. Add a `agent_inject_knowledge()` function called during the pre-launch phase (before the agent process starts) that invokes the appropriate adapter for the current agent.
- **D-07:** For Claude Code (real-time per-prompt injection), keep the existing UserPromptSubmit hook mechanism. For other agents (session-start injection), the adapter runs once at session start and writes a context file that persists for the session duration.

### Relevance Focusing (XREL-01)
- **D-08:** Pass additional context to the retrieval service to improve relevance:
  - `project`: basename of the current working directory (e.g., "rapid-automations", "coding")
  - `cwd`: full working directory path
  - `recent_files`: output of `git diff --name-only HEAD~3` (last 3 commits' changed files)
- **D-09:** The retrieval service already accepts a `query` parameter. Extend it to also accept `context` object with `project`, `cwd`, and `recent_files`. The service uses these for boosting relevance scores of matching entities.
- **D-10:** For Claude Code (per-prompt), the hook already sends the prompt text as the query. Append project context to improve results.
- **D-11:** For session-start agents (Copilot, OpenCode, Mastra), use a generic query like "project context for {project_name}" combined with the context object. This returns the most relevant knowledge for the overall project rather than a specific question.
- **D-12:** The retrieval service changes (accepting `context` object) require updating `src/retrieval/retrieval-service.js` — this is a backward-compatible addition (existing callers without `context` continue to work).

### Claude's Discretion
- How to structure the `.opencode/knowledge-context.md`, `.github/copilot-instructions.md`, and `.mastra/context.md` files (formatting, headers, max length)
- Whether to refresh session-start context files mid-session or only at launch
- Exact relevance boosting weights for project/cwd/recent_files context signals
- Whether to include a "last refreshed" timestamp in context files
