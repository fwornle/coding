  <action>
1. Create `src/embedding/content-hash.ts`:
   ```typescript
   import crypto from "node:crypto";

   /**
    * Produce a truncated SHA-256 hash of text content.
    * Used for idempotency checks during Qdrant upsert â€” if a point
    * with the same content_hash already exists, skip re-embedding.
    */
   export function contentHash(text: string): string {
     return crypto.createHash("sha256").update(text).digest("hex").substring(0, 16);
   }
   ```

2. Create `src/embedding/embedding-service.ts` following the singleton + lazy-init pattern from `scripts/fast-embedding-generator.js`:
   - Import `EmbeddingModel, FlagEmbedding` from `fastembed`
   - Read `embedding-config.json` via `readFileSync` using `import.meta.url` for path resolution
   - Singleton pattern: module-level `let instance: EmbeddingService | null = null`
   - Class `EmbeddingService` with:
     - Private `model: Awaited<ReturnType<typeof FlagEmbedding.init>> | null = null`
     - Private `initPromise: Promise<void> | null = null`
     - `async initialize(): Promise<void>` â€” guarded init (if model exists return, if initPromise exists await it, else create initPromise calling `FlagEmbedding.init({ model: EmbeddingModel.AllMiniLML6V2 })`)
     - `async embedOne(text: string): Promise<number[]>` â€” calls `await this.initialize()`, then `this.model!.queryEmbed(text)`, returns the number array
     - `async *embedBatch(texts: string[], batchSize = 64): AsyncGenerator<number[][]>` â€” calls `await this.initialize()`, then yields from `this.model!.embed(texts, batchSize)` (fastembed returns async generator of batches)
     - `getConfig()` â€” returns the parsed config object
   - Export `getEmbeddingService(): EmbeddingService` â€” creates singleton if null
   - All logging via `process.stderr.write("[EmbeddingService] ...")`
   - NOTE: The fastembed enum name `EmbeddingModel.AllMiniLML6V2` is assumed. If the enum doesn't exist under that name, log `Object.keys(EmbeddingModel)` at init time to find the correct enum key and use it. The model string "all-MiniLM-L6-v2" can also be passed directly as a string if the enum approach fails.

3. Create `src/embedding/qdrant-collections.ts`:
   - Import `QdrantClient` from `@qdrant/js-client-rest`
   - Import `readFileSync` from `node:fs`
   - Read config from `embedding-config.json` (same path resolution as embedding-service)
   - Module-level `let client: QdrantClient | null = null`
   - Export `getQdrantClient(url?: string): QdrantClient` â€” creates singleton using `url ?? process.env.QDRANT_URL ?? "http://localhost:6333"`
   - Export `async ensureCollections(qdrant: QdrantClient): Promise<void>`:
     - Call `qdrant.getCollections()` to list existing
     - For each collection in `config.collections`:
       - If collection does not exist: create with `vectors: { size: config.dimensions, distance: config.distance }`
       - Create payload indexes from `collection.payloadIndexes` using `qdrant.createPayloadIndex(name, { field_name, field_schema: "keyword" })` for each index
     - Log creation/skip via `process.stderr.write`
   - Follow the idempotent creation pattern from `scripts/index-ontology-to-qdrant.js` lines 171-192
   - Use the fail-fast connectivity check pattern from the same file (lines 80-92): try `getCollections()`, catch and exit with helpful error message
