Create `src/retrieval/` directory and three utility modules:

**1. `src/retrieval/rrf-fusion.js`** -- ESM module with named exports, no class:

- `rrfFuse(rankedLists, k = 60)` -- Reciprocal Rank Fusion per D-02. Takes array of ranked lists (each item has `id`, `tier`, `tierWeight`, `payload`). For each list, score items as `1 / (k + rank + 1)`. Accumulate scores per item ID. After fusion, multiply each item's score by its `tierWeight` per D-03 (insights 1.5x, digests 1.2x, kg_entities 1.0x, observations 0.8x). Sort descending by final score. Return array with `rrfScore` added to each item.

- `recencyScore(dateStr, halfLifeDays = 14)` -- Exponential decay: `Math.pow(0.5, ageDays / halfLifeDays)`. Returns 0.5 if dateStr is null/undefined.

- `buildRecencyList(items)` -- Dedup items by `id`, compute recency from `payload.date` (or `payload.created_at` as fallback), sort descending by recency score. Used as the third input list to `rrfFuse()`.

- `TIER_WEIGHTS` constant exported: `{ insights: 1.5, digests: 1.2, kg_entities: 1.0, observations: 0.8 }`.

**2. `src/retrieval/token-budget.js`** -- ESM module, import `countTokens` from `gpt-tokenizer` per D-07:

- `TIER_ORDER` constant: `['insights', 'digests', 'kg_entities', 'observations']` -- fill order per D-08.

- `formatResult(item)` -- Format a single result as markdown with source attribution per D-05. Switch on `item.tier`:
  - insights: `**{topic}** (confidence: {confidence})\n{summary_preview}\n`
  - digests: `**{theme}** ({date}, agents: {agents})\n{summary_preview}\n`
  - kg_entities: `**{entityType}** (level: {hierarchyLevel})\n{summary_preview}\n`
  - observations: `*{agent}* ({date}, {project})\n{summary_preview}\n`
  All fields from `item.payload`.

- `truncateResult(item, tokenBudget)` -- Per D-09. Create the header (formatResult with empty summary_preview), count its tokens, compute available tokens for preview, slice summary_preview to fit. Return new item with truncated payload. Return null if available <= 0.

- `assembleBudgetedMarkdown(sortedResults, budget = 1000)` -- Per D-08. Walk sorted results (already sorted by RRF score). For each result: format it, count tokens. If adding would exceed budget: truncate to fit remaining (if remaining > 50 tokens), then break. Bucket results by tier. Build final markdown with tier headers (## Insights, ## Digests, ## Entities, ## Observations) per D-05. Only include headers for tiers that have results. Return `{ markdown, tokensUsed }`.

**3. `src/retrieval/keyword-search.js`** -- ESM module with KeywordSearch class:

- Constructor takes `options = {}` with `limit` (default 20).

- `search(db, query)` -- Takes a better-sqlite3 db instance (readonly, passed from server.js's `_getObservationsDb()`). Returns `{ observations: [], digests: [], insights: [] }`. Each item has `id`, `summary_preview`, tier-specific fields, and `tier` string.

- `_searchObservations(db, query)` -- FTS5 probe pattern: try `SELECT 1 FROM observations_fts LIMIT 0`, if succeeds use `MATCH` query on `observations_fts`, else fallback to `LIKE '%query%'`. Select `id, summary as summary_preview, agent, created_at as date, quality`. Limit to `this.limit`.

- `_searchDigests(db, query)` -- LIKE only (no FTS table exists per Research Pitfall 2). Search `summary LIKE ?` and `theme LIKE ?`. Select `id, summary as summary_preview, theme, date, agents, quality`. Limit to `this.limit`.

- `_searchInsights(db, query)` -- LIKE only. Search `summary LIKE ?` and `topic LIKE ?`. Select `id, summary as summary_preview, topic, confidence`. Limit to `Math.floor(this.limit / 2)`.

- Each `_search*` method wrapped in try/catch returning `[]` on failure (table may not exist).

- Each returned item gets `tier: 'observations'|'digests'|'insights'` added in the SELECT.

All logging uses `process.stderr.write('[ComponentName] message\n')` -- NEVER console.log.

**Class: RetrievalService**

Constructor(options = {}):
- `this.scoreThreshold = options.scoreThreshold ?? 0.75` (per D-04)
- `this.defaultBudget = options.defaultBudget ?? 1000` (per D-08)
- `this.embeddingService = null` (set in initialize)
- `this.qdrantClient = null` (set in initialize)
- `this.keywordSearch = new KeywordSearch()`
- `this.dbGetter = options.dbGetter ?? null` -- function that returns a better-sqlite3 db instance (injected by server.js via `() => this._getObservationsDb()`)

`async initialize()`:
- Guard pattern (if already initialized, return; if initPromise exists, await it)
- `this.embeddingService = getEmbeddingService()`
- `await this.embeddingService.initialize()` -- warm fastembed model (Pitfall 1)
- `this.qdrantClient = getQdrantClient()`
- Log: `process.stderr.write('[RetrievalService] Initialized (fastembed warm, Qdrant connected)\n')`

`async retrieve(query, options = {})`:
- Destructure: `const { budget = this.defaultBudget, threshold = this.scoreThreshold } = options`
- Step 1: `const vector = await this.embeddingService.embedOne(query)` -- embed query (~20ms warm)
- Step 2: Parallel search via `Promise.all([this._semanticSearch(vector, 20, threshold), this._keywordSearch(query)])`
- Step 3: Build recency list from combined unique results: `const recencyResults = buildRecencyList([...semanticResults, ...keywordHits])`
- Step 4: Fuse: `const fused = rrfFuse([semanticResults, keywordHits, recencyResults])`
- Step 5: Assemble: `const { markdown, tokensUsed } = assembleBudgetedMarkdown(fused, budget)`
- Return per D-06: `{ markdown, meta: { query, budget, results_count: fused.length, latency_ms: 0 } }` (latency_ms set by caller)

`async _semanticSearch(queryVector, limit = 20, threshold = 0.75)`:
- `const COLLECTIONS = ['insights', 'digests', 'kg_entities', 'observations']`
- `const TIER_WEIGHTS = { insights: 1.5, digests: 1.2, kg_entities: 1.0, observations: 0.8 }`
- `Promise.all` over collections, each calling `this.qdrantClient.search(collection, { vector: queryVector, limit, score_threshold: threshold, with_payload: true, with_vector: false })`
- Map each point to `{ id: p.id, score: p.score, payload: p.payload, tier: collection, tierWeight: TIER_WEIGHTS[collection] }`
- `.catch()` per collection: log error, return `[]` (graceful degradation -- if Qdrant is down, keyword search still works)
- Flatten results

`_keywordSearch(query)`:
- Get db: `const db = this.dbGetter ? this.dbGetter() : null`
- If no db, return `[]`
- Call `this.keywordSearch.search(db, query)`
- Flatten the `{ observations, digests, insights }` result into a single array
- Add `tierWeight` to each item based on its `tier` field
- Add a synthetic `id` if missing (use `${tier}-${item.id}` to avoid collisions with Qdrant point IDs)

**Singleton factory:**
```javascript
let instance = null;
export function getRetrievalService(options) {
  if (!instance) instance = new RetrievalService(options);
  return instance;
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> /api/retrieve | Untrusted query string from HTTP POST body |
| Node.js -> Qdrant | Network call to Docker container on localhost:6333 |
| Node.js -> SQLite | File system access to .observations/observations.db (readonly) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-29-01 | Tampering | keyword-search.js | mitigate | Use parameterized queries (? placeholders) for all SQLite operations -- never interpolate query string into SQL |
| T-29-02 | Denial of Service | retrieval-service.js | mitigate | Limit Qdrant results to 20 per collection (80 max total); token budget caps output size; query string length not validated here but will be in server.js handler (Plan 02) |
| T-29-03 | Information Disclosure | token-budget.js | accept | summary_preview content comes from trusted internal knowledge base; no PII filtering needed |
| T-29-04 | Denial of Service | retrieval-service.js | mitigate | Graceful degradation: catch Qdrant errors per-collection, return [] for failed collections; fastembed initialized eagerly to avoid cold-start timeout |
</threat_model>
