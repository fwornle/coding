    Create a new component `TraceHistoryPanel` with two modes: list view and comparison view.

    **List View (default):**
    - Fetch trace summaries from `/api/trace-history` on mount
    - Display as a table/list: date, workflow name, status, LLM calls, tokens, entity counts, duration
    - Each row has a checkbox for selection (max 2)
    - "Compare Selected" button enabled when exactly 2 are checked
    - Anomaly badges on rows that meet anomaly criteria (see below)
    - Show "No trace history yet" message if empty

    **Comparison View (when 2 selected):**
    - Fetch full trace detail for both selected traces via `/api/trace-history?file=X`
    - Two-column layout with aligned wave rows:
      - Left column: Trace A (older)
      - Right column: Trace B (newer)
      - Each column shows: wave name, duration, LLM calls, tokens, entity flow
    - **Delta highlighting** per locked decision:
      - Duration delta: green if B faster, red if B slower (>10% threshold)
      - LLM calls delta: green if fewer, red if more
      - Entity counts: red if B produced fewer entities, green if more
      - Show delta values: "+12 calls", "-3s", "-2 entities"
    - "Back to list" button to return to list view

    **Anomaly Detection** per locked decision:
    - Calculate averages across all traces in history
    - Flag a trace as anomalous if:
      - Entity count drops: persisted < 50% of average
      - High error rate: any step has status 'failed'
      - Duration 2x+ longer than average
      - High QA rejection: (produced - persisted) / produced > 50%
    - Show anomaly as a red/amber badge next to the trace row: "Entity drop", "Slow run", "High rejection"

    **Props:**
    ```typescript

      // No props needed -- self-contained, fetches own data
    }
    ```

    **Styling**: Use shadcn/ui components (Badge, Button, ScrollArea, Separator). Tailwind for layout. Table with alternating row colors. Delta values in colored badges.

    **Error handling**: Show error message if API call fails. Loading spinner while fetching.
