**Fix:** Replace the inline heredoc (lines 104-113) with a call to a new function `_generate_mastra_hooks_config` that creates hooks.json with real shell commands.

**Hook event names to use (from research):** `SessionStart`, `SessionEnd`, `UserPromptSubmit`, `Stop`. Note: the current code uses `onSessionStart`/`onSessionEnd`/`onMessage` which do NOT match the research-documented event names.

**Transcript file path:** The NDJSON must land in `.observations/transcripts/` relative to CODING_REPO (or TARGET_PROJECT_DIR if set). This matches `findMastraTranscriptDir()` which returns `path.join(projectPath, '.observations', 'transcripts')`.

**The function `_generate_mastra_hooks_config`** should:

1. Accept one argument: the output path for hooks.json
2. Determine the transcript directory: `${CODING_REPO:-.}/.observations/transcripts`
3. Ensure the transcript directory exists (mkdir -p)
4. Determine a session-unique transcript filename. Since hooks.json is static (generated once at pre-launch), the filename should use a pattern the hook command can resolve at runtime. Use the env var `$MASTRACODE_SESSION_ID` if available, falling back to a date-based name. The safest approach: each hook command appends to a file named with a date prefix, e.g., `TRANSCRIPT_DIR/mastra-$(date +%Y%m%d-%H%M%S).jsonl`. BUT since all hooks in the same session should write to the SAME file, use a session-level approach:
   - The `SessionStart` hook creates a symlink or file marker
   - OR simpler: all hooks write to a single rolling file `mastra-current.jsonl` and `SessionStart` rotates it

   **Simplest approach (use this):** All hooks write to `TRANSCRIPT_DIR/mastra-transcript.jsonl`. MastraTranscriptReader already handles incremental tailing. SessionStart events in the NDJSON allow the reader to detect session boundaries. No rotation needed -- the reader handles file growth via offset tracking.

5. Generate hooks.json with these hook entries:

**SessionStart hook command:**
```
echo '{"type":"session_start","sessionId":"'$MASTRACODE_SESSION_ID'","cwd":"'$MASTRACODE_CWD'","timestamp":"'$(date -u +%Y-%m-%dT%H:%M:%SZ)'"}' >> TRANSCRIPT_DIR/mastra-transcript.jsonl
```

**SessionEnd hook command:**
```
echo '{"type":"session_end","sessionId":"'$MASTRACODE_SESSION_ID'","cwd":"'$MASTRACODE_CWD'","timestamp":"'$(date -u +%Y-%m-%dT%H:%M:%SZ)'"}' >> TRANSCRIPT_DIR/mastra-transcript.jsonl
```

**UserPromptSubmit hook command:**
The user message content ($MASTRACODE_MESSAGE) may contain quotes, newlines, and special characters that would break JSON. Use a small inline script with proper escaping:
```
MSG=$(echo "$MASTRACODE_MESSAGE" | sed 's/\\/\\\\/g; s/"/\\"/g; s/\t/\\t/g' | tr '\n' ' '); echo '{"type":"message","role":"user","content":"'"$MSG"'","sessionId":"'$MASTRACODE_SESSION_ID'","timestamp":"'$(date -u +%Y-%m-%dT%H:%M:%SZ)'"}' >> TRANSCRIPT_DIR/mastra-transcript.jsonl
```

**Stop hook command** (captures assistant's final response):
```
MSG=$(echo "$MASTRACODE_STOP_MESSAGE" | sed 's/\\/\\\\/g; s/"/\\"/g; s/\t/\\t/g' | tr '\n' ' '); echo '{"type":"message","role":"assistant","content":"'"$MSG"'","reason":"'$MASTRACODE_STOP_REASON'","sessionId":"'$MASTRACODE_SESSION_ID'","timestamp":"'$(date -u +%Y-%m-%dT%H:%M:%SZ)'"}' >> TRANSCRIPT_DIR/mastra-transcript.jsonl
```
