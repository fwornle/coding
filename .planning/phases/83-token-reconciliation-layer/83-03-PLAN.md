<objective>
Build the reconcile matcher — the heart of Phase 83 — as a testable unit before it is wired into the adapter (Plan 04). It generalizes `insertTokenRowDeduped`'s authority model into a match-then-enrich-or-flag engine:
- **D-04 (fill gaps only):** wire values are authoritative and never overwritten. Transcript enriches only the fields the wire row lacks (`reasoning_tokens`, `granularity_tier`, `parent_call_id`, and the cache split when wire has 0 and transcript has data). Count disagreements are recorded, never applied.
- **D-05 (record all deltas, flag beyond tolerance):** every nonzero per-field delta on a matched pair is recorded per-request; deltas beyond a calibrated tolerance are `flagged: true` and roll up into the span summary.

The join is the load-bearing subtlety: wire rows and transcript rows carry **different** `user_hash` by design, so the matcher joins on `tool_call_id` (== upstream request-id) ACROSS `user_hash` boundaries — it must NOT reuse the `(user_hash, tool_call_id)` dedup probe.

Purpose: A pure, unit-tested matcher isolates the trickiest logic (cross-key join, fuzzy fallback, delta/tolerance) from the I/O wiring, keeping Plan 04 thin.
Output: `lib/lsl/token/reconcile.mjs` + a widened token-db probe/enrich surface + a matcher unit suite.
</objective>
