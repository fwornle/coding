<objective>
Implement the OpenCode Path B (sweep) adapter — the second concrete adapter in Wave 2. Unlike Claude (filesystem tree) and Copilot (events.jsonl), OpenCode stores sub-agents in a SQLite database (`~/.local/share/opencode/opencode.db`) with first-class `session.parent_id` modeling. 388 of 502 sessions (77%) on the inspected box are sub-sessions — this is in active use here, and 46 sessions live under `/Users/Q284340/Agentic/coding`.

Per RESEARCH-opencode.md key finding: the SQLite layer IS the structured payload — no path-encoding tricks, no worktree-cleanup races, no recursive directory walks. The adapter reads `session.directory` directly (unencoded absolute cwd) for project mapping.

Per D-Reuse: Phase 50's `lib/lsl/scan-and-convert.mjs` is NOT directly applicable — it walks JSONL files; OpenCode is SQLite. The adapter is a parallel implementation that produces ObservationWriter-compatible message arrays. Phase 50 function signatures unchanged.

Critical landmines from RESEARCH-opencode.md:
- `time_created` is MILLISECONDS not seconds (Drizzle integer-timestamp default)
- Long-running transactions WILL conflict with WAL checkpoint → use prepared statements, short-lived; pragma busy_timeout=5000
- SQLite mirror at `~/.copilot/session-store.db::turns` does NOT contain sub-agent events for Copilot — but for OpenCode, the `opencode.db::session` table IS authoritative (different agent, different rule)
- Schema-version mismatch: read `MAX(id) FROM __drizzle_migrations` first; hard-fail on unknown migration ID

Purpose: Close CONTEXT.md AC #4 (agent-agnostic — OpenCode produces equivalent LSL + observation output to Claude Code's adapter). The OpenCode adapter is the simplest of the three agent-specific Path B plans by file count (one SQL adapter file) but the most novel by mechanism (SQLite vs filesystem tree).

Output: One new adapter module, one new test file, a fixture seeder helper.
</objective>

```sql
-- session table (relevant columns):
CREATE TABLE session (
  id TEXT PRIMARY KEY,
  project_id TEXT NOT NULL,
  parent_id TEXT,                -- NULL for top-level; child session id for sub-agent
  slug TEXT NOT NULL,
  directory TEXT NOT NULL,       -- absolute cwd
  title TEXT NOT NULL,
  workspace_id TEXT,
  agent TEXT,                    -- 'explore' | 'general' | 'build' | NULL
  model TEXT,
  cost REAL,
  tokens_input INTEGER, tokens_output INTEGER, tokens_reasoning INTEGER,
  tokens_cache_read INTEGER, tokens_cache_write INTEGER,
  time_created INTEGER,          -- MS since epoch (Drizzle integer-timestamp)
  time_updated INTEGER
);

-- message table:
-- one row per role-turn; data is JSON blob with {id, sessionID, role, time:{created}, agent, model:{providerID,modelID}}

-- part table:
-- one row per content fragment (text / tool-call / tool-result); data is JSON
-- Indexed by (message_id, id) and by session_id (part_session_idx)
-- Tool calls: json_extract(data,'$.tool') is set
```
