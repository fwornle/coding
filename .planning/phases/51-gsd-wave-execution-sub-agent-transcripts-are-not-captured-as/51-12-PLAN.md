<objective>
Fix CR-04 — the BLOCKING gap from Phase 51 verification. All 3 live-daemon plists and the sweep plist's interfaces block reference `/usr/local/bin/node` which does not exist on Apple Silicon hosts. The current installer copies these plists verbatim to `~/Library/LaunchAgents/`, so every install on an Apple Silicon Mac yields 4 jobs that exit immediately with code 78 (EX_CONFIG) because launchd resolves ProgramArguments[0] BEFORE applying EnvironmentVariables.PATH.

Sweep plist is also affected per REVIEW Addendum — verify whether it uses `/bin/bash` wrapper (proven working pattern) or `/usr/local/bin/node` directly. If the latter, fix it the same way as the live daemons.

The fix is per REVIEW Addendum line 428: EITHER (a) the installer resolves `node` via `command -v node` at install time and templatizes the plists (substitute the absolute path before copying to `~/Library/LaunchAgents/`), OR (b) the plists invoke a `/bin/sh -c` wrapper so `EnvironmentVariables.PATH` resolves node at exec time (the sweep wrapper pattern in `scripts/sub-agent-sweep-job.sh` proves this works).

The integration test gap (per REVIEW Addendum: "Why this isn't caught") that allowed CR-04 to ship — `tests/integration/sub-agent-launchd-install.test.js` only checks plist structure, NEVER asserts `ProgramArguments[0]` is an executable file on the host — MUST be closed in this plan so this regression cannot recur.

This plan is BLOCKING for Plan 51-16 (AC #3 + HUMAN-UAT re-attempt). Without Path A daemons booting cleanly, the live-tier observation pipeline is non-functional on Apple Silicon.

Purpose: Restore Path A functionality on Apple Silicon Macs (the primary developer hardware). Close the test-gap that allowed CR-04 to ship undetected.
Output: Installer + 4 plists (templatized OR wrapper-based) + extended integration test.
</objective>
