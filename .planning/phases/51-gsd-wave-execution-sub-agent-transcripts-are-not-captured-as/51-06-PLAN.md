<objective>
Build the LSL parity writer for sub-agents. This is the Wave 3 plan — depends on ALL FOUR Wave 2 adapters (51-02 through 51-05) — because the writer is agent-agnostic and must work across all four agent shapes.

CONTEXT.md AC #2 (LSL parity) requires: "a fresh wave-execution produces sub-agent LSL files under `.specstory/history/{YYYY}/{MM}/` matching the proposed naming convention, with valid frontmatter". This plan delivers that AND backfills LSL files for the 2026-05-23 Claude Code transcripts (CONTEXT.md AC #2 implicit retroactive scope; Plan 51-02 already wrote observations for those, but did NOT write LSL files — that work is here).

**D-LSL-Filename verbatim:**

```
{YYYY-MM-DD}_{HHHH-HHHH}_S{parent-slot}-{sub-index}-{sub-hash}[-part{N}].md
```

- `S{parent-slot}` — 1-indexed slot per active parent session within the local LSL day
- `{sub-index}` — 1-based index of sub-agent within parent's fan-out (already in Plan 51-01's row schema)
- `{sub-hash}` — 7-char prefix, **per-agent rule**:
  - Claude: agentId[:7] (Plan 51-02 confirms — 17-char hex agentId is the sub-agent's identity)
  - Copilot: toolCallId.replace('toolu_vrtx_', '').slice(0, 7) (Plan 51-04 confirms — sub-agents share parent's session uuid)
  - OpenCode: session.id.slice(0, 7) (Plan 51-03 confirms — child session has its own id with `ses_` prefix)
  - Mastra: subAgentSessionId.slice(0, 7) (Plan 51-05 forward-compat)
- `[-part{N}]` — existing chunking suffix from parent LSL pattern; for sub-agents, omitted unless the transcript exceeds the per-file LSL byte threshold (planner-locked at 100KB matching parent LSL chunking)

**Backward-compatibility (CONTEXT.md D-LSL-Filename):** existing parent-session LSL files (no `S{n}-` segment) are NEVER modified by this writer. Verified by Test 8.

Per D-Reuse: Phase 50's `lib/lsl/window.mjs` is NOT used by this plan (window parsing is for reading LSL, not writing). `lib/lsl/scan-and-convert.mjs` is not used directly. The writer is new code that produces files of the same shape Phase 50's parser consumes.

Purpose: Close CONTEXT.md AC #2 (LSL parity) AND extend AC #1 backfill to produce LSL files (not just observations) for the 2026-05-23 incident.

Output: Two new lib modules (writer + slot allocator), one CLI script, two test files.
