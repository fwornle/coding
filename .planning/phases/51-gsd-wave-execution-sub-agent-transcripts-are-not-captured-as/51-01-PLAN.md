<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Phase 51 dispatcher → adapter modules | Adapters are loaded dynamically; they read from disk paths owned by the user but potentially writable by other processes (e.g., the live Claude Code process is appending to the same JSONL files the sweep reads) |
| Adapters → Phase 50 primitives | Adapters call convertTranscriptsToObservations which calls ObservationWriter which calls the LLM proxy — same boundary as Phase 50, no new surface |
| Sweep CLI argv → adapter searchPaths | Operator-controlled flags (--project, --since) reach the adapter — but the searchPaths themselves come from env vars + defaults, not directly from argv |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-51-01-FI | Information Disclosure | adapter discover() reads transcript paths | mitigate | Plan 51-01 specifies that adapters apply `fs.statSync(file).uid === process.getuid()` before reading; non-owner files are skipped with stderr notice. This plan locks the interface; 51-02..51-05 implement the check per-adapter. |
| T-51-01-TR | Tampering | Map-backed registry has no persistence | accept | Phase 51 v1 uses in-memory Map per CONTEXT.md Claude's Discretion. Sweep runs are short-lived (≤ 5 min). Persistence to SQLite is a documented follow-up — does not affect correctness in v1 since each sweep run re-discovers from disk. |
| T-51-01-DR | Denial of Service | unbounded discover() output | mitigate | --limit flag caps per-agent rows at 100 default. Adapters that produce >limit results MUST slice from the front of the list (oldest discovered first). Stderr-logs the cap was hit. |
| T-51-01-RL | Repudiation | observation rows lack provenance | mitigate | D-Live-Sweep-Tags requires `metadata.source = "sub-agent-backfill"` + parent_session_id + sub_index + sub_hash + agent + project. This plan passes the global tag; adapters fill the per-row metadata. |
| T-51-01-PI | Tampering (via prompt injection) | adapter content reaches LLM via Phase 50 resolver | accept | Already mitigated by Phase 50 D-Confidence + prompt-injection-resistant system prompt in scripts/resolve-observations-from-lsl.mjs. Phase 51 does not call the resolver directly; sweep writes raw observations. |
| T-51-01-SC | Tampering | npm dependencies | mitigate | This plan adds ZERO new package installs. `git diff package.json` clean. Future plans (51-03 better-sqlite3 — already in deps; 51-08 chokidar — to be evaluated in 51-08) will go through the Package Legitimacy Gate. |
