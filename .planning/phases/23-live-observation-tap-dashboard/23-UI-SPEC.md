### Responsive Behavior

- **Desktop (>= 1024px):** Sidebar 280px fixed, content area flex-1. Sidebar visible by default.
- **Tablet (768-1023px):** Sidebar collapses to a filter button that opens a sheet/drawer overlay.
- **Mobile (< 768px):** Full-width observation list, filter accessible via top bar button.

### Navigation

The dashboard currently has no router. This phase adds `react-router-dom` with two routes:
- `/` -- existing SystemHealthDashboard (default)
- `/observations` -- new Observations page

Navigation bar: horizontal tabs at top of page. Two items: "Health" (links to `/`) and "Observations" (links to `/observations`). Use shadcn `Tabs` visually but backed by router links. Active tab uses `border-b-2 border-primary` indicator.

## Interaction Contracts

### Observation List

- **Default state:** Show 50 observations, newest first (per D-09 pagination defaults).
- **Click observation card:** Expand inline to show full observation content. Only one card expanded at a time (accordion behavior). Second click collapses.
- **Expanded card:** Shows full `content` field with preserved whitespace. Card background shifts to `hsl(var(--accent))` with subtle left border (3px) in agent color.

### Filter Sidebar

- **Agent checkboxes:** All four checked by default. Unchecking removes that agent from results. Changes apply on "Apply Filters" button click (not live).
- **Time range:** Two date inputs (From / To). Empty = no constraint on that bound. Defaults: From = 7 days ago, To = now.
- **Project selector:** Dropdown populated from unique project values in current results. "All projects" default.
- **Search input:** Text field for full-text search (`q` parameter). Debounce 300ms after typing stops, then apply. Placeholder: "Search observations..."
- **Apply Filters button:** Sends GET request to `/api/observations` with all filter params. Resets pagination to page 1.

### Pagination

- **Style:** `< Prev  1 2 3 ... N  Next >` -- standard page buttons.
- **Limit:** 50 per page (per D-09).
- **Current page:** Highlighted with `bg-primary text-primary-foreground`.

### Auto-Refresh

- **Strategy:** Polling every 30 seconds (Claude's discretion item from CONTEXT.md).
- **Visual indicator:** Small spinning `RefreshCw` icon (lucide) in the top-right corner during fetch. Icon is 16px, muted-foreground color.
- **Manual refresh:** Explicit refresh button next to the auto-refresh indicator. Uses `RefreshCw` icon, ghost variant button.

### Navigation Badge (Claude's discretion item)

- Show a count badge next to "Observations" nav tab showing total observation count.
- Badge uses `Badge` component with `variant="secondary"` and `text-xs`.
- Count fetched from `/api/observations?limit=0` response metadata (total count without fetching rows).

## States

### Loading State

- Observation list area shows 3 skeleton cards (shimmer animation using shadcn skeleton pattern).
- Each skeleton card: 64px height, full width, rounded-lg, `bg-muted animate-pulse`.

### Empty State

- Centered vertically in content area.
- Icon: `Eye` (lucide), 48px, `text-muted-foreground`.
- Heading and body text below icon, `text-center`.
- If filters are active, show "No matching observations" variant.

### Error State

- Alert component (`variant="destructive"`) at top of content area.
- Shows error message + retry button.

### Offline/API Unavailable

- Alert with `AlertTriangle` icon: "Health API unreachable. Observation browsing is unavailable."

## Data Contract (API Response Shape)

Per CONTEXT.md D-10, the component expects this JSON shape from `GET /api/observations`:

```typescript
interface ObservationResponse {
  data: Observation[]
  total: number
  limit: number
  offset: number
}

interface Observation {
  id: string
  content: string
  agent: 'claude' | 'copilot' | 'opencode' | 'mastra'
  project: string
  sessionId: string
  timestamp: string  // ISO 8601
  source: string
}
```
