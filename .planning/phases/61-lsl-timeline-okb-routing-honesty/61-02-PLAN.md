<objective>
Bridge the unified viewer's OKB tab to OKM Express's legacy `/api/entities` contract on `:8090`, scoped to `system==='okb'` only, so operators see real RaaS / KPI-FW / business entities instead of a 404-induced empty graph — and never the coding-KG mirror.

Purpose: OKB is already retargeted to `:8090` (RESEARCH Unknown 4 — the "mirror entities" symptom was a stale build, not live code). The live failure is `okb -> :8090/api/v1/entities -> 404 -> empty graph`. OKM's entity and relation shapes are ALREADY viewer-compatible and share the `{success,data}` envelope, so the fix is a path-rewrite (`/api/v1/` -> `/api/`), not a new normalizer. This plan also resolves two delegated micro-decisions honestly: the 18,958-edge scale (cap + indicator) and the missing neighbors endpoint (client-side 1-hop).

Output: okb-scoped ApiClient routing, a documented relation cap with a visible honesty indicator, client-side node-expand for okb, and an extended E2E spec.
</objective>
