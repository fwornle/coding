<objective>
Give the OpenAI shim native tool-call passthrough (Phase 81 spike gap #1, a HARD requirement) and add the dedicated `/v1/copilot/chat/completions` path. Today the shim's `internalBody` drops `tools[]`/`tool_choice`, so no native `tool_calls` ever return and agentic BYOK/opencode loops are text hallucinations (spike probe 3: copilot claimed file creation, no file on disk; opencode v9 scored goal:null — same cause).

This plan wires the three drop-points (internalBody forward, copilot-provider body+parse, response envelope) plus capability gating so a tools-bearing request never silently lands on the tools-incapable claude-code CLI provider. The dedicated /v1/copilot path fixes the spike's agent mis-stamping (BYOK stamped agent='opencode', row 163286).

| proxy → upstream provider | tools payload sent to Copilot HTTP; tool_calls returned and echoed to client |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-82-03-01 | Tampering | arbitrary function schemas forwarded to providers | accept | function schemas are opaque to the proxy and executed only by the DOWNSTREAM agent runtime (copilot/opencode) which already owns tool execution + user allow-all gating; the proxy is a passthrough on a localhost bind; documented residual |
| T-82-03-02 | Elevation of privilege | tools silently dropped onto claude-code CLI producing hallucinated tool use | mitigate | capability gating: tools-bearing requests cannot select the --tools '' claude-code provider; the chain prefers a capable provider or fails loudly — NEVER silent-strip (the exact spike gap) |
| T-82-03-03 | Spoofing | /v1/copilot path lets any caller stamp agent='copilot' | mitigate | path-derived default only; X-Agent header still wins per Plan-02 precedence; localhost-bound source; process forced downstream |
| T-82-03-04 | Denial of Service | malformed tools array crashes the envelope | mitigate | Array.isArray guards on tools/toolCalls; no-toolCalls path preserved byte-identical; provider fetch stays in existing error handling |
| T-82-03-SC | Tampering | npm install of dependencies | accept | no new packages; extends existing provider + shim seams |
