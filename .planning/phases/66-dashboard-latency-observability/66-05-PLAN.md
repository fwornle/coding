<objective>
Close the SC-2 gap left open by Plan 66-04: the dashboard's red 'Regressed' overhead
badge could not be demonstrated on a fast local host because genuine claude-code
cold-spawn overhead tops out around ~2.5s, never crossing the 3s amber / 5s red
envelope (calibrated for ~14s VPN cold boots).

Add a host-independent, opt-in test seam — `LLM_PROXY_WORKER_SPAWN_DELAY_MS` — to the
rapid-llm-proxy worker pool that injects an artificial delay squarely into the
spawn/queue overhead window (the `firstOutputAt - dispatchedAt` interval 66-03 measures,
EXCLUDING model generation). Setting it above 5000ms inflates the per-model
`p50_overhead_ms` median past the red threshold so SC-2 (regression detectable -> red)
can be exercised deterministically. Then live-prove SC-2 on :3032 and close PERF-03.

This is a proxy-side test affordance, not a behaviour change: when the env is
unset/0 the dispatch path is byte-for-byte identical to today.
</objective>

<context>
Overhead window (locked by 66-03, worker-pool.mjs):
- `dispatchedAt = this._now()` is stamped in `_dispatch()` (~line 484), just before `_writeGuarded` writes the prompt (~line 527).
- `firstOutputAt` is stamped ONCE by `_onStdout` on the first parsed line of the turn (~line 576).
- `overheadMs = (firstOutputAt ?? now) - dispatchedAt` (~line 662), additive on the {content,model,tokens} completion contract via `toCompletion(ev, model, overheadMs)`.
- Injectable clock seam already exists: `this._now = deps.now || Date.now` (constructor `(model, systemPrompt, deps = {})`, ~line 271/294).

Cleanest seam: read `_spawnDelayMs` (from `deps.spawnDelayMs` ?? `parseInt(process.env.LLM_PROXY_WORKER_SPAWN_DELAY_MS, 10)` ?? 0) in the constructor. In `_dispatch()`, AFTER stamping `dispatchedAt`, if `_spawnDelayMs > 0` defer `_writeGuarded(...)` by that many ms (event-loop friendly — `setTimeout`, NOT a busy-wait); when 0 keep the synchronous write exactly as today. Because the write (and therefore first output) is delayed but `dispatchedAt` is not, the delay lands entirely in the overhead window and not in generation time. The per-request timeout already armed in `_dispatch` must still cover the delayed write (raise it or arm relative to actual write if needed — the delay must not trip the request timeout for normal SC-2 values like 6000ms).

For the unit test, drive the overhead with the injected `deps.now` clock (no real sleep): the existing overhead-window tests added by 66-03 show the pattern — advance the fake clock between dispatch and first-output and assert the computed `overheadMs`. For the delay seam, assert that the deferred-write path yields `overheadMs >= spawnDelayMs` under the fake clock, and that `spawnDelayMs: 0` leaves the existing assertions intact.
</context>
