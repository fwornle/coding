<objective>
Instrument the rapid-llm-proxy worker pool to record, per claude-code fallback call, the SPAWN/QUEUE OVERHEAD the pool actually affects — the time from request-received-by-the-pool to the worker emitting its first output for that turn (spawn + CLI boot + queue-wait + dispatch), EXCLUDING model generation time. Persist it as a new `overhead_ms` token-usage field alongside the existing `latency_ms`, expose a per-model `p50_overhead_ms` median (mirroring 66-01's `p50_latency_ms` pattern), and surface `overhead_ms` on the `/recent` per-call feed.

Purpose: PERF-03's three-second green threshold was calibrated on PERF-01's warm-overhead probe, but Phase 66 renders TOTAL end-to-end latency (sonnet median ~123s live, dominated by generation time the pool does not affect). The bar can never be met by a real-traffic total-latency median, and a pool regression (cold-spawn storm / pool disabled) is invisible against generation-time noise. Measuring the OVERHEAD component restores SC-1 (warm overhead near zero, green) and SC-2 (regression makes overhead climb toward ~14s, red).

Output: a new `overhead_ms` column + `p50_overhead_ms` summary median + `/recent` passthrough that Plan 66-04 (dashboard re-point) consumes. This is the proxy-side half, mirroring the 66-01 then 66-02 split.
</objective>
