Surface the per-model median latency (the `p50_latency_ms` field Plan 66-01 added) on BOTH
dashboard surfaces required by D-01: (a) a headline per-model tile on the primary system-health
dashboard (`:3032`) for glanceability, and (b) a Median Latency column on the existing Token Usage
page by-model table as the drill-down. Both apply the D-03 regression treatment: green ≤3s /
amber band / red as the median climbs toward the ~14s baseline (worked example >5s = regressed),
with a trend indicator vs the prior window; haiku is the no-threshold direct-path reference row.

probe. SC-2 — because the figure is sourced from the same fallback-path token-usage traffic the
pool serves (D-07), disabling the pool (`LLM_PROXY_DISABLE_WORKER_POOL=1`) or worker thrashing
shows up as the median climbing and the badge flipping red.

Output: token-usage.tsx Median column + threshold badge; new llm-latency-tile.tsx; the tile
registered into the :3032 tile grid; visually verified via gsd-browser.
</objective>
