## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LevelDB read on `hydrate()` | LevelDB content is trusted (written by the same process tree); but JSON-fallback path reads from `.data/exports/*.json` which is git-tracked - potentially attacker-modified across a clone. |
| `exportDir` from constructor opts | The caller (Plan 04's `GraphKMStore`) supplies `exportDir`. The path crosses the library boundary; defense-in-depth required. |
| `JSON.parse` on every fallback hydrate | Untrusted JSON input - same threat as T-37-02 (untrusted JSON during snapshot import). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-03-01 | Tampering | `JSON.parse` during `hydrateFromJsonExports` | mitigate | Node 22's `JSON.parse` does NOT pollute prototypes on plain object parsing. The hydrate path reads keys onto the parsed object's own properties only. No `Object.assign` into a prototype-shared object; no `lodash.merge`-style recursive merge; no `eval`. Documented in JSDoc on `hydrateFromJsonExports`. |
| T-37-03-02 | Tampering | `Exporter.exportNow()` writes to constructor-supplied `exportDir` | mitigate | Constructor defensively resolves the path via `path.resolve()`. Consumer (Plan 04) supplies a vetted path. The atomic temp+rename pattern means a half-written file is never visible to readers. |
| T-37-03-03 | Denial of Service | `PersistenceManager.persist` infinite re-entry | mitigate | The `writing` re-entry guard (preserved verbatim from OKM line 9 + 98-99) returns early on concurrent calls. Same guard added to `Exporter.exportNow()`. |
| T-37-03-04 | Tampering | LevelDB content tampering at rest | accept | LevelDB lives under user's `~/.data` (or consumer-supplied `dbPath`); same trust level as any local SQLite/LevelDB. No encryption-at-rest in v0.1. Phase 39+ may revisit if multi-user. |
| T-37-03-05 | Information Disclosure | Temp files (`.tmp.<pid>.<ts>`) leak intermediate writes if process is killed mid-export | accept | Temp files use `process.pid + Date.now()` suffix; orphaned temps are harmless dead files. A janitor sweep is out of scope for v0.1. Documented as known operator concern. |
