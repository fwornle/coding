<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| agent session files → route readers | untrusted/malformed JSONL content (Claude session JSONL, Copilot events.jsonl) crosses here — parsed line-by-line |
| filesystem → route readers | per-file ownership boundary (another user's session file must not be read) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-72-03-FI | Information disclosure | reading another user's session file | mitigate | uid-check gate (`st.uid === process.getuid()`) copied verbatim from claude-token-rows.mjs:82-101 (Claude) and readOwnedFile() (copilot-token-rows.mjs:83-110); non-owned file → []. (V4 access control.) |
| T-72-03-DOS | Denial of Service | malformed JSONL line crashing the close at run-end | mitigate | per-line `JSON.parse` try/catch (claude-token-rows.mjs:145-153) — malformed line skipped, never aborts; parseCopilot gate wrapped in try/catch. A reader that can't read returns [] so the close still completes. (V5/T-69-dos.) |
| T-72-03-POLLUTION | Tampering | prototype pollution via JSON.parse of attacker-influenced session content | mitigate | Only own-uid files are read (T-72-03-FI); parsed objects are read by explicit field access (rec.message.content[], data.toolCallId) — never spread/merged into a prototype-reachable target; no `Object.assign`/`{...rec}` into a shared base. |
| T-72-03-CRYPTO | Information disclosure | inputs_digest over tool arguments (may contain secrets) | mitigate | sha256 digest only via the Plan-01 inputsDigest (node:crypto); raw arguments are NOT carried into RouteEvent beyond the digest + target_path (research recommends inputs_digest, not raw inputs). |
| T-72-03-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs (in-repo modules + stdlib only). No checkpoint needed. |
</threat_model>
