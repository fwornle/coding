<objective>
GUARD-03: replace the silent stderr discard with drain-and-throttle so persistent-worker CLI warnings (e.g. "no stdin data received") surface in the proxy logs without flooding them; plus the WR-02 fold-in (D-08) so the context-leak recycle ceiling measures the true cache-inclusive context the worker carries.

Output: a per-worker throttled stderr logger (<=1 line/min, <=200-char sample, logger threaded via deps with a no-op default), the `_lastInputTokens` token-sum fix at worker-pool.mjs:468, and unit tests proving both (fake clock for throttle, summed-token payload for the ceiling).
