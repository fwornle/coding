<objective>
GUARD-02: pin the `claude` CLI version at worker boot and recycle a worker when the CLI drifts from its boot version, so prompt-cache assumptions don't silently rot when the `claude` binary is upgraded under a long-lived worker.

Purpose: a worker booted against CLI vX keeps serving with vX's prompt-cache behavior even after the operator upgrades `claude` to vY; without drift-recycle the cache assumptions go stale invisibly. This plan makes the worker self-flag for the EXISTING drain+respawn path on drift.

Output: boot-version capture on `ClaudeWorker` (deps-injectable reader), a process-level current-version snapshot on `WorkerPool`, a drift comparison at reuse-time that sets `needsRecycle`, and a unit test that simulates a version change via the injected reader and asserts dispose+respawn.
</objective>
