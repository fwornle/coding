Rewrite the dashboard data pipeline so the Redux store receives typed WorkflowState snapshots from SSE events and stores them directly -- no granular event reconstruction, no inference, no fallback logic in selectors.

Purpose: This is the data foundation for Phase 18. The WebSocket hook currently dispatches ~13 granular event types that get reconstructed into state. The new approach: server sends STATE_SNAPSHOT containing the full WorkflowState, the hook dispatches a single action, and the slice stores it as-is. Selectors use deriveStepStatuses/deriveSubstepStatuses from shared types instead of hand-rolled inference.

Output: Rewritten useWorkflowWebSocket.ts and ukbSlice.ts with STATE_SNAPSHOT handling and derived-status selectors.
