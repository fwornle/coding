The executor discovered that `Project` is not declared in any on-disk `.data/ontologies/*.json` — it is minted at runtime by `ensureProjectAnchor`, not pre-declared. This was reported via Rule 4 and the user approved Option A (soften assertion).

**Verification confirms the softening is correct and complete:**
- No `.data/ontologies/*.json` declares a `Project` class (verified by SUMMARY's class-name sample table).
- Test C instead asserts `domains.length === 8` (all 8 JSONs loaded) + `isValidClass('Component')` (canary from `coding-ontology.json`).
- This still proves the entire loader chain is working (CONTEXT.md verification boundary bullets 1 and 2).
- The `Project` class registration is the documented layer-2 follow-up; it does NOT block 42.1.1's own gate per CONTEXT.md lines 93-94.
