# Phase 42.2 Plan 06: SC#1–6 End-to-End Verification Gate Summary

Closes the four 42-07 deferred-work blocks: runs the SC verifier against the post-everything canonical store, surfaces and fixes the API drift / regression issues that had accumulated since Phase 42-07, and produces the verification report that gates Phase 42 + Phase 42.2 close.

## What was built

- **Verifier rename + 4 patches.** `42-07-end-to-end-verify.mjs` → `verify-knowledge-graph-store.mjs`. Domain filter removed (was hiding the 802 general-domain migrated entities). `iterate(filter, opts)` arg order corrected. SC#2 scope-split (legacy cohort vs in-scope Details). Registry API drift fix (`hasClass` → `isValidClass`, `listDomains` → `getLoadedDomains`).
- **km-core-adapter findEntityByName fix.** The SC#6 root cause: helper used default `iterate()` which drops `validUntil: null` entities via the isActive filter. The migrated Coding Project (legacyId={system:'B',id:'unknown'}, validUntil:null) was therefore unreachable, causing every anchor-pass `storeRelationship('Coding', …)` to silently fail in the try/catch. Fix: pass `{includeSuperseded: true}` opts. Verified live: pre-fix `NOT FOUND`, post-fix `FOUND`.
- **One-shot Coding-anchor backfill script.** `scripts/backfill-coding-anchor-42.2.mjs` — idempotent (`store.addRelation` upsert), fail-loud (5% error budget), mirrors Plan 02's augment-team-field pattern. Useful for any future regression that leaves leveldb-resident entities orphaned. Dry-run on current store: 0 in-scope candidates (because the 18 SC#6 orphans are JSON-only, not in leveldb — see VERIFICATION.md SC#6 root-cause analysis).
- **VERIFICATION.md.** Full SC#1-6 verdict matrix, operator-eyeballing checklist, known-issues list (Wave 4 hang, JSON/leveldb sync, embedding backfill), commit trail.
- **`.gitignore` *.bak.* rule.** Covers the `.bak.old` rename-displaced backup from Plan 05.

## Key decisions

- **Scope expanded mid-plan with justification.** Plan 06 was originally "run verifier + close phase." It grew to include 4 verifier patches + 1 adapter fix + 1 backfill utility as the verifier surfaced API drifts (3) and a real regression (1). All fixes are small and targeted; the alternative was to declare verification deferred, which would have left Phase 42 unclosed.
- **Wave 4 hang is OUT OF SCOPE.** Plan 02's wrapper only covered wave1/2/3 LLM dispatches (the only sites the forensics report identified). Wave 4 InsightGenerationAgent uses the SDK directly. The hang is filed for follow-up but doesn't block Phase 42.2 close.
