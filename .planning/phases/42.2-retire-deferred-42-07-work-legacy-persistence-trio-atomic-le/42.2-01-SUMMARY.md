# Phase 42.2 Plan 01: Forensics — Canonical Emit + LLM Dispatch Audit

Locks the scope of Wave 1 Plan 02 by enumerating every code path that produces a canonical km-core `Entity` in B, every `LLMService.complete()` invocation in the wave-analysis hot path, and the team-field presence on a representative sample of the 802-entity migrated cohort. Per D-Emit (CONTEXT.md): the report is the source-of-truth Plan 02 reads instead of re-running discovery.

## What was built

- **`.planning/forensics/report-42.2-00-canonical-emit.md`** — a 16.6 KB four-section report:
  - **§1 canonical-mapper emit paths.** `toCanonicalEntity` per-field assignment table (canonical-mapper.ts:180-195 + baseMetadata at 143-177). `CanonicalMapperOptions` declared at 69-78 has no `team` field. `grep -n "team" canonical-mapper.ts` → 0 matches. Three production callers enumerated (wave1-project-agent.ts:336, wave2-component-agent.ts:276, wave3-detail-agent.ts:253), all of which have `this.team` in scope but none pass it.
  - **§2 LLM dispatch sites.** 15 `llmService.complete({...})` invocations across 6 files, none pass `process`. SDK surface analysis: `LLMCompletionRequest` (types.ts:35-54) doesn't declare `process`; `ProxyProvider.complete()` (proxy-provider.ts:57-87) doesn't serialize it; server defaults to `'unknown'` (server.mjs:1561). Recommended fix: Option B (single-repo direct-fetch wrapper mirroring `scripts/backfill-raw-observations.mjs:63,95`).
  - **§3 802-entity team-field audit.** Sampled 10 entities from `general.json` (908 nodes); 0/10 carry `team` at top-level or `metadata.team`. Corroborated by migration script grep (zero team refs in `migrate-leveldb-to-kmcore.mjs`).
  - **§4 Actionable gap list for Plan 02.** 11 numbered edits across 4 gaps. Binary verdict: RE-MIGRATION REQUIRED (§4.1). File-budget estimate: 8–10 submodule files, 0 outer-repo files (§4.2).

## Key decisions

- **Bonus gap 4 surfaced:** km-core-adapter.ts:232 silently drops `team` via `_options` dead-arg. Defence-in-depth fix recommended for Plan 02 alongside the primary canonical-mapper fix.
- **wave-controller.ts excluded:** confirmed zero direct LLM dispatch — the 42-07 SUMMARY's mention of "wave-controller LLM dispatch" was misleading; the calls are owned by wave1/wave2/wave3 agents. Plan 02 must NOT edit wave-controller.ts for the LLM gap.
- **RE-MIGRATION REQUIRED:** the 802 (now 908) entities need a one-shot team-augmentation script modeled on `migrate-leveldb-to-kmcore.mjs`. Idempotent, fail-loud 5% error budget.
