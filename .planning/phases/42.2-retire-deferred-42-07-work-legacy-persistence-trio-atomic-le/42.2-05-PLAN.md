## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-42.2-05-01 | Tampering | LevelDB during rename (torn writes) | mitigate | Container stopped at Task 1; no process holds an open file handle on either LevelDB during the rename pair; both `mv` calls are individually atomic on the same filesystem |
| T-42.2-05-02 | Denial of Service | Data loss if rename fails mid-swap | mitigate | Task 1 creates `.bak` snapshot BEFORE any rename; if step 5 fails, rollback recipe restores from `.bak.old` (the original legacy dir, renamed not deleted); both backups retained until Wave 3 SC verifier passes |
