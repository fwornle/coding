## Decisions Made

- **Option B over Option A for the process-attribution wrapper** (per forensics §2.3): created `llm-with-process.ts` direct-fetch wrapper inside the submodule rather than modifying the cross-repo `@rapid/llm-proxy` SDK. - **Metrics tracker passthrough** rather than re-wiring wave-controller's tracer instrumentation: the wrapper accepts a `MetricsTrackerLike` interface (duck-typed) and records every `/api/complete` call into the SDK's tracker so `wave-controller.ts` lines 622, 645, 812, 992, 1762 (`agent.getDetailedCalls()` / `agent.getLLMMetrics()`) keep working unchanged. Avoids touching 5+ wave-controller sites.
- **Augmentation script run live** (not deferred to post-Plan-05): stopped the container, ran the script (181 ms), restarted the container. The 802-entity gap is closed immediately so any Wave-2/3 verifier can rely on `metadata.team` being present. Idempotency means a post-swap re-run is a free no-op.
- **Default canonical-mapper `metadata.team` is `undefined` when no option** (NOT `'coding'`): the plan's Task 1 Test 2 locks this. Production wave agents ALWAYS pass `this.team` so the omit-case is a test-only edge — but defaulting to `'coding'` in the mapper would mask wiring bugs.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Augmentation script needs `includeSuperseded:true` to see the 802 entities**

- **Found during:** Task 3 (augmentation script first dry-run)
- **Issue:** `store.iterate()` returned 0 entities despite `graph:state` in LevelDB containing 802 nodes. Root cause: the 42-05 migration script (commit `1eaaa5e`, line 291) set `validUntil: null` on every entity. km-core's `isActive` helper at `GraphKMStore.js:454-458` returns `false` when `validUntil !== undefined` — `new Date(null).getTime() === 0` is less than `Date.now()`, so every migrated entity is treated as expired and dropped from default iterate. The forensics report didn't catch this because it read `general.json` directly (not via iterate).
- **Fix:** Pass `iterate(undefined, { includeSuperseded: true })` in the augmentation script. Documented as Rule 1 deviation in the script header so future readers don't think it was carelessly added.
- **Files modified:** `integrations/mcp-server-semantic-analysis/scripts/augment-team-field-42.2.mjs`
- **Verification:** Dry-run after fix scanned 802 entities (matches forensics §3.1 baseline); live run augmented 802/802 with 0 errors.
- **Committed in:** `c8c6cc7` (submodule commit)

**2. [Rule 1 - Bug] Augmentation script defaulted `domains: ['coding']` but exports live in `general.json`**

- **Found during:** Task 3 (augmentation script first dry-run, before fix #1)
- **Issue:** Per km-core `GraphKMStore.d.ts:23`, the default `domains` is `['general']`. The 802 migrated entities live in `exports/general.json` (per forensics §3.1: "every migrated entity lives in `general.json`; `coding.json` is empty"). My initial script passed `domains: ['coding']` which excluded the general domain entirely.
- **Fix:** Changed to `domains: ['general', 'coding']` so both domains hydrate. (The dir-swap in Plan 05 may consolidate this — Plan 05's job, not Plan 02's.)
- **Files modified:** `integrations/mcp-server-semantic-analysis/scripts/augment-team-field-42.2.mjs`
- **Verification:** Same dry-run as deviation #1.
- **Committed in:** `c8c6cc7` (submodule commit)

**3. [Rule 3 - Blocking] Container holds LevelDB LOCK on `.data/knowledge-graph-migrated/`**

- **Found during:** Task 3 (before running the augmentation script live)
- **Issue:** `coding-services` container was up (status "Up 23 hours") and `wave-controller.ts:507` points its `dbPath` to `.data/knowledge-graph-migrated/` per Phase 42-07 Phase A, so the LOCK file was held by the container's km-core process. Phase 42-05 SUMMARY documented the same issue for the legacy dir.
- **Fix:** Stopped `coding-services` via `docker-compose stop`, ran the augmentation script (181 ms), then restarted via `docker-compose up -d`. Same atomic-write window protocol the dir-swap plan (Plan 05) will use.
- **Files modified:** None — operational step.
- **Verification:** Container restart logged "Started"; final test pass 37/37 confirmed no functional regression.
- **Committed in:** N/A (operational, not a code change)
