    3. In `server.js`, replace the `setupWorkflowProgressWatcher()` method. The current implementation polls the progress file every 500ms and converts file diffs into WebSocket events. Replace with:

       **New `setupWorkflowEventStream()` method:**
       - Use `EventSource` (from the `eventsource` npm package -- check if already a dependency, if not add it) to connect to `http://localhost:3848/workflow-events`
       - Store `this.lastKnownState = null` to cache the most recent WorkflowState for new WebSocket client connections
       - On `initial-state` event: parse JSON, store in `this.lastKnownState`, broadcast to all WS clients as `{ type: 'STATE_SNAPSHOT', payload: { state: parsedState } }`
       - On `state-change` event: parse JSON, store in `this.lastKnownState`, broadcast to all WS clients as `{ type: 'STATE_SNAPSHOT', payload: { state: parsedState, transition: parsedTransition } }`
       - On EventSource error: log to stderr, EventSource auto-reconnects (built-in behavior)
       - ALSO keep broadcasting the legacy event types (WORKFLOW_STARTED, STEP_STARTED, etc.) derived from the WorkflowState for backward compatibility with the existing dashboard React code. Map from state.status:
         - state.status === 'running' && previous was 'idle' -> broadcast WORKFLOW_STARTED
         - state.status === 'paused' -> broadcast WORKFLOW_PAUSED
         - state.status === 'completed' -> broadcast WORKFLOW_COMPLETED
         - state.status === 'failed' -> broadcast WORKFLOW_FAILED
         - (These legacy mappings will be removed in Phase 18 when dashboard is rewritten)

       **IMPORTANT: Keep the old `handleUKBStream()` SSE endpoint working** -- it's used by the dashboard's `healthRefreshMiddleware.ts`. Only replace the WebSocket event source (file polling -> SSE client). The old SSE endpoint will be removed in Phase 19.

    4. Update `setupWebSocketServer()` to call `setupWorkflowEventStream()` instead of `setupWorkflowProgressWatcher()`

    5. In the WebSocket `connection` handler, when a new client connects, if `this.lastKnownState` is not null, send it as `{ type: 'STATE_SNAPSHOT', payload: { state: this.lastKnownState } }` -- this satisfies SSE-03's reconnection requirement at the WebSocket layer

    6. Check if `eventsource` package is available. If not, use Node.js native `fetch` with streaming (Node 18+ has built-in fetch), or add eventsource to package.json. Alternative: use a minimal SSE client with `http.get()` and manual parsing of `event:` and `data:` lines -- this avoids adding a dependency and is ~30 lines of code.
