    2. Create a module-level broadcaster instance: `const broadcaster = createSSEBroadcaster()`

    3. Register the broadcaster as a state machine subscriber: `subscribe(broadcaster.subscriber.bind(broadcaster))` -- do this at module load, before any routes, so it catches all transitions from the moment the server starts

    4. Add new GET endpoint `/workflow-events`:
       ```
       app.get('/workflow-events', (req: Request, res: Response) => {
         // SSE headers
         res.setHeader('Content-Type', 'text/event-stream');
         res.setHeader('Cache-Control', 'no-cache');
         res.setHeader('Connection', 'keep-alive');
         res.setHeader('X-Accel-Buffering', 'no');
         res.setHeader('Access-Control-Allow-Origin', '*');
         res.flushHeaders();

         // Add client -- broadcaster sends initial-state event with current WorkflowState
         broadcaster.addClient(res, getState());

         // Heartbeat every 15s to keep connection alive
         const heartbeat = setInterval(() => {
           if (!res.writableEnded && !res.destroyed) {
             res.write(': heartbeat\n\n');
           }
         }, 15000);

         // Cleanup on disconnect
         req.on('close', () => {
           clearInterval(heartbeat);
           broadcaster.removeClient(res);
         });
       });
       ```

    5. Update /health endpoint to include `workflowEventClients: broadcaster.clientCount` in the response JSON

    6. Do NOT modify the existing /sse or /messages endpoints -- those are for MCP protocol, completely separate concern

    7. IMPORTANT: Use `import type { Response } from 'express'` if needed for the broadcaster, but sse-server.ts already imports express types. The broadcaster should accept any object with a `.write(chunk: string): boolean` method to stay testable.
