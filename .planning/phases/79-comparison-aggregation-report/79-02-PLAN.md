<objective>
Create the pure aggregator `lib/experiments/compare.mjs` — the heart of Phase 79. It takes the pre-read `readRuns` rows, groups by `variant` within a `task_hash`, applies the D-01 success gate, computes `{mean,stddev,median,min,max,n}` per metric (D-09), ranks successful-gated variants by the composite `total_tokens / rubric_score` ascending (D-05, overridable D-06), and returns a structured report object with the ranked/failed/ungated/unscored groupings (D-13). Delivers CMP-01 (gate separation), CMP-02 (variance), and the aggregation core of CMP-03 (ranking).

Purpose: Never let a failed/ungated/unscored run masquerade as a cheap winner — honesty over ranking completeness. Pure in-memory group-by, no store, no schema migration.
Output: `buildComparison` (+ pure helpers) and a node:test suite driven entirely by synthetic Run+Score fixtures (D-04b — no dependency on live gate-persisted runs).
</objective>

Gate classification (Phase 79 D-01/D-02/D-03, PATTERNS lines 56):
- gate = row.score?.gate_passed ?? null
- null  → UNGATED (no test_command, D-02): shown, NOT ranked
- false → FAILED objective test (D-03): failed section, never averaged into cost
- true  AND terminal_state === 'complete' → SUCCESSFUL: eligible for ranked cost comparison
- terminal_state 'timeout'/'abort' → FAILED (D-03), regardless of gate
