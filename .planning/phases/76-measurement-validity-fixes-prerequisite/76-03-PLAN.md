<objective>
Extend the evidence harness so a non-GSD / ad-hoc run gets all 5 rubric dimensions scored — specifically `code_quality`, `test_coverage`, `regressions` must be non-null when `VERIFICATION.md`/`REVIEW.md` are absent but a working-tree diff exists and the task's test command runs (VALID-03 / O3). Today `gatherEvidence` reads ONLY on-disk GSD artifacts and explicitly never runs tests, so for a straight-coding run the judge has null evidence for these three dims and returns null. The fix derives them deterministically (D-08): (a) the diff (`git diff --stat`) → a `code_quality` signal; (b) running the task's test command → `test_coverage`/`regressions` from exit code + pass/fail counts. These three dims are derived deterministically — NOT by an LLM judge (D-08 / security note) — and overlaid onto the judgment before `writeScore`, honoring the strict null-never-zero calibration `score-write.mjs §42-140` documents.

Purpose: VALID-03 — the "straight vs GSD/SDD" canonical comparison is corrupted while non-GSD runs cannot be scored on these dims. This closes that gap.
Output: An extended `evidence-harness.mjs` (diff-derived `code_quality` + fail-soft test-command run) and a small deterministic-dim overlay wired into both score consumers, with tests.
</objective>
