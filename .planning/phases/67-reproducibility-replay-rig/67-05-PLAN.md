<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| snapshot artifact → filesystem | dirty.patch / untracked files applied into a worktree (untrusted content) |
| restore target → live checkout + live KB | --in-place can overwrite the working environment |
| CLI confirmation → destructive action | human gate for the only destructive path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-67-05-01 | Denial of Service / destructive overwrite | `--in-place` restore over live checkout + KB | mitigate | Auto-backup via captureSnapshot BEFORE any write + explicit typed confirmation (`yes-overwrite-live`); abort-before-write when unconfirmed (test-asserted) — D-05 |
| T-67-05-02 | Tampering | default restore accidentally touching live | mitigate | Default path writes ONLY the sandbox worktree + sandbox LLM_PROXY_DATA_DIR; live-HEAD-unchanged asserted in the live-gated test — D-04 |
| T-67-05-03 | Tampering / injection | git worktree/apply/submodule invocation | mitigate | Fixed-argv spawnSync only; worktree at captured SHA; submodule update --init --recursive (A2 caveat) |
| T-67-05-04 | DoS / corruption | opening store on live KB during restore | mitigate | KB restored via hydrateSandbox into the SANDBOX dir; live single-owner DB never contested (Pitfall 5) |
| T-67-05-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs (RESEARCH Package Legitimacy Audit: N/A) |
</threat_model>
