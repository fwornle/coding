# Plan 06 Summary — E2E coverage + PATTERNS-LOCK + visual smoke + multi-round operator gap closure

The operator smoke surfaced multiple bugs and UX gaps that expanded the plan via two distinct gap-closure cycles with operator approval at each step.

### Round 1 — Bidirectional wiring bugs from operator visual smoke (commits `5511094ec`, `a96683f82`, `de5a3ae08`, merged at `b7732bb19`)

**Bug 1 — Timeline tick click → Graph (missing halo + missing ancestry trace):**
- Root cause: `LslTimelineStrip.onTickClick` wrote raw bucket entityIds to `lslFilterEntityIds`. The visibility predicate then culled all non-structural entities not in that filter — including halo ancestors returned by `pickAllResolvable`. Halo nodes never mounted in D3.
- Fix: union the resolved-ancestor set into the LSL filter on both plain-click (`LslTimelineStrip.tsx:558-560`) and Cmd/Ctrl additive branches (`:504-505`).

**Bug 2 — Graph node click → Timeline (no tick highlight):**
- Root cause: D3 click handler captured `nodeToBuckets` via closure inside the main render effect whose dep list is locked verbatim per Contract #3. When sessions arrived after first paint, the captured value stayed as the empty Map from initial render.
- Fix: `nodeToBucketsRef = useRef(...)` at `D3GraphCanvas.tsx:296` + lightweight `useEffect([nodeToBuckets])` at `:298-300` syncing the ref. Click handler at `:635` reads `nodeToBucketsRef.current.get(d.id)` instead of the closure-captured value. Contract #3 preserved verbatim.

**Test extension:**
- E2E AC#1 bug-1 regression: scans live ticks, asserts every member of `selectedNodeIds` is DOM-mounted with `data-focal`/`data-halo`.
- E2E AC#2 bug-2 regression: iterates `.node` via dispatchEvent, asserts ≥1 graph click produces `selectedBucketKeys.size ≥ 1`.
- Unit `LslTimelineStrip.test.tsx` Test 37: phantom-id fixture asserts `lslFilterEntityIds` contains both raw bucket id AND resolved ancestor.
- Unit `D3GraphCanvas.test.ts` G21: source-grep gate locking `nodeToBucketsRef` pattern + negative grep preventing closure-captured regression.

### Round 2 — Single-node tick UX shortcut (commits `cdaa35387`, `e10b6d7b2`, merged at `30f1bd5c2`)

Operator-added requirement: when a timeline tick resolves to exactly 1 visible node via `pickAllResolvable`, sidebar should jump directly to single-focal mode (EntityDetailPanel), skipping the BucketCardList step that requires clicking the lone card to drill in.

- Placement: Option B in `SidePanel.tsx` mode-switch predicate (relaxed `isSingleFocalMode` from `selectedNodeIds.size === 1 && selectedBucketKeys.size === 0` to just `selectedNodeIds.size === 1`).
- Rationale: grep evidence shows `D3GraphCanvas` no longer subscribes to `selectionSource` (Phase 56 G11 retracted 2026-06-13), so only `BucketCardList` reads it. Option A (mutate `selectedBucketKeys` in `onTickClick`) would have broken Contract #1 focal bucket-ring + BucketCardList's `selectionSource === 'timeline'` candidate-id detection.
- Test coverage: 3 new SidePanel tests covering single-node + multi-node mode-switch.

### Round 3 — Selection-history stack + viewport fit + LLS suppression + drill-preserves-filter (commits `efc2c6afd`, `9eeb9ea5b`, `dd3e5d90b`, restored at `89ee72bcd`, `0d893b785`, `f3c0de774`)

Operator approved three intertwined decisions via the visual-smoke continuation channel:

**Decision 1 — Selection-history stack:**
- NEW `SelectionSnapshot` type + `selectionHistory: SelectionSnapshot | null` store field
- NEW `popSelection()` action: restores snapshot if present; falls through to `clearSelection()` otherwise
- `setSelection({ pushHistory: true })` opt-in arg used by drill writers (BucketCardList card click, D3GraphCanvas halo-node click)
- Esc handler in `useKeyboardShortcuts.ts` and X button in `SidePanel.tsx` both route through `popSelection()` for one-step-back UX

**Decision 2 — Layer 0→1 fit-to-bounds + Contract #3 amendment:**
- NEW separate `useEffect` in `D3GraphCanvas.tsx` keyed on `[selectionSource, selectedNodeIds, pathToSelected]`
- Fires ONLY on Layer 0→1 transition (selectionSource null→non-null AND selectedNodeIds.size ≥ 1)
- Computes bounds over current d3 simulation positions for `selectedNodeIds ∪ pathToSelected` and animates the zoom transform via `svg.transition().duration(500).call(zoomBehaviorRef.current.transform, ...)`
- Does NOT live in the main render dep list, does NOT rebuild SVG, does NOT restart force simulation
- PATTERNS-LOCK Contract #3 amended with explicit Part B clause documenting this single one-shot zoom animation

**Decision 3 — Drill preserves LSL filter:**
- `BucketCardList.onCardClick` PRESERVES `lslFilterEntityIds` from the current store state instead of writing `null`
- Drill collapses selection set but keeps LSL filter in place → visibility stays narrow → main render `useEffect` doesn't fire → no rebuild, no fit, viewport stays where Layer 1's fit put it

**Decision C — LLS suppression in pickAllResolvable + auto-drill on single resolution:**
- `pickAllResolvable` extended with `noiseAncestors: ReadonlySet<string>` parameter (default empty)
- Strip + reverse-index pass a singleton Set containing `LiveLoggingSystem`'s id
- When resolution set has size ≥ 2 and contains LLS, drop LLS (preserves LLS when it's the only result)
- Logger.warn observability per suppression
- Auto-drill: when LslTimelineStrip's `onTickClick` resolves to exactly 1 ancestor (post-suppression), it writes the drill payload directly (single-focal mode) instead of the multi-set payload

This decision replaced the simpler "size === 1 always single-focal" UX shortcut from Round 2 with LLS-aware logic.

### Round 4 — Timeline scale auto-switch bug (commits `dab66ab1d`, `270ba7fca`, `397f795a9`, merged at HEAD~1)

Operator second smoke revealed two symptoms after a tick click:
- Timeline scale auto-switches to `all` even when the click was inside the current 7d/24h/30d window
- After auto-switch, 24h/7d/30d buttons don't work — clicking them re-triggers the auto-slide back to `all`

Root cause: auto-slide `useEffect` keyed on `[selectedTs, windowKey]` at `LslTimelineStrip.tsx:296-346` (pre-fix). When the selection focal becomes a long-lived ancestor (LLS, days/weeks old `createdAt`) via auto-drill, the age check at line 328-333 fires `setWindowKey('all')`. The same loop re-trips on every subsequent scale button click because `selectedTs` is unchanged and `windowKey` is in the dep list.

Fix:
- Commit `dab66ab1d`: gate the selection-active auto-slide on `isNewSelection` (`prevSelectedTsRef.current !== selectedTs`). `windowKey` changes alone no longer re-trip the slide.
- Commit `270ba7fca`: subscribe to `selectionSource`; early-return from the auto-slide when source is `'timeline'` or `'history'` (both originate from tick clicks — the slide would override the operator's deliberate window choice).
- Commit `397f795a9`: 3 new tests (recent-focal-no-slide, old-focal-via-Decision-C-no-slide, graph-source-old-focal-slides-then-sticks).

Contract #3 compliance: auto-slide effect lives in the strip, not the canvas; only writes local `windowKey` state. The main canvas render dep list is untouched.
