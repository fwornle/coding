<objective>
Add `pickAllResolvable(entityIds, visibleIds, relations): Set<string>` to `ancestry.ts` — the multi-resolution sibling of `pickFirstResolvable` that returns the FULL set of visible-ancestor resolutions (deduplicated via Set semantics) instead of stopping at the first. This is the touched-node predicate the timeline strip (Plan 05) writes via and the reverse-lookup hook (Plan 05) consumes for `nodeToBuckets`. Fix WR-02 by extending `computeAncestryPath` to record EVERY parent edge during the BFS rather than only the first-parent-wins linear walk — diamond hierarchies (a node with two visible parents) now have both ancestor edges in the trace, and `resolveToVisibleAncestor` reaches ancestors via secondary parent edges transparently.

Purpose: Plans 03/04/05 depend on `pickAllResolvable` for halo-set computation and for the reverse-index pre-build. The `pickFirstResolvable` signature stays untouched so Phase 56's locked contract #6 carries forward verbatim — Plan 05's timeline strip plain-click path migrates from First to All explicitly, but other Phase 56 consumers (if any) keep working.

Output: `ancestry.ts` with the new exported function + extended BFS; `ancestry.test.ts` with tests for pickAllResolvable (3+ cases), the diamond-hierarchy WR-02 fix (2+ cases), and a regression for the 1-hop fallback orphan-node branch.
</objective>
