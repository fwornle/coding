<objective>
Evolve the Phase 56 selection slice in `viewer-store.ts` from one-to-one (`selectedNodeId: string | null`, `selectedSessionId/StartAt: string | null`) to many-to-many (`selectedNodeIds: ReadonlySet<string>`, `selectedBucketKeys: ReadonlySet<string>`) with derived focal singletons (`focalNodeId`, `focalBucketKey`). Extend `setSelection` to accept Sets/arrays + an explicit focal override, deriving focal from insertion order when absent. Extend the `sameSetMembership` deep-equal guard to the two new Set fields so identical-content writes preserve references (audit-locked invariant — viewport stability). Bring `reset()` to full clear-coverage parity with `clearSelection()` per WR-04.

Purpose: Foundation of Phase 56.1. Every downstream pane (graph, timeline, sidebar) consumes these fields; their plans assume this shape. Without the schema landed first, plans 03/04/05 compile-break against the old `selectedNodeId` accessors.

Output: `viewer-store.ts` with the new multi-selection slice + `viewer-store.test.ts` with initial-state + setter + reset/clear assertions for the new fields. All Phase 45/55 baseline tests must still pass (only the Phase 56 selection-slice tests are rewritten).
</objective>
