## Contract #1 ‚Äî Tick-ring predicate (EVOLVED per D-7 row 1)

**Statement:**

> The tick ring SHOWS on bucket X iff:
> `selectedBucketKeys.has(`${X.id}|${X.startAt}`)` ‚Üí halo (`ring-2 ring-blue-300/60` + `bg-blue-200/40`)
> OR `focalBucketKey === `${X.id}|${X.startAt}`` ‚Üí focal (`ring-2 ring-blue-500` ‚Äî existing Phase 56 focal style)
> OR `(selectedTs !== null && selectedTs >= X.startMs && selectedTs < X.endMs)` ‚Üí reverse-cascade fallback (preserved verbatim from Phase 56)
>
> No local React state shadows any selection-related store field. The `clickedTickKey` useState that Phase 56 round 4 deleted MUST stay deleted.

**Implementation cite:** `integrations/unified-viewer/src/panels/coding/LslTimelineStrip.tsx` ‚Äî two-tier render block (Plan 05 Task 1 step 8 ‚Äî see `56.1-05-SUMMARY.md` ¬ß(c) "Tick render block (two-tier visual)").

**Source-grep gate:** `LslTimelineStrip.test.tsx` Test 56.1-T1 (focal renders `ring-blue-500`) + Test 56.1-T2 (halo renders `ring-blue-300/60` + `bg-blue-200/40`).

**Relaxation requires:** SUMMARY justification against `56.1-CONTEXT.md` D-4 (two-tier visual treatment) + this LOCK contract.

## Contract #2 ‚Äî Graph-side selection (EVOLVED per D-7 row 2)

**Statement:**

> The graph node ring SHOWS on node X iff:
> `focalNodeId === X.id` ‚Üí red ring (`#ff0000`, opacity 1, stroke-width 4) + ancestry trace
> OR `selectedNodeIds.has(X.id) && X.id !== focalNodeId` ‚Üí lighter blue halo ring (`#60a5fa`, opacity 0.6, stroke-width 2), no trace
> ELSE no ring (opacity 0)
>
> `pathToSelected` trace is drawn FROM `focalNodeId` ONLY ‚Äî not from every halo node. Drawing per-halo traces is O(N√ódepth) chaos at N‚â•5.

**Implementation cite:** `integrations/unified-viewer/src/graph/D3GraphCanvas.tsx applySelectionStyling` callback (Plan 03 Task 1 step 2 ‚Äî see `56.1-03-SUMMARY.md` ¬ß(a) "Two-tier render mechanism").

**Source-grep gate:** `D3GraphCanvas.test.ts` G20 (applySelectionStyling useCallback dep list lock) + G15 (two-tier ring colors `#60a5fa` halo + `#ff0000` focal) + G16 (`data-focal` + `data-halo` markers present) + G19 (trace reads `focalNodeId` only).

**Relaxation requires:** SUMMARY justification against `56.1-CONTEXT.md` D-1 (focal derivation) + D-4 (graph two-tier) + this LOCK contract.

## Contract #3 ‚Äî Viewport stability (UNCHANGED ‚Äî AUDIT-LOCKED)

**Statement:**

> The graph viewport DOES NOT animate / re-fit / re-layout on non-graph selection. The `D3GraphCanvas` main render-effect dep list MUST be **verbatim** `[visibleEntities, visibleRelations, theme, isLoading]`. The dep list MUST NEVER contain any of: `selectedNodeIds`, `focalNodeId`, `selectedBucketKeys`, `focalBucketKey`, `pathToSelected`, or any historical Phase 56 single-selection field name (`selectedNodeId`, `selectedSessionId`, `selectedSessionStartAt`).
>
> `visibleEntities` MUST NOT invalidate on tick clicks. The `sameSetMembership` guard for `lslFilterEntityIds`, `selectedNodeIds`, and `selectedBucketKeys` preserves Set references on identical-content writes.

**Implementation cite:** `integrations/unified-viewer/src/graph/D3GraphCanvas.tsx` main render `useEffect` dep list ‚Äî preserved verbatim through Plan 03 (see `56.1-03-SUMMARY.md` ¬ß(c) "Audit-locked dep-list invariant preserved").

**Source-grep gate:** `D3GraphCanvas.test.ts` G9 (dep-list lock + negative gates on all 8 forbidden field names).

**Live E2E gate:** AC #8 in `tests/e2e/unified-viewer/56.1-many-to-many.spec.ts` ‚Äî captures `svg g` transform before tick click and asserts before === after.

**Relaxation requires:** This contract is audit-locked. Relaxation requires CONTEXT.md amendment + operator approval + 2-pane (graph + timeline) live smoke proving viewport invariance still holds via an alternative mechanism.

## Contract #4 ‚Äî 0-obs tick policy (UNCHANGED ‚Äî AUDIT-LOCKED)

**Statement:**

> Sessions whose `observationCount === 0` render with classes `opacity-40 pointer-events-none cursor-default`, carry `aria-disabled="true"`, and are NOT selectable. The `onTickClick` handler early-exits BEFORE any store write when the clicked session is 0-obs. `isDisabled` wins over `isFocalBucket` / `isHaloBucket` in the tick render predicate.

**Implementation cite:** `integrations/unified-viewer/src/panels/coding/LslTimelineStrip.tsx` ‚Äî 0-obs branch preserved verbatim through Plan 05 (see `56.1-05-SUMMARY.md` ¬ß(c) "0-obs early-exit preserved verbatim").

**Source-grep gate (NEW per Plan 05):** `LslTimelineStrip.test.tsx` Test 12 asserts the regex `/opacity-40[^']*pointer-events-none/` is present in the strip source.

**Relaxation requires:** This contract is audit-locked. Relaxation requires CONTEXT.md amendment + operator approval ‚Äî the 0-obs render is a load-bearing UX contract (operator must visually distinguish empty buckets from real buckets).

## Contract #6 ‚Äî Phantom-id resolution (EVOLVED per D-7 row 6 to multi-set)

**Statement:**

> Every member of `selectedNodeIds` (and every focal write) satisfies `visibleIds.has(memberId)`. The resolution helper `pickAllResolvable` returns `Set<string>` of only visible-ancestor resolutions ‚Äî non-resolving inputs do not appear in the output set.
>
> A non-graph writer (timeline tick, sidebar card, history row) calls `pickAllResolvable` (or `pickFirstResolvable` for single-focal contexts) once and writes the resulting set atomically. Phase 56's `pickFirstResolvable` contract is preserved verbatim for backward compatibility ‚Äî Phase 56 callsites that still consume the single-id path continue to compile and behave identically.

**Implementation cite:**
- `integrations/unified-viewer/src/graph/ancestry.ts pickAllResolvable` ‚Äî NEW sibling export (Plan 02 ‚Äî see `56.1-02-SUMMARY.md` ¬ß1).
- `integrations/unified-viewer/src/graph/ancestry.ts pickFirstResolvable` ‚Äî preserved verbatim (Plan 02 backward-compat lock).

**Source-grep gate:**
- `ancestry.test.ts` ‚Äî 4 tests for `pickAllResolvable` (multi-resolution, dedup-collapse, empty-Set on no resolutions, order-independence) + 10 preserved Phase 56 tests covering `pickFirstResolvable`.
- Plan 06 AC #9 source-grep (Contract #7 below) ensures `pickAllResolvable` is not leaked to ad-hoc callsites that bypass the visible-ids guarantee.

**Relaxation requires:** SUMMARY justification against `56-PATTERNS.md` Locked Contract #6 + `56.1-CONTEXT.md` D-2 (touched predicate) + this LOCK contract.

## Contract #7 ‚Äî Pre-index integrity (NEW per D-7 row 7)

**Statement:**

> `nodeToBuckets: Map<string, Set<string>>` is rebuilt **iff** `sessions`, `visibleIds`, or `relations` change. The hook `useNodeToBucketsIndex` is the canonical consumer of this map; `LslTimelineStrip.onTickClick` is the canonical forward-direction click handler that calls `pickAllResolvable` directly.
>
> **Acceptance grep:** active-source callsites of `pickAllResolvable` under `integrations/unified-viewer/src/` (excluding `*.test.*` and `*.spec.*` files) MUST be limited to exactly three paths:
>
> 1. `graph/ancestry.ts` ‚Äî the function definition site.
> 2. `graph/useNodeToBucketsIndex.ts` ‚Äî the pre-index hook (rebuild trigger via `useMemo` on `[sessions, visibleIds, relations]`).
> 3. `panels/coding/LslTimelineStrip.tsx` ‚Äî the forward-direction click handler (D-2 exception per `56.1-PATTERNS.md` ¬ß3).
>
> Any 4th callsite is a smell ‚Äî it indicates someone is scanning buckets on demand instead of reading from the pre-index, which violates O(1)-per-click and re-introduces the "graph re-layouts on every tick click" risk class that the audit's `sameSetMembership` guard was designed to prevent at a downstream layer.

**Implementation cite:**
- `integrations/unified-viewer/src/graph/useNodeToBucketsIndex.ts` ‚Äî Plan 05 Task 1 step 2-3 (see `56.1-05-SUMMARY.md` ¬ß(b)).
- `integrations/unified-viewer/src/panels/coding/LslTimelineStrip.tsx onTickClick` plain-click branch ‚Äî Plan 05 Task 1 step 5 (see `56.1-05-SUMMARY.md` ¬ß(c)).
- `integrations/unified-viewer/src/graph/D3GraphCanvas.tsx` node click ‚Äî Plan 05 Task 2 (D-2 reverse direction reads `nodeToBuckets.get(d.id)` from the pre-index hook; does NOT call `pickAllResolvable` directly ‚Äî see `56.1-05-SUMMARY.md` ¬ß(d)).

**Source-grep gate:**
- AC #9 in `tests/e2e/unified-viewer/56.1-many-to-many.spec.ts` ‚Äî walks `integrations/unified-viewer/src/` (excluding test/spec files) and asserts the unexpected callsite set is empty AND that the 3 allowed callsites are all present.
- `LslTimelineStrip.test.tsx` Test 36 (acceptance grep extended in Plan 05) cross-checks the same invariant at the unit-test layer.

**Relaxation requires:** Plan-level operator approval. Adding a 4th `pickAllResolvable` callsite is an architectural change ‚Äî the alternative (consuming `useNodeToBucketsIndex` or extending the pre-index map shape) preserves O(1)-per-click and does not require a contract relaxation.
