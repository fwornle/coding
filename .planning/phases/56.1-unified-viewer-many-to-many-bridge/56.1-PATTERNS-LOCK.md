## Contract #1 — Tick-ring predicate (EVOLVED per D-7 row 1)

**Statement:**

> The tick ring SHOWS on bucket X iff:
> `selectedBucketKeys.has(`${X.id}|${X.startAt}`)` → halo (`ring-2 ring-blue-300/60` + `bg-blue-200/40`)
> OR `focalBucketKey === `${X.id}|${X.startAt}`` → focal (`ring-2 ring-blue-500` — existing Phase 56 focal style)
> OR `(selectedTs !== null && selectedTs >= X.startMs && selectedTs < X.endMs)` → reverse-cascade fallback (preserved verbatim from Phase 56)
>
> No local React state shadows any selection-related store field. The `clickedTickKey` useState that Phase 56 round 4 deleted MUST stay deleted.

**Implementation cite:** `integrations/unified-viewer/src/panels/coding/LslTimelineStrip.tsx` — two-tier render block (Plan 05 Task 1 step 8 — see `56.1-05-SUMMARY.md` §(c) "Tick render block (two-tier visual)").

**Source-grep gate:** `LslTimelineStrip.test.tsx` Test 56.1-T1 (focal renders `ring-blue-500`) + Test 56.1-T2 (halo renders `ring-blue-300/60` + `bg-blue-200/40`).

**Relaxation requires:** SUMMARY justification against `56.1-CONTEXT.md` D-4 (two-tier visual treatment) + this LOCK contract.

## Contract #2 — Graph-side selection (EVOLVED per D-7 row 2)

**Statement:**

> The graph node ring SHOWS on node X iff:
> `focalNodeId === X.id` → red ring (`#ff0000`, opacity 1, stroke-width 4) + ancestry trace
> OR `selectedNodeIds.has(X.id) && X.id !== focalNodeId` → lighter blue halo ring (`#60a5fa`, opacity 0.6, stroke-width 2), no trace
> ELSE no ring (opacity 0)
>
> `pathToSelected` trace is drawn FROM `focalNodeId` ONLY — not from every halo node. Drawing per-halo traces is O(N×depth) chaos at N≥5.

**Implementation cite:** `integrations/unified-viewer/src/graph/D3GraphCanvas.tsx applySelectionStyling` callback (Plan 03 Task 1 step 2 — see `56.1-03-SUMMARY.md` §(a) "Two-tier render mechanism").

**Source-grep gate:** `D3GraphCanvas.test.ts` G20 (applySelectionStyling useCallback dep list lock) + G15 (two-tier ring colors `#60a5fa` halo + `#ff0000` focal) + G16 (`data-focal` + `data-halo` markers present) + G19 (trace reads `focalNodeId` only).

**Relaxation requires:** SUMMARY justification against `56.1-CONTEXT.md` D-1 (focal derivation) + D-4 (graph two-tier) + this LOCK contract.

## Contract #3 — Viewport stability (AMENDED 2026-06-14 — Plan 06 Decision 2 multi-set fit)

**Statement (post-amendment):**

> **Part A — SVG / force-simulation stability (AUDIT-LOCKED, UNCHANGED):**
> The graph viewport DOES NOT trigger an SVG rebuild or force-simulation restart on selection. The `D3GraphCanvas` main render-effect dep list MUST be **verbatim** `[visibleEntities, visibleRelations, theme, isLoading]`. The dep list MUST NEVER contain any of: `selectedNodeIds`, `focalNodeId`, `selectedBucketKeys`, `focalBucketKey`, `pathToSelected`, `selectionSource`, or any historical Phase 56 single-selection field name (`selectedNodeId`, `selectedSessionId`, `selectedSessionStartAt`). `visibleEntities` MUST NOT invalidate on tick clicks or graph clicks — the `sameSetMembership` guard for `lslFilterEntityIds`, `selectedNodeIds`, and `selectedBucketKeys` preserves Set references on identical-content writes.
>
> **Part B — Viewport zoom transform (AMENDED — Decision 2):**
> The viewport zoom transform MAY animate ONCE on the **Layer 0 → Layer 1 transition** (operator UX feedback 2026-06-14: halo nodes are useless if they sit off-screen). Predicate:
> - previous `selectionSource === null` AND
> - new `selectionSource !== null` AND
> - `selectedNodeIds.size >= 1`
>
> The animation fits `selectedNodeIds ∪ pathToSelected` into the viewport via the same `d3.zoomIdentity.translate(x, y).scale(scale)` primitive the force-simulation auto-fit already uses (`svg.transition().duration(500).call(zoomBehaviorRef.current.transform, transform)`). Implementation lives in a SEPARATE `useEffect` with dep list `[selectionSource, selectedNodeIds, pathToSelected]` — NOT in the main render useEffect; Part A stays inviolate.
>
> **Transitions that MUST NOT trigger the fit:**
> - Layer 1 → Layer 2 drill (selectionSource transitions between non-null values; the multi-set is collapsing, not entering). Drill behaviour preserves the fitted viewport so the focal + path-trace render legibly.
> - Layer 2 → Layer 1 pop (Esc/X via popSelection; selectionSource also transitions between non-null values).
> - Layer 1 → Layer 1 re-selection (additive Cmd-click; selectionSource stays at current value).

**Layer model (Decision 1 selection-history stack — adjacent contract):**
- Layer 0: no selection. `selectionSource === null`, `selectionHistory === null`.
- Layer 1: multi-set entry (tick click halo OR graph node click). `selectionSource ∈ {'timeline', 'graph'}`.
- Layer 2: drill via card click OR halo-node click. `selectionSource === 'history'` OR `'graph'`. `selectionHistory` populated with the pre-drill Layer 1 snapshot.

**Implementation cite:**
- `integrations/unified-viewer/src/graph/D3GraphCanvas.tsx` — main render `useEffect` dep list (Part A); `prevSelectionSourceRef` + fit-to-bounds `useEffect` keyed on `[selectionSource, selectedNodeIds, pathToSelected]` (Part B).
- `integrations/unified-viewer/src/panels/BucketCardList.tsx onCardClick` — drill writer that PRESERVES `lslFilterEntityIds` + `lslSessionFilter` (Decision 3) so `visibleEntities` reference-stable across the drill ⇒ Part A still holds.

**Source-grep gate:**
- `D3GraphCanvas.test.ts` G9 (Part A — main render dep list lock, UNCHANGED).
- `D3GraphCanvas.test.ts` G8 amended (Part B — TWO `transition().duration(500)` callsites: force-sim auto-fit + multi-set fit).
- `D3GraphCanvas.test.ts` G11 amended (Part B — `selectionSource` subscription restored for the transition detector).
- `D3GraphCanvas.test.ts` G22 (NEW — locks the transition predicate `prevSource === null && selectionSource !== null && selectedNodeIds.size >= 1` + the fit useEffect dep list).

**Live E2E gate:** AC #8 in `tests/e2e/unified-viewer/56.1-many-to-many.spec.ts` continues to assert viewport transform stability on subsequent tick clicks (Layer 1 → Layer 1 re-selection has no fit). A future test for the Layer 0 → Layer 1 fit-to-bounds is in the gap-closure backlog.

**Relaxation requires:** Part A is audit-locked. Relaxation of Part A requires CONTEXT.md amendment + operator approval + 2-pane (graph + timeline) live smoke proving viewport invariance still holds via an alternative mechanism. Part B was itself a relaxation (2026-06-14, operator UX feedback); further amendments to Part B require operator approval but not full re-audit — the predicate is bounded by G22 and a regression to "fit on every selection change" would surface as ≥3 `transition().duration(500)` callsites (G8 ratchet).

## Contract #4 — 0-obs tick policy (UNCHANGED — AUDIT-LOCKED)

**Statement:**

> Sessions whose `observationCount === 0` render with classes `opacity-40 pointer-events-none cursor-default`, carry `aria-disabled="true"`, and are NOT selectable. The `onTickClick` handler early-exits BEFORE any store write when the clicked session is 0-obs. `isDisabled` wins over `isFocalBucket` / `isHaloBucket` in the tick render predicate.

**Implementation cite:** `integrations/unified-viewer/src/panels/coding/LslTimelineStrip.tsx` — 0-obs branch preserved verbatim through Plan 05 (see `56.1-05-SUMMARY.md` §(c) "0-obs early-exit preserved verbatim").

**Source-grep gate (NEW per Plan 05):** `LslTimelineStrip.test.tsx` Test 12 asserts the regex `/opacity-40[^']*pointer-events-none/` is present in the strip source.

**Relaxation requires:** This contract is audit-locked. Relaxation requires CONTEXT.md amendment + operator approval — the 0-obs render is a load-bearing UX contract (operator must visually distinguish empty buckets from real buckets).

## Contract #6 — Phantom-id resolution (EVOLVED per D-7 row 6 to multi-set)

**Statement:**

> Every member of `selectedNodeIds` (and every focal write) satisfies `visibleIds.has(memberId)`. The resolution helper `pickAllResolvable` returns `Set<string>` of only visible-ancestor resolutions — non-resolving inputs do not appear in the output set.
>
> A non-graph writer (timeline tick, sidebar card, history row) calls `pickAllResolvable` (or `pickFirstResolvable` for single-focal contexts) once and writes the resulting set atomically. Phase 56's `pickFirstResolvable` contract is preserved verbatim for backward compatibility — Phase 56 callsites that still consume the single-id path continue to compile and behave identically.

**Implementation cite:**
- `integrations/unified-viewer/src/graph/ancestry.ts pickAllResolvable` — NEW sibling export (Plan 02 — see `56.1-02-SUMMARY.md` §1).
- `integrations/unified-viewer/src/graph/ancestry.ts pickFirstResolvable` — preserved verbatim (Plan 02 backward-compat lock).

**Source-grep gate:**
- `ancestry.test.ts` — 4 tests for `pickAllResolvable` (multi-resolution, dedup-collapse, empty-Set on no resolutions, order-independence) + 10 preserved Phase 56 tests covering `pickFirstResolvable`.
- Plan 06 AC #9 source-grep (Contract #7 below) ensures `pickAllResolvable` is not leaked to ad-hoc callsites that bypass the visible-ids guarantee.

**Relaxation requires:** SUMMARY justification against `56-PATTERNS.md` Locked Contract #6 + `56.1-CONTEXT.md` D-2 (touched predicate) + this LOCK contract.

## Contract #7 — Pre-index integrity (NEW per D-7 row 7)

**Statement:**

> `nodeToBuckets: Map<string, Set<string>>` is rebuilt **iff** `sessions`, `visibleIds`, or `relations` change. The hook `useNodeToBucketsIndex` is the canonical consumer of this map; `LslTimelineStrip.onTickClick` is the canonical forward-direction click handler that calls `pickAllResolvable` directly.
>
> **Acceptance grep:** active-source callsites of `pickAllResolvable` under `integrations/unified-viewer/src/` (excluding `*.test.*` and `*.spec.*` files) MUST be limited to exactly three paths:
>
> 1. `graph/ancestry.ts` — the function definition site.
> 2. `graph/useNodeToBucketsIndex.ts` — the pre-index hook (rebuild trigger via `useMemo` on `[sessions, visibleIds, relations]`).
> 3. `panels/coding/LslTimelineStrip.tsx` — the forward-direction click handler (D-2 exception per `56.1-PATTERNS.md` §3).
>
> Any 4th callsite is a smell — it indicates someone is scanning buckets on demand instead of reading from the pre-index, which violates O(1)-per-click and re-introduces the "graph re-layouts on every tick click" risk class that the audit's `sameSetMembership` guard was designed to prevent at a downstream layer.

**Implementation cite:**
- `integrations/unified-viewer/src/graph/useNodeToBucketsIndex.ts` — Plan 05 Task 1 step 2-3 (see `56.1-05-SUMMARY.md` §(b)).
- `integrations/unified-viewer/src/panels/coding/LslTimelineStrip.tsx onTickClick` plain-click branch — Plan 05 Task 1 step 5 (see `56.1-05-SUMMARY.md` §(c)).
- `integrations/unified-viewer/src/graph/D3GraphCanvas.tsx` node click — Plan 05 Task 2 (D-2 reverse direction reads `nodeToBuckets.get(d.id)` from the pre-index hook; does NOT call `pickAllResolvable` directly — see `56.1-05-SUMMARY.md` §(d)).

**Source-grep gate:**
- AC #9 in `tests/e2e/unified-viewer/56.1-many-to-many.spec.ts` — walks `integrations/unified-viewer/src/` (excluding test/spec files) and asserts the unexpected callsite set is empty AND that the 3 allowed callsites are all present.
- `LslTimelineStrip.test.tsx` Test 36 (acceptance grep extended in Plan 05) cross-checks the same invariant at the unit-test layer.

**Relaxation requires:** Plan-level operator approval. Adding a 4th `pickAllResolvable` callsite is an architectural change — the alternative (consuming `useNodeToBucketsIndex` or extending the pre-index map shape) preserves O(1)-per-click and does not require a contract relaxation.
