### (b) useNodeToBucketsIndex hook (Task 1 steps 2-4)

NEW `integrations/unified-viewer/src/graph/useNodeToBucketsIndex.ts` — pre-builds the reverse `Map<nodeId, Set<bucketKey>>` once per data-load. The useMemo dep list is `[sessions, visibleIds, relations]` — Contract #7 pre-index integrity. Composition:

```typescript
const visibleIds = useVisibleEntityIds(apiClient, system)
const { relations } = useGraphData(apiClient, system)
const { data: sessions } = useLslSessions(apiClient)
return useMemo(() => {
  const map = new Map<string, Set<string>>()
  for (const bucket of sessions ?? []) {
    if (typeof bucket.startAt !== 'string') continue
    const bucketKey = `${bucket.id}|${bucket.startAt}`
    const touched = pickAllResolvable(bucket.entityIds ?? [], visibleIds, relations)
    for (const nodeId of touched) {
      let existing = map.get(nodeId)
      if (!existing) { existing = new Set<string>(); map.set(nodeId, existing) }
      existing.add(bucketKey)
    }
  }
  return map
}, [sessions, visibleIds, relations])
```

5 vitest cases cover: empty map; single touched node; multi-bucket accumulation per node; parallel reverse entries per bucket; no-resolution buckets contribute no entries.
