**Focal-derivation rule (NEW invariant, 56.1 contract #2):**

```typescript
// Insertion-order semantics — last id added is focal.
// Computed inside setSelection (and clearSelection) so the action is the
// single writer site. Consumers NEVER write focalNodeId directly.
function deriveFocal(
  prev: ReadonlySet<string>,
  next: ReadonlySet<string>,
  explicit: string | null | undefined,
): string | null {
  if (explicit !== undefined) return explicit  // caller override wins
  if (next.size === 0) return null
  // "Last added" = element in next NOT in prev. If multiple new (set replace),
  // pick the LAST one in iteration order (Set preserves insertion order).
  const added: string[] = []
  for (const id of next) if (!prev.has(id)) added.push(id)
  if (added.length > 0) return added[added.length - 1]
  // No new additions (caller passed identical set or subset): preserve current
  // focal if it's still a member; else fall back to last element of next.
  // Phase 56.1 plan author picks the exact "preserve" semantics — flag as
  // discretion point if unclear.
  let last: string | null = null
  for (const id of next) last = id
  return last
}
```

### 2. `src/graph/ancestry.ts` — EXTEND with `pickAllResolvable` + diamond BFS (WR-02)

**Analog:** itself. The existing `pickFirstResolvable` (lines 174-184) is the EXACT shape to mirror; `pickAllResolvable` returns `Set<string>` instead of `string | null`.

**Existing `pickFirstResolvable` signature (lines 174-184) — the SHAPE to mirror:**

```typescript
/**
 * Walk a bucket's `entityIds[]` and return the first id (or its closest
 * visible ancestor) that lands in the graph's visible set.
 *
 *  - For each id in `entityIds`, calls `resolveToVisibleAncestor`.
 *  - Returns the FIRST non-null resolution.
 *  - Returns `null` when no id in the bucket has any visible ancestor —
 *    the sidebar-only mode the timeline strip uses to render the bucket
 *    selection without a graph target.
 */
export function pickFirstResolvable(
  entityIds: readonly string[],
  visibleIds: ReadonlySet<string>,
  relations: readonly Relation[],
): string | null {
  for (const id of entityIds) {
    const resolved = resolveToVisibleAncestor(id, visibleIds, relations)
    if (resolved !== null) return resolved
  }
  return null
}
```

**56.1 sibling — `pickAllResolvable` (NEW, mirror the contract):**

```typescript
/**
 * Walk a bucket's `entityIds[]` and return the SET of resolved ids
 * (each id resolved to its closest graph-visible ancestor).
 *
 *  - Mirrors `pickFirstResolvable` — same predicate, same ancestry walk —
 *    but accumulates ALL resolutions instead of stopping at the first.
 *  - Returns an EMPTY Set when no id in the bucket has any visible ancestor
 *    (the sidebar-only / halo-empty mode).
 *  - Dedup-safe: a Set is the right return type so duplicate resolutions
 *    (two raw entity ids that resolve to the same ancestor) collapse.
 *
 * Used by:
 *   - `LslTimelineStrip.onTickClick` to compute selectedNodeIds halo
 *     (Forward direction — D-2 in 56.1 CONTEXT.md)
 *   - `useNodeToBucketsIndex` to compute the reverse `nodeId -> bucketKeys`
 *     map (D-3 in 56.1 CONTEXT.md)
 */
export function pickAllResolvable(
  entityIds: readonly string[],
  visibleIds: ReadonlySet<string>,
  relations: readonly Relation[],
): Set<string> {
  const out = new Set<string>()
  for (const id of entityIds) {
    const resolved = resolveToVisibleAncestor(id, visibleIds, relations)
    if (resolved !== null) out.add(resolved)
  }
  return out
}
```

**WR-02 — `computeAncestryPath` diamond-hierarchy fix:**

**Current implementation (lines 37-98) only follows the FIRST parent per child:**

```typescript
const visited = new Set<string>([selectedId])
const queue: string[] = [selectedId]
const parentOf = new Map<string, string>()
while (queue.length) {
  const cur = queue.shift() as string
  const parents = childToParents.get(cur) ?? []
  for (const p of parents) {
    if (visited.has(p)) continue
    visited.add(p)
    if (!parentOf.has(cur)) parentOf.set(cur, p)   // <-- "first parent wins" — WR-02 root cause
    queue.push(p)
  }
}
let node = selectedId
let depth = 0
nodeDepths.set(node, depth)
while (parentOf.has(node)) {
  const parent = parentOf.get(node) as string
  edges.add(`${parent}||${node}`)
  edges.add(`${node}||${parent}`)
  depth++
  nodeDepths.set(parent, depth)
```

**56.1 fix (WR-02) — record ALL parent edges + walk every reachable ancestor:**

```typescript
// 56.1 WR-02 fix: record EVERY parent edge during BFS so diamond/
// multi-parent hierarchies have all ancestor edges in the trace. The
// previous "first parent wins" + linear walk hid the secondary-parent
// edges from the visual trace, which made `resolveToVisibleAncestor`
// silently miss visible ancestors reachable only via secondary edges.
const visited = new Set<string>([selectedId])
const queue: string[] = [selectedId]
nodeDepths.set(selectedId, 0)
while (queue.length) {
  const cur = queue.shift() as string
  const curDepth = nodeDepths.get(cur) as number
  const parents = childToParents.get(cur) ?? []
  for (const p of parents) {
    // Record EVERY parent edge — not just the first one per child.
    edges.add(`${p}||${cur}`)
    edges.add(`${cur}||${p}`)
    if (visited.has(p)) continue
    visited.add(p)
    nodeDepths.set(p, curDepth + 1)
    queue.push(p)
  }
}
const pathLength = Math.max(0, ...Array.from(nodeDepths.values()))
```

**56.1 — `useNodeToBucketsIndex.ts` (NEW), same scaffold:**

```typescript
// Phase 56.1 — reverse-lookup pre-index for graph node -> touching buckets.
//
// Per CONTEXT.md D-3, when the user clicks a graph node, the timeline must
// light up every bucket whose entityIds resolve (via pickAllResolvable) to
// that node. We pre-build the reverse map ONCE per data-load instead of
// scanning all buckets on each click.
//
// Rebuild triggers (dep list):
//   - sessions (initial fetch / refetch)
//   - visibleIds (filter changes — class/level/layer/learning source)
//   - relations (graph refetch)
//
// Memory budget (CONTEXT.md D-3 estimate): ~50 visible nodes × ~3 touching
// buckets/node = ~150 entries; well under 1MB.
//
// Contract:
//   useNodeToBucketsIndex(apiClient, system): ReadonlyMap<string, ReadonlySet<string>>
//     - Keys: visible node ids
//     - Values: Set of `${bucketId}|${startAt}` composite keys
//     - Reference-stable across renders with identical content (useMemo's
//       Map is the same object until any dep changes)

import { useMemo } from 'react'
import type { ApiClient } from '@/api/ApiClient'
import type { System } from '@/config/system-endpoints'
import { useViewerStore } from '@/store/viewer-store'  // not strictly required;
                                                       // but included if writer needs it
import { useGraphData } from './useGraphData'
import { useVisibleEntityIds } from './useVisibleEntityIds'
import { pickAllResolvable } from './ancestry'

// `useSessions` or equivalent must surface the LSL sessions cache; if the
// existing strip uses TanStack Query directly, hoist a `useSessions` hook
// (see CONTEXT.md "estimated plan structure" Wave 1).
import { useSessions } from '@/panels/coding/useSessions'

export function useNodeToBucketsIndex(
  apiClient: ApiClient,
  system: System,
): ReadonlyMap<string, ReadonlySet<string>> {
  const visibleIds = useVisibleEntityIds(apiClient, system)
  const { relations } = useGraphData(apiClient, system)
  const sessions = useSessions(apiClient, system)  // OR pass in as prop

  return useMemo<ReadonlyMap<string, ReadonlySet<string>>>(() => {
    const map = new Map<string, Set<string>>()
    for (const bucket of sessions) {
      const bucketKey = `${bucket.id}|${bucket.startAt}`
      const touched = pickAllResolvable(bucket.entityIds, visibleIds, relations)
      for (const nodeId of touched) {
        let existing = map.get(nodeId)
        if (!existing) {
          existing = new Set<string>()
          map.set(nodeId, existing)
        }
        existing.add(bucketKey)
      }
    }
    return map
  }, [sessions, visibleIds, relations])
}
```

## Locked Design Contracts (re-statement of Phase 56's 6 + new contract #7)

Per CONTEXT.md D-7, 6 of Phase 56's contracts need re-statement for the
many-to-many shape. The audit-locked invariants (#3 viewport stability,
#4 0-obs tick policy) carry forward verbatim.

### Contract #1 — Tick-ring predicate (EVOLVED)

> The tick ring SHOWS on bucket X iff:
> `selectedBucketKeys.has(${X.id}|${X.startAt})` (halo: ring-blue-300/60)
> OR `focalBucketKey === ${X.id}|${X.startAt}` (focal: ring-blue-500)
> OR `(selectedTs !== null && selectedTs >= X.startMs && selectedTs < X.endMs)` (reverse cascade — preserved)
>
> No local React state shadows any selection-related store field. The
> `clickedTickKey` useState was already deleted in Phase 56 round 4.

### Contract #2 — Graph-side selection (EVOLVED)

> The graph node ring SHOWS on node X iff:
> `focalNodeId === X.id` (red ring, ancestry trace)
> OR `selectedNodeIds.has(X.id) && X.id !== focalNodeId` (lighter blue halo, no trace)
>
> `pathToSelected` trace is drawn FROM focalNodeId ONLY. The ancestry
> walk reads the store's `pathToSelected` first; inline `computeAncestryPath`
> is the fallback for mount-time first paint.

### Contract #3 — Viewport stability (UNCHANGED, AUDIT-LOCKED)

> The graph viewport DOES NOT animate / re-fit / re-layout on non-graph
> selection. The D3GraphCanvas main render effect dep list MUST be
> `[visibleEntities, visibleRelations, theme, isLoading]` ONLY — never
> `selectedNodeIds`, `focalNodeId`, `selectedBucketKeys`, `focalBucketKey`,
> `pathToSelected`. `visibleEntities` MUST NOT invalidate on tick clicks
> — the `sameSetMembership` guard for `lslFilterEntityIds` (and now also
> `selectedNodeIds`, `selectedBucketKeys`) preserves Set references on
> identical-content writes.

### Contract #4 — 0-obs tick policy (UNCHANGED, AUDIT-LOCKED)

> Sessions whose `observationCount === 0` render `opacity-40 pointer-events-none
> cursor-default`, carry `aria-disabled="true"`, and are NOT selectable.
> The click handler in `onTickClick` early-exits before any store write.

### Contract #5 — Single-source-of-truth (UNCHANGED in spirit; field renames propagate)

> NO local React state may shadow a selection-related store field. All
> cross-pane selection writes route through `setSelection` /
> `clearSelection`. Acceptance grep: zero `useViewerStore.setState({...})`
> call sites in any consumer component that participates in selection.

### Contract #6 — Phantom-id resolution (EVOLVED to multi)

> Every member of `selectedNodeIds` (and every focal write) satisfies
> `visibleIds.has(memberId)`. The resolution helper is extended to
> `pickAllResolvable` which returns `Set<string>` (only visible-ancestor
> resolutions, filtered). A non-graph writer (timeline tick, sidebar card)
> calls `pickAllResolvable` once and writes the resulting set atomically.

### Contract #7 — Pre-index integrity (NEW, additive)

> `nodeToBuckets: Map<string, Set<string>>` is rebuilt iff `sessions`,
> `visibleIds`, or `relations` change. Node click reads from the pre-index,
> never scans buckets on demand. Acceptance grep: any code path calling
> `pickAllResolvable` from inside a click handler is a smell.
