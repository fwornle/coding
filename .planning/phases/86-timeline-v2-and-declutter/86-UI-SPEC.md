## Spacing Scale

The dashboard uses Tailwind's default 4px-based scale via utility classes (`px-1.5`, `px-2`, `px-3`, `py-0.5`, `py-1`, `py-2`, `gap-1`, `gap-1.5`, `gap-2`, `gap-3`, `pl-4`, `pl-5`, `mt-4`). All multiples of 4 (Tailwind `1`=4px), with the 6px half-steps (`1.5`=6px, `0.5`=2px) reserved for badge/chip inner padding — an established, pre-existing exception this phase reuses verbatim.

| Token | Value | Usage (this phase) |
|-------|-------|--------------------|
| 2xs | 2px (`0.5`) | Badge vertical padding (`py-0.5`), chip mt |
| xs | 4px (`1`) | Icon gaps (`gap-1`), chip inline gaps |
| — | 6px (`1.5`) | **Existing exception** — badge/chip horizontal padding (`px-1.5`), swatch gaps (`gap-1.5`) |
| sm | 8px (`2`) | Compact element spacing (`gap-2`, `px-2`), tool-chip row gaps |
| md | 12px (`3`) | Row inner padding (`px-3 py-2`), between-badge gaps (`gap-3`) |
| lg | 16px (`4`) | Sub-band indent (`pl-4`), section padding, tab body top margin (`mt-4`) |
| xl | 24px (`6`) | Card padding, panel gaps |
| 2xl | 32px+ (`8`+) | Page-level section breaks |

**Mini per-turn band height:** 8px (`h-2`) — glanceable, matches the existing role swatch scale (`h-3 w-1`). **Cumulative band height:** 12–16px (`h-3`/`h-4`) in modal/fullscreen (taller = readable segment labels on hover).

Exceptions: 6px (`1.5`) badge/chip padding is inherited from shadcn `badge.tsx` (`px-2 py-0.5`) and existing timeline chips — **do not "correct" to 8px**; conforming to the existing surface is the higher-priority rule for an extension phase.

## Typography

Inter throughout (`src/index.css:16`). The dashboard uses `text-2xl` for the page H1 and a **two-role** body/mono system for data rows. Numeric/token/byte values are ALWAYS `font-mono` with `tabular-nums` (the `.item-progress-badge` rule sets `font-variant-numeric: tabular-nums`). Only two weights appear in the performance surfaces: regular (400) and semibold/bold (600–700).

| Role | Size | Weight | Line Height | Usage (this phase) |
|------|------|--------|-------------|--------------------|
| Page heading | `text-2xl` (24px) | `font-bold` (700) | 1.2 (default) | "Performance" H1 (unchanged); fullscreen view title |
| Body / label | `text-sm` (14px) | 400 | 1.5 (`--line-height` root) | Turn-row prompt excerpt, tool-chip labels, tab labels, quarantine toggle |
| Data / mono | `text-sm` (14px) `font-mono` | 400, tabular-nums | 1.5 | Token counts, cache read/write, byte sizes, cumulative token deltas |
| Micro / meta | `text-xs` (12px) / `text-[10px]` | 400/500 | 1.4 | Badge text, chip themes, "N/A" note, "no per-turn context captured" note, raw-preview meta |

**Rules:**
- Prompt excerpt: single line, `truncate` (existing idiom `truncate font-mono text-xs`), `text-sm` weight 400, `text-foreground`.
- All numbers `font-mono` tabular-nums — cumulative token deltas in the diff viewer MUST align vertically.
- Never introduce a 5th size or a 3rd weight. If a display size is wanted for the fullscreen header, reuse `text-2xl font-bold`.

## Color

**Do NOT invent a palette.** The dashboard is fully theme-tokenized (light + dark) via CSS vars. The 60/30/10 split maps to the existing shadcn semantic tokens; the "accent" 10% for THIS phase is the small set of new interactive/status elements listed below. All colors are theme-aware — light values shown; dark equivalents already defined in `src/index.css`.

| Role | Value (token) | Usage |
|------|---------------|-------|
| Dominant (60%) | `--background` `hsl(0 0% 100%)` / dark `hsl(222 84% 4.9%)` (`bg-background`) | Page canvas, timeline panel surface |
| Secondary (30%) | `--card` / `--muted` / `--secondary` (`bg-card`, `bg-muted`, `border-muted`) | Turn-row cards (`rounded-md border`), sub-bands, sidebar, table rows |
| Accent (10%) | `--primary` (buttons/CTA) + the semantic status tokens below | See reserved-for list |
| Destructive | `--destructive` `hsl(0 84% 60%)` (`bg-destructive` / existing `bg-red-*` remapped in dark) | Delete-runs confirm ONLY (already exists; unchanged) |

**Accent reserved for (explicit — never "all interactive elements"):**
- **Primary CTA** ("Compare selected (n)") → `--primary` filled `Button` (default variant).
- **Quarantine toggle count** → neutral `Badge variant="secondary"`; the *toggle-on* state uses the existing checkbox `--primary`.
- **"edited" score badge** (D-11) → **reuse the EXISTING** `border-yellow-200 bg-yellow-50 text-yellow-700` (`runs-table.tsx:138`) — do not re-theme.
- Everything else stays on `secondary`/`muted`/`outline` — chips, tier badges, process pills are all `Badge variant="secondary"` or `variant="outline"` (existing).

### Context-window band palette (D-05) — REUSE `SEGMENTS`, do not fork

Import `SEGMENTS` + `scaledBand` from `context-cache-explainer.tsx` (RESEARCH Pattern 4). One taxonomy across both surfaces (D-05). Verbatim palette:

| Category key | Label | fill | stroke | cached |
|--------------|-------|------|--------|--------|
| `sys` | System Instructions | `#d9f99d` | `#84cc16` | yes |
| `tools` | Tool Descriptions | `#cffafe` | `#06b6d4` | yes |
| `know` | Retrieved Knowledge | `#e9d5ff` | `#a855f7` | yes |
| `hist` | Conversation History | `#fecaca` | `#ef4444` | yes |
| `tout` | Tool Outputs | `#fed7aa` | `#f97316` | no |
| `user` | User Input | `#bfdbfe` | `#3b82f6` | no |

- **Segment LENGTH** encodes category byte share (via `scaledBand`, which floors tiny-but-real segments to ≥1.2% and renormalizes to 100). Do not re-derive.
- **Cached-vs-fresh OVERLAY** (D-05): render the `cache_read` portion of each segment as a **hatched / reduced-opacity overlay** on the same segment (opacity ~0.55 fill under a 45° `repeating-linear-gradient` hatch at ~4px pitch, stroke unchanged). The overlay length = `usage.cache_read` share within that category. Legend: solid = fresh, hatched = cache-read. Choose hatching over pure opacity so it survives colorblind + greyscale.
- **Honesty gate (LOCKED):** when `usage.cache_write === null` (OpenAI-wire discriminator), render the cache-write portion as the verbatim constant `CACHE_WRITE_NA` = `"N/A (provider reports no cache-creation)"` (import from `context-cache-explainer.tsx:189`) — **never `0`, never an inferred fill.** No amber cache-write segment for copilot/opencode runs.

### Status-badge vocabulary & colours (Claude's discretion — PINNED HERE, honest + accessible)

Reconciliation badge (D-12) and loop badge (D-09) get a fixed, accessible vocabulary using the **existing semantic status tokens** (`text-status-*` / `bg-status-*-subtle` / `border-status-*-line`, single-source light/dark parity — `src/index.css`). Never a raw green/amber that skips the dark remap.

| Badge | State | Token | Icon (lucide) | Copy |
|-------|-------|-------|---------------|------|
| Reconciliation | reconciled (`flaggedCount===0 && fallback===0 && matched>0`) | `status-success` | `Check` | `✓ reconciled` |
| Reconciliation | discrepancy (`flaggedCount>0`) | `status-warning` | `AlertTriangle` | `⚠ Δ discrepancy` |
| Reconciliation | transcript-fallback (`fallback>0`) | `status-neutral` | `FileText` | `transcript-fallback` |
| Reconciliation | absent (`reconciliation === null`) | — | — | **no badge** (D-06 honesty — never a placeholder) |
| Loop (advisory) | possible loop | `status-warning` (subtle, `variant="outline"`) | `Repeat` | `possible loop` |

- **Loop badge is advisory, NEVER asserted** (D-09): use the *subtle/outline* rendering (`border-status-warning-line text-status-warning`), NOT a filled destructive badge — the heuristic is fuzzy. Tooltip copy is mandatory (see Copywriting).
- Semantic tokens are colorblind-supported by pairing colour with a distinct **icon per state** — never colour alone.
- Header reconciliation note (detailed, on the open run's timeline header, D-12) reuses the explainer's existing note phrasing, e.g. `"wire 12.0k = transcript 12.0k, 0 discrepancies"` — read verbatim from the reconciliation summary, never recomputed.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (diff viewer entry, D-08) | **"Compare selected (2)"** — verb + noun + live count; disabled unless exactly 2 runs selected, tooltip when disabled: "Select two runs to compare." |
| Turn-row open affordance (D-01) | Whole row is the click target (mirrors existing `CollapsibleTrigger`); no separate label. Cursor `cursor-pointer`. |
| Fullscreen button (D-02) | Icon button `Maximize2` + `aria-label="Open fullscreen timeline"`; tooltip "Fullscreen timeline". |
| Empty state — no runs | Heading: "No runs yet" · Body: "Runs appear here after an experiment or a measured session completes." (reuse existing runs-table empty pattern if present; do not fabricate data.) |
| Empty state — no context-turns (D-06 degradation) | Inline subtle note, `text-xs text-muted-foreground`: **"no per-turn context captured"** — rendered on the v1 row; band, cache split, and tool digests are simply absent. Never an error, never a spinner-forever. |
| Empty state — diff viewer, identical runs | "These two runs are identical — no divergence to show." (when `firstDivergence === null`). |
| Empty state — diff viewer, single run selected | "Select a second run to compare." |
| Error state — context-turns fetch fails | `text-sm text-status-error`: "Couldn't load per-turn context for this run. It may be a pre-Phase-84 run — showing the v1 timeline." (fall through to D-06 degradation, not a dead panel.) |
| Error state — score PATCH rejected (D-11) | 400 → inline under the cell, `text-xs text-status-error`: "{server validation message}". 404 → "This score changed on the server — reopen the run to edit." Revert the optimistic value on non-2xx (RESEARCH Pattern 5). |
| Cache-write N/A note | `CACHE_WRITE_NA` verbatim: "N/A (provider reports no cache-creation)" — with a tooltip: "This provider's wire protocol doesn't report cache creation; we show N/A rather than a fabricated 0." |
| Loop badge tooltip (mandatory) | "possible loop — this turn repeats a recent tool signature; args are matched fuzzily, so this is a hint, not a fact." |
| Reconciliation header note | Verbatim from summary: "wire {w} = transcript {t}, {n} discrepancies" — never recomputed client-side. |
| Inline score edit affordance (D-11) | Cell shows the value; on focus/edit becomes an `Input` (numeric); autosave on blur/Enter. "edited" `Badge` (existing yellow) marks an override; tooltip "Judged: {n}". |

**Destructive actions in this phase:** NONE new. The only destructive action on the page (delete runs) already exists with its confirm dialog (`confirmOpen` in `runs-table.tsx`) and is **not modified** by this phase. Inline score edits are server-re-validated, non-destructive overrides (reversible — the judged value is retained and shown in the tooltip).

## New Surface Contracts (this phase's deliverables)

Prescriptive contracts for each NEW surface. Executor implements from these; checker/auditor validate against them.

### 1. Timeline v2 turn row (D-01) — evolves `timeline.tsx` ParentRow

Anatomy, left→right, single scannable line inside the existing `rounded-md border` card (`px-3 py-2`):
1. Role/tier: existing `TierBadge` (`granularity-tier-badge`) + role swatch — **MUST survive** (DASH-02 regression anchor).
2. Prompt excerpt: 1 line, `truncate text-sm`, `text-foreground`.
3. Tool-name chips: `Badge variant="secondary" text-xs` per `tool_use` name (from `messages[].tool.name`); overflow → "+N" chip. Never show raw args here (D-03 — name only in the row).
4. Token/cache summary: `font-mono text-sm` in+out total; cache read/write as `text-xs text-muted-foreground` (existing idiom), N/A honesty applies.
5. Mini context band: `h-2` segmented bar (§ band contract), right-aligned or under the summary.
6. Advisory loop badge (D-09) when `loopFlags[i]` true.
Whole row opens the drill-down modal (D-01). Reasoning sub-bands (`children`) remain collapsible (DASH-02). `ProcessPill` preserved.

### 2. Single-turn drill-down modal (D-01) — Radix `Dialog`, mirror `KbDetailDialog`

`DialogContent className="max-w-[760px] w-[90vw] max-h-[85vh] overflow-y-auto"` (RESEARCH code example). Contents: full message list; per `tool_use` name + arg-size + ETM observation intent line (D-03 semantic-first); full arg text ONLY when `capture_raw_bodies` was ON (else name+size+intent, no fabrication); cache-breakpoint positions; per-message byte sizes (`font-mono`); raw preview passed through `scrubSecrets` (security V7). Focus-trap/ESC/scroll-lock from Radix — do not hand-roll. `data-testid="turn-modal"`.

### 3. Fullscreen whole-run timeline (D-02) — react-router child route

Route `/performance/timeline/:taskId` (full viewport). Header: `text-2xl font-bold` run title + reconciliation note (D-12) + back affordance (router). More turns visible; keyboard nav (↑/↓ move turn focus, Enter opens modal, Esc/back exits). Renders the SAME v2 turn components + the **cumulative** band (not the mini). Distinct purpose from the modal (whole run vs one turn) — not the same content at two sizes.

### 4. Context-window band (D-04/D-05) — CSS width-% div segments

Reuse `scaledBand`/`SEGMENTS`. **Mini** (row): `h-2`, byte-share WITHIN the turn (relative composition, not absolute window size). **Cumulative** (modal/fullscreen): `h-3`/`h-4`, per-turn category bytes stacked cumulatively across the run (history accreting); width = cumulative / final-turn bytes. Cached overlay + N/A honesty as in Color §. No SVG, no recharts (RESEARCH Pattern 4 — recharts `fill` can't use `var()`).

### 5. Difference viewer (D-07/D-08) — new component, reuses turn components + bands

Entry: runs-table multi-select (`toggleRunSelected`) → "Compare selected (2)" CTA (D-08). Lives as a Compare-tab surface beside `run-compare.tsx` (extend or sit beside — Claude's discretion, default: new surface in the Compare tab). Layout: two columns from first divergence; identical prefix (`[0, prefixLen)`) COLLAPSED by default via existing Radix `Collapsible` ("Show N identical turns"); per-aligned-pair cumulative token delta `font-mono` (+green `status-success` / −neutral or muted, sign always shown); loop badges on either side; canonical-model header read VERBATIM (`run.canonical_model`, null → `italic "unmeasured"` — ATTR-02 regression anchor). Alignment computed by pure `run-align.ts` (no network). One-sided (insert/delete) turns render in one column with the other side dimmed/empty.

### 6. Declutter IA

- **D-10 quarantine → page header:** remove from `faceted-sidebar.tsx`; place near SummaryCards/table header. Control = existing `Checkbox` + label **with count**: "Show quarantined (3)". Count from the quarantined-run query (Claude's discretion on the exact query; default: `runs.filter(quarantined).length`). `data-testid="include-pending-toggle"` preserved.
- **D-11 inline score cells:** `ScoreCell` becomes edit-in-place — click/focus → shadcn `Input` (numeric, `text-sm font-mono`, `w-16`), autosave on blur/Enter via the EXISTING `saveOverride` PATCH (server re-validates; client never authoritative). "edited" badge (existing yellow) + judged tooltip retained. Keep `score-drawer.tsx` mounted as a fallback for bulk edits.
- **D-12 reconciliation badge:** row badge (vocabulary above) in the runs-table row + detailed note on the open run's timeline header. `null` → no badge.

### 7. Graceful degradation (D-06) — explicit fallback state

Runs with no context-turns data render today's v1 row (turn label + tokens + observation lines) with v2 enrichments (band, cache split, tool digests, modal) **absent** and the subtle note "no per-turn context captured" (`text-xs text-muted-foreground`). Nothing is lost; the run stays readable. This is the DESIGNED path, not an error — no error styling.
