    Each `this.updateProgress(...)` call becomes a typed `dispatch()` call. Use the following mapping:

    - **State transitions** (actual state changes via high-level events):
      - Step completion: `dispatch({ type: 'step-complete', stepName: 'wave1', nextStep: 'wave2', duration })` — called once per wave completion
      - Pause at single-step boundary: `dispatch({ type: 'pause', reason: 'single-step-boundary' })`
      - Failure: `dispatch({ type: 'fail', error, step })` — on catch blocks
      - Completion: `dispatch({ type: 'complete', summary })` — at end of executeWaveAnalysis

    - **Substep tracking** (progress within running state via substep-update):
      - `subPhase: 'init'` on wave 1 → `dispatch({ type: 'substep-update', substepId: 'wave1_init', wave: 1, totalWaves })`
      - `subPhase: 'analyze'` on wave N → `dispatch({ type: 'substep-update', substepId: 'waveN_analyze', wave: N, totalWaves })`
      - `subPhase: 'qa'` on wave N → `dispatch({ type: 'substep-update', substepId: 'waveN_qa', wave: N })`
      - `subPhase: 'qa_retry'` on wave N → `dispatch({ type: 'substep-update', substepId: 'waveN_qa_retry', wave: N })`
      - `subPhase: 'classify'` on wave N → `dispatch({ type: 'substep-update', substepId: 'waveN_classify', wave: N })`
      - `subPhase: 'persist'` on wave N → `dispatch({ type: 'substep-update', substepId: 'waveN_persist', wave: N })`
      - `subPhase: 'operators'` → `dispatch({ type: 'substep-update', substepId: 'wave3_operators_<name>' })`
      - `subPhase: 'insights'` → `dispatch({ type: 'substep-update', substepId: 'wave4_insights' })`
