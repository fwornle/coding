  1. **Deterministic seed dataset** — a JSON file ingested into a fresh OKM instance produces a known graph (entities with fixed UUIDs, relations, both `layer:'evidence'` and `layer:'pattern'` represented, multiple ontology classes touched). The seed must use OKM's existing ingest path so it exercises the current pipeline shape. Use fixed UUIDv7 strings (not freshly minted) so re-running the seed produces identical IDs.

  2. **Fixture recorder script** (`scripts/record-rest-fixtures.mjs`) — operator-runnable; spins up an in-process OKM server, ingests the seed, hits each REST endpoint in D-G5.1's list, writes the JSON response to `tests/fixtures/pre-migration/api-<endpoint>.json`. Idempotent: a second run produces byte-identical files.

  3. **Zod contract tests** (`tests/integration/rest-contract.test.ts`) — per-endpoint Zod schemas; for each endpoint, the test (a) ingests the same seed, (b) hits the endpoint, (c) calls `<EndpointSchema>.parse(res.body)` (throws on shape drift), (d) calls `expect(res.body).toEqual(JSON.parse(await readFile(fixturePath)))` (byte-equal lock).

**The fixtures + Zod tests CAN pass in this plan** — the storage cutover hasn't happened yet, so the pre-cutover responses become the locked baseline. After Plan 08's cutover, Plan 10 re-runs the SAME tests; byte-equal pass = D-G5.1 SC#3 satisfied. If shape drifted (different field name, different array ordering, different ID format), Plan 10 fails loudly with a Zod / toEqual diff.

**Anti-shallow guard:** the seed dataset MUST be small enough to round-trip cleanly (5-15 entities, 3-7 relations is plenty) but DIVERSE enough to exercise every endpoint. Specifically: at least 2 distinct ontology classes, at least 1 entity per layer ('evidence' AND 'pattern'), at least 1 cluster-forming pair, at least 1 RCA-traceable chain.

Out of scope: VOKB viewer smoke (Plan 10 covers it post-cutover).

Purpose: Lock REST contract before any storage cutover; provide the SC#3 evidence Plan 10 will produce.

Output: ~12 new files in tests/fixtures + tests/integration + scripts/; zero source changes to src/.
