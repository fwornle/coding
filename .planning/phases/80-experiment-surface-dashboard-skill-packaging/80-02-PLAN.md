## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator → experiment skill → spawned CLI | user-supplied `--goal`/`--variants`/`--agents`/`--repeats` flow into a synthesized spec YAML and a spawned `node scripts/*.mjs` process |
| skill → filesystem (spec YAML write) | the synthesized spec is written under config/experiments/ |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-80-02-01 | Elevation of Privilege | user --goal/--variants/--agents → spawned CLI argv | mitigate | Skill passes flags as discrete argv to `node scripts/experiment-run.mjs` (no shell string interpolation); reject shell metacharacters (`;` `` ` `` `$(` `&&` `\|` newline) in --goal before it becomes goal_sentence, mirroring the spec's "fixed argv, no shell meta" test_command discipline |
| T-80-02-02 | Tampering | synthesized spec YAML path | mitigate | Derive the spec filename from a sanitized experiment_id (path-safe allowlist), write only under config/experiments/; no user-controlled path traversal into the write target |
| T-80-02-03 | Information Disclosure | skill prose | accept | Skill is documentation distributed to the operator's own agents; no secrets embedded (env knobs are named, not valued) |
| T-80-02-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs — the skill shells to existing scripts; if any install is proposed, halt for a legitimacy checkpoint |
</threat_model>
