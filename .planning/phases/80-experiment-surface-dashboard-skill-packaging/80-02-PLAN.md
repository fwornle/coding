task_hash derivation — the ONE handoff fact (scripts/measurement-stop.mjs:805-806):
  const taskHash = span.goal_sentence
    ? crypto.createHash('sha256').update(span.goal_sentence).digest('hex')
    : null;
  → task_hash is sha256(goal_sentence). It is CONSTANT across every cell of one matrix (all cells share
    the spec's goal_sentence). It is persisted only in Run.metadata.task_hash (lib/experiments/run-write.mjs:102),
    NOT emitted on stdout. There is NO exported derivation helper — the sha256 line above is inline in
    measurement-stop.mjs, and exporting it would touch the Phase-78 runner engine (OUT of scope this phase).
    So the skill computes the SAME sha256 itself: it already OWNS the goal_sentence (it authored the spec),
    so `sha256(goal_sentence hex)` is deterministic, store-free, and equals the persisted task_hash.

Spec YAML shape (config/experiments/compare-fizzbuzz.yaml — the canonical, FULLY-POPULATED example):
  version: 1
  experiment_id: <id>
  snapshot_id: smoke-spec           # a RESOLVABLE baseline id (must exist; absent/bad → no restore baseline)
  goal_sentence: "<full goal>"      # ← the task_hash source (sha256 of THIS string)
  repeats: N
  task_class: new-feature           # closed-taxonomy class; absent → Run quarantines as `unclassified`
  test_command: "<score-time gate, fixed argv, no shell meta>"
  variants:
    - { agent: claude,   model: sonnet,                         framework: straight, env: default }
    - { agent: opencode, model: rapid-proxy/claude-haiku-4-5,  framework: straight, env: default }
    - { agent: copilot,  model: auto,                           framework: straight, env: default }

Closed task_class taxonomy (lib/experiments/taxonomy.mjs:26 CLOSED_6):
  refactor | bugfix | new-feature | migration | debug | docs

  (run-write.mjs:39,103) → hidden from the default dashboard runs query → EMPTY comparison. So the spec MUST
  carry a member of CLOSED_6.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator → experiment skill → spawned CLI | user-supplied `--goal`/`--variants`/`--agents`/`--repeats`/`--task-class`/`--snapshot-id` flow into a synthesized spec YAML and a spawned `node scripts/*.mjs` process |
| skill → filesystem (spec YAML write) | the synthesized spec is written under config/experiments/ |
| skill → task_hash compute → compare argv | the goal_sentence is hashed (sha256) skill-side and the hex becomes the `--task-hash` argv for the compare CLI (which uses it as a report filename) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-80-02-01 | Elevation of Privilege | user --goal/--variants/--agents/--task-class → spawned CLI argv | mitigate | Skill passes flags as discrete argv to `node scripts/experiment-run.mjs` (no shell string interpolation); reject shell metacharacters (`;` `` ` `` `$(` `&&` `\|` newline) in --goal before it becomes goal_sentence, mirroring the spec's "fixed argv, no shell meta" test_command discipline |
| T-80-02-02 | Tampering | synthesized spec YAML path | mitigate | Derive the spec filename from a sanitized experiment_id (path-safe allowlist), write only under config/experiments/; no user-controlled path traversal into the write target |
| T-80-02-04 | Tampering | task_hash → compare report filename | mitigate | task_hash is a sha256 hex digest ([0-9a-f]{64}) computed skill-side from goal_sentence — it is inherently path-safe (no separators/traversal). The compare CLI additionally re-validates it via `sanitizeTaskHash` (allowlist regex) before it becomes `.data/experiments/reports/<hash>.json`, so a malformed hash is rejected, not written |
| T-80-02-03 | Information Disclosure | skill prose | accept | Skill is documentation distributed to the operator's own agents; no secrets embedded (env knobs are named, not valued) |
| T-80-02-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs — the skill shells to existing scripts and uses stdlib sha256 (shasum / node:crypto); if any install is proposed, halt for a legitimacy checkpoint |
</threat_model>
