## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dashboard/operator → GET /api/experiments/comparison | untrusted `task_hash` + `rank_by` query params cross into a store read |
| endpoint → experiment LevelDB (via openExperimentStore) | read-only store access on the single-owner LevelDB |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-80-01-01 | Tampering | task_hash query param → store read | mitigate | Validate with the reused `sanitizeTaskHash` allowlist (`^[A-Za-z0-9._-]+$`, rejects `..`/`/`/`\`/null) and return 400 BEFORE opening the store; the route writes no filename, so path-traversal has no sink, but validation is enforced anyway |
| T-80-01-02 | Information Disclosure | comparison JSON | accept | Report contains aggregate token/score metrics for the operator's own experiments; no PII, same trust level as existing `/api/experiments/runs` |
| T-80-01-03 | Denial of Service | rank_by param → buildComparison | mitigate | `rank_by` is only consumed as a sort key by buildComparison (bounded enum-like set; unknown values fall through to default ordering); endpoint is read-only open→read→close, no unbounded work |
| T-80-01-04 | Repudiation | 79→80 schema drift | mitigate | gate_outcome stamping is a SINGLE shared `withGateOutcomes`/`GROUP_GATE_OUTCOME` in compare.mjs imported by both CLI and endpoint; the store-backed test asserts endpoint JSON deep-equals the CLI writeReportJson doc |
| T-80-01-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this plan (endpoint reuses existing express/km-core deps); if any install is proposed, halt and require a legitimacy checkpoint |
</threat_model>
