<objective>
Build the LLM judge: a single structured `/api/complete` Haiku call that, given the
consequential trace events + on-disk evidence + goal_sentence, returns per-event
toward/neutral/away labels AND the 5-dimension rubric with rationales (D-02/D-03).
The ratio is computed deterministically in code from the returned labels (not trusted
to LLM arithmetic). Trivial runs (D-04) skip the call; proxy/parse failures (D-03)
quarantine to all-null + pending without ever throwing — the close must never block.

Purpose: ROUTE-03 (goal_aligned_ratio) + SCORE-01 (rubric synthesis), with the judge
call itself token-measured for free because it traverses the proxy (Phase 68).
Output: judge.mjs (prompt + schema + defensive parser) + test suite.
</objective>

<threat_model>
External surface: an OUTBOUND call to the local proxy (port 12435). Trust boundary:
judge → rapid-llm-proxy.
- T-73-04-PARSE (Tampering/DoS): the proxy returns a STRING content that may be
  malformed/oversized — mitigate by parse-in-try/catch + shape validation + range
  clamp, quarantining to pending on any failure (asserted Task 2); AbortSignal.timeout
  bounds a hung proxy.
- T-73-04-BLOCK (DoS): a slow/unreachable proxy must not block run-close — mitigate by
  try/catch → pending (NEVER throw); timeout bounded (reuse REQUEST_TIMEOUT_MS).
- T-73-04-LEAK (Information disclosure): raw tool inputs must not be sent to the judge —
  mitigate by packing only seq/tool_name/target_path/outcome (no inputs/file contents).
- T-73-04-PORT (Misconfiguration): wrong port (3033) silently returns HTML — mitigated by
  fixed 12435 + the Haiku live-probe acceptance.
No package installs (T-73-SC N/A — judge reuses fetch + existing proxy).
</threat_model>
