## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → PATCH /api/experiments/scores/:taskId | untrusted JSON body (dimension, value, overridden_by) crosses here |
| handler → dedicated experiment LevelDB | single-owner store write via applyOverride |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-73-05-01 | Tampering | request body `value` | mitigate | range-validate per dimension (0-1, or 0|1 for regressions) at handler entry; reject → 400 before any store write (Task 2) |
| T-73-05-02 | Tampering | request body `dimension` | mitigate | allowlist the 5 dimension names; unknown → 400 (also applyOverride throws as defense-in-depth) |
| T-73-05-03 | Injection | `overridden_by` / stored rationale | mitigate | type-check string + length cap; stored as plain km-core metadata (no eval/SQL sink); reject non-string |
| T-73-05-04 | Spoofing/AuthZ | who may override | accept | no auth on the existing vkb-server surface (same posture as PUT /api/entities); localhost-bound internal dashboard API; overridden_by stamp records actor; revisit with Phase 74 UI |
| T-73-05-05 | Repudiation | override provenance | mitigate | persist overridden_by + overridden_at stamp on every override (applyOverride) |
| T-73-05-06 | DoS / Replay | repeated PATCH | accept | idempotent last-write-wins on corrected_<dim>; bounded single-store write; no amplification |
| T-73-05-07 | Tampering (integrity) | judged fields | mitigate | applyOverride writes ONLY corrected_* slots; judged fields read-and-preserved (asserted in 73-02 + this plan's judged-untouched test) |
| T-73-05-SC | Tampering | npm installs | mitigate | no new packages introduced this plan (no install task); reuses existing express + km-core |
</threat_model>
