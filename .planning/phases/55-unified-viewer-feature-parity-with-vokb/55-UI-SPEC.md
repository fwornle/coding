# Phase 55 — UI Design Contract (Unified Viewer Feature Parity with VOKB)

> Visual and interaction contract for closing the ~85% surface gap between the unified viewer (Phase 45 scaffold) and the legacy VOKB. Generated by `gsd-ui-researcher`, consumed by `gsd-planner`, `gsd-executor`, and `gsd-ui-checker`.
>
> **Scope:** 12 ported VOKB surfaces + 4 coding-specific additions + Markdown/Entity panel harmonization + encoding overlay schema extension + CAP-tab removal + OKB routing fix. Per D-55-02a, "roughly similar from screenshots" is **insufficient** — every ported surface must be implemented against the VOKB source files referenced in the Component Inventory table.
>
> **Inheritance:** All Phase 45 UI-SPEC decisions (design tokens, spacing scale, typography, base color contract, theme system, shadcn preset, icon library, keyboard shortcuts, layout shell) carry forward verbatim. Phase 55 only documents **additions and amendments**.

## 2. Spacing Scale

Inherited verbatim from Phase 45 UI-SPEC. 8-point scale, all multiples of 4.

| Token | Value | Phase-55 Specific Usage |
|-------|-------|------------------------|
| xs | 4px | Stats-bar inter-metric padding, ontology tree row indent step, sparkline tick gap |
| sm | 8px | Filter group internal gap, sub-tab pill padding, LSL timeline tick gap, ETM live-tail item gap |
| md | 16px | Default container padding (panels, cards), gap between top-level filter sections, gap between sub-tab content and Entity header |
| lg | 24px | Page-shell padding around stats bar, section gap between Evidence/Provenance/Neighbors/Sources panels in Entity detail |
| xl | 32px | Vertical rhythm between major surface modes (graph canvas ↔ Issue Triage mode shell) |
| 2xl | 48px | Empty-state vertical padding (no-data states across all 16 surfaces) |

**Phase 55 exceptions (additive to Phase 45):**

- **Stats bar height:** 40px (`h-10`). Single-row horizontal strip; never wraps. Items separated by `gap-6`.
- **Ontology tree row height:** 24px (`h-6`). Compact density per VOKB convention (`OntologyFilter.tsx:107-123` — `space-y-1`, `text-xs`, `w-3 h-3` checkbox).
- **Sparkline cell:** 60px wide × 16px high (`w-15 h-4`). Inline trend visualization fits inside a list row.
- **Sub-tab pill height:** 24px (`h-6`). Matches VOKB `NodeDetails.tsx:894-935` `px-2 py-0.5 text-[10px]` — same compact density.
- **LSL timeline strip height:** 32px (`h-8`). Single-row horizontal scroll; not a separate scrollable panel.
- **ETM tail max height:** 480px (`max-h-[30rem]`). Caps the sidebar; older items scroll out of view.

## 3. Typography

Inherited verbatim from Phase 45 UI-SPEC. 4 sizes, 2 weights.

| Role | Size | Weight | Line Height | Phase-55 Specific Use |
|------|------|--------|-------------|----------------------|
| Body | 14px (`text-sm`) | 400 | 1.5 | Entity sub-tab content, occurrence list rows, evidence link labels |
| Label | 12px (`text-xs`) | 500 | 1.4 | Stats-bar metric labels, filter group headers, sparkline counts, sub-tab labels |
| Heading | 20px (`text-xl`) | 600 | 1.2 | Side-panel section titles ("Relationships", "Sources & Evidence", "Occurrence History") |
| Display | 24px (`text-2xl`) | 700 | 1.2 | Entity name in Entity detail header (preserved from Phase 45) |

**Phase 55 micro-type exceptions (verbatim VOKB convention, preserved on port):**

- **`text-[10px]`** for: confidence-percentage badges, run-id chips, occurrence-count footers, sparkline trend numbers (`7d/30d/90d`), filter sub-group headers ("Execution Model", "Failure Model", "RaaS", "KPI-FW"). VOKB source: `OntologyFilter.tsx:131-187`, `NodeDetails.tsx:1117-1138`, `TrendingPanel.tsx:80-92`. NOT a new size token — `[10px]` is a documented "fine-print exception" treated as a sub-token of `text-xs` for high-density status metadata.
- **`text-[9px]`** for "all | none" select-toggle controls inside filter group headers (`OntologyFilter.tsx:139-187`). Same exception class.
- **`text-[8px]`** for filter-group disclosure triangles only (`OntologyFilter.tsx:136`). NOT used for text content.

**`tabular-nums`** MUST be applied to every numeric span in:
- Stats bar (all 7 metrics)
- Sparkline counts
- ETM tail timestamps
- Workflow progress percentages
- Confidence percentage badges
- Occurrence/connection counts in lists

**Surface positioning rules:**

- **FilterRail (existing):** w-64 expanded, w-12 collapsed. - **SidePanel (existing):** w-96 default, w-[30rem] when Markdown tab is active. Phase 55 adds sub-tabs INSIDE the Entity tab (Default/Evolution/Confidence/Timeline) and Markdown harmonization (metadata header).
- **RCA tab (existing):** REMOVED. The tab disappears entirely — not hidden, not placeholder.
- **ETM tail (NEW, coding-only):** Right-side overlay via shadcn `<Sheet>` primitive. Triggered by a 📡 icon button in NavBar (next to keyboard help). Docked behavior: when LIVE indicator is on AND `system === 'coding'`, auto-opens on first observation arrival; user can dock or dismiss.
- **LSL Timeline strip (NEW, coding-only):** h-8, between main content and Footer. Horizontal scroll with time-window controls (last 24h / 7d / 30d). Hidden when `system !== 'coding'`.
- **Footer (existing):** Phase 45 unchanged.
- **Workflow Status panel (NEW, coding-only):** h-10 collapsed (chip-only view), expandable Card via shadcn `<Collapsible>`. Lives below Footer. Hidden when `system !== 'coding'`.

**Responsive breakpoints:**

- ≥1280px (default): all surfaces visible, no horizontal scroll.
- 1024-1279px: FilterRail auto-collapses to w-12; SidePanel stays w-96.
- 768-1023px: SidePanel becomes a drawer (shadcn `<Sheet>` from right); LSL Timeline strip horizontal-scrolls.
- <768px: Single-column stack (NavBar, StatsBar, canvas, drawer-triggered side surfaces). Mode switch persists.

## 12. LIVE Indicator & Pulse Behavior

**Where:** Top-right of the StatsBar (last metric slot).

**When LIVE shows `LIVE` (vs `Polling`):**

- The viewer attempts SSE connection on mount (`GET /api/v1/stream` if available; coding-specific stream for ETM at `/api/coding/observations/stream`).
- If SSE handshake succeeds: `state = 'LIVE'`. Pulse dot is rendered, 1.5s `pulse` animation cycle (Tailwind `animate-pulse` with `bg-emerald-500` — accent reserved use).
- If SSE handshake fails or disconnects: `state = 'Polling'`. Dot is gray (`bg-muted-foreground`), no animation. A poll cycle every 30s keeps stats fresh.
- Reconnection: exponential backoff (1, 2, 4, 8, 16s capped). Each attempt logs to `Logger.debug(LogCategories.NETWORK)`.

**Pulse rule per node (D-55-03 `pulseRule` overlay):**

The encoding overlay's `pulseRule` field is an expression evaluated against each entity. Supported expressions in Phase 55 v1:

- `null` (default) — no pulse
- `lastUpdatedWithin:60s` — node pulses if `Date.now() - new Date(entity.updatedAt).getTime() < 60_000`
- `lastUpdatedWithin:5m` — same with 5-minute window
- `recentlyMerged:1h` — node pulses if any `entity.metadata.occurrences[].timestamp` is within 1 hour

**Pulse animation:** SVG halo ring around the node, opacity `0 → 0.5 → 0` over 1500ms, radius `nodeSize+4 → nodeSize+12 → nodeSize+4`. Color = node fill color at 50% opacity. Renderer: extend `SigmaCanvas.tsx` node program with a halo layer. Disabled when `prefers-reduced-motion: reduce` is set — falls back to a static 50%-opacity ring.

**No pulse beyond the rule:** Phase 55 does NOT introduce general "attention attractors" (e.g., a "scroll wheel waiting for click" pulse on every empty state). Pulse is reserved for the LIVE indicator (1 instance) and the D-55-03 pulse-rule mechanism (per-node, opt-in via overlay).

**CRITICAL — error copy honesty (Phase 45 G03 inheritance, D-55-01c):**

The viewer MUST classify network failures and produce the right copy:

```ts
function classifyNetworkError(err: unknown): 'dns' | 'cors' | 'tls' | 'timeout' | 'http-status' | 'unknown' {
  if (!(err instanceof TypeError)) return 'unknown'
  const msg = err.message.toLowerCase()
  // Chrome/Firefox/Safari error message patterns
  if (msg.includes('failed to fetch') && msg.includes('cors')) return 'cors'
  if (msg.includes('failed to fetch') && /name|dns|resolve/.test(msg)) return 'dns'
  if (msg.includes('tls') || msg.includes('certificate') || msg.includes('ssl')) return 'tls'
  if (msg.includes('timeout')) return 'timeout'
  return 'unknown'
}
```

A genuine CORS error means the response was received but lacked `Access-Control-Allow-Origin`. A DNS error means the hostname did not resolve. The viewer MUST NOT confuse them in user-facing copy. If the `classify` returns `'unknown'`, the generic "Cannot reach {label} backend at {baseUrl}" copy is used (see § 5).

**Loading skeletons:** use `<Skeleton>` shadcn primitive (if absent, `npx shadcn add skeleton`). Match section dimensions to avoid layout shift.
