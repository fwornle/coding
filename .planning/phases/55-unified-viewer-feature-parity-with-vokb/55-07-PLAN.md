<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SSE consumer (StatsBar) → backend `/api/v1/stream` | server-side trust; received events parsed via `JSON.parse` in try/catch; malformed events silently dropped |
| URL `?mode=triage` → Zustand state | user-controlled URL parameter; validated against the `'kg' | 'triage'` union before applying |
| Lazy import path `./TriagePlaceholder` ← UnifiedViewer | static string literal; Vite resolves at build time; 55-10 replaces with another static literal |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-55-07-01 | Tampering | malicious URL `?mode=arbitrary-value` injected | mitigate | validate against `'kg' | 'triage'` literal union before calling `setMode`; unknown values default to `'kg'` |
| T-55-07-02 | XSS | StatsBar renders numeric values from backend | accept | React text-node escaping; values are numbers (Number-coerced) |
| T-55-07-03 | Denial of service | runaway SSE reconnect loop | mitigate | exponential backoff capped at 16s; tested in Task 1 |
| T-55-07-04 | Information disclosure | Legend exposes internal entity-type categorization | accept | legend reflects the same encoding the user already sees on nodes; no sensitive info |

Package legitimacy gate: this plan installs 3 shadcn primitives (`popover`, `toggle-group`, `skeleton`). All are from the **official shadcn registry** (UI-SPEC §20 — "shadcn official only"; safety gate not required per registry vetting rule).
</threat_model>
