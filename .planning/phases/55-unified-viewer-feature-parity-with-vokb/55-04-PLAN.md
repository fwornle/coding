<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SSE consumer → `pushObservation` action | observations from `/api/coding/observations/stream` arrive client-side; Zustand store buffers them. Untrusted input crosses here in the sense of "can an upstream backend flood the store?" — yes, mitigated by the 100-item ring buffer cap. |
| localStorage persistence (Phase 45 pattern) | If Phase 45's Zustand store is configured with persistence middleware, new slices inherit that — verify in implementation that coding-only slices (etmObservations, etmStreamConnected) are EXCLUDED from persistence (ephemeral by nature). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-55-04-01 | Denial of service | unbounded `etmObservations` array growth | mitigate | hard ring-buffer cap at 100 (per `<behavior>` Task 2 test); UI burst-debounce in EtmTailSheet (55-12) coalesces >10/sec into 250ms batches |
| T-55-04-02 | Information disclosure | localStorage persists user's filter preferences across reloads (e.g., `selectedLayers`) | accept | filter prefs are not sensitive; matches Phase 45 baseline (`memory feedback_logger_class.md` — localStorage Logger state is precedent) |
| T-55-04-03 | Tampering | XSS via Observation payload rendered in EtmTailSheet | mitigate (downstream) | mitigated in 55-12 panel rendering (React text-node escaping); store stores raw text, panel escapes on render |

Package legitimacy: no new packages.
</threat_model>
