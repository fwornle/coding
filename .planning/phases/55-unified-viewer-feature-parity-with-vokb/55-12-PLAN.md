## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SSE `/api/coding/observations/stream` → frontend ring buffer | server-side trust; React text-node escapes render |
| polling `/api/ukb/status` on :3033 → frontend | localhost-only; Health API trust class |
| coding-only gate | enforced at TWO layers (component-level + mount-level) per defense-in-depth |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-55-12-01 | Cross-system data leak | OKB tab renders ETM observations (coding data) | mitigate | gate `system === 'coding'` at component + mount; OKB never receives the SSE connection |
| T-55-12-02 | XSS | observation summary text rendered in row | mitigate | React text-node escaping; observation text is truncated server-side as well |
| T-55-12-03 | Denial of service | infinite SSE reconnect loop after permanent backend failure | mitigate | exponential backoff capped at 16s; tested in Task 1 |
| T-55-12-04 | Denial of service | runaway polling for workflow status | mitigate | idle-skip after 5min; resumes on user interaction |
| T-55-12-05 | Information disclosure | LIVE indicator timing reveals server-state to all clients | accept | localhost-only; same class as obs-api SSE Phase 50 / 51 patterns |

Package legitimacy: no new packages (`lucide-react` already Phase 45 dep per UI-SPEC §1).
</threat_model>
