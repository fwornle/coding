**Decision evidence gathered in the Plan 44-16 pre-flight audit:**

1. **Pre-cutover SQL handler** (git history of `scripts/observations-api-server.mjs`, documented inline at `lib/km-core/src/adapters/observation-view.ts:74-75` and `95-96`) explicitly aliased columns to camelCase: `observation_ids AS observationIds`, `files_touched AS filesTouched`, `digest_ids AS digestIds`, `last_updated AS lastUpdated`, `created_at AS createdAt`. None of them touch snake_case.

3. **Post-cutover adapter output** (Plan 44-05) emits camelCase by design: `digestToLegacy` returns `{observationIds, filesTouched, createdAt}`; `insightToLegacy` returns `{digestIds, lastUpdated, createdAt}`.

4. **The Wave 0 RED stub** (`tests/integration/typed-views.test.js`, authored before Plan 44-07 mounted the endpoints) asserted snake_case based on the SQL column names alone, **without consulting the SQL-alias contract**. Its `REQUIRED_DIGEST_KEYS` + `REQUIRED_INSIGHT_KEYS` lists were a spec error.

**Decision: lock camelCase as the canonical wire shape for `/api/coding/{digests,insights}`.** Plan 44-16 rewrites the test to assert camelCase, lands a load-bearing LOCKED-contract comment in the adapter, and writes 44-CONTEXT-amendment-4.md ratifying the choice.

Dual-emit (additive snake_case aliases alongside camelCase) was considered and rejected — see T-44-16-01. The single-shape contract is the cleanest deliverable.

## Why dual-emit was rejected

| Option | Cost | Benefit | Verdict |
|--------|------|---------|---------|
| **Lock camelCase, rewrite test** (chosen) | 1 test file edit, 1 amendment doc, 1 adapter comment block. No runtime change. | Wire stays as-is. Dashboard works. Future planners read a clear contract. | ✓ Cleanest. Matches "no parallel versions" lib/km-core/CLAUDE.md rule. |
| Lock snake_case, rewrite dashboard | 10 TS reader sites + 4 wire-key shifts in `observation-view.ts` + km-core rebuild + dashboard rebuild + risk of missed reader. | Aligns with stored metadata field names. Cleaner for Python consumers. | ✗ Breaks production dashboard; high risk. |
| Dual-emit (additive snake_case keys alongside camelCase) | Adapter emits 5 extra keys per Digest/Insight row. ~10 % bytes-on-wire growth. Defers the decision. | Zero consumer breakage either direction. | ✗ Projects decision deferral; bloats payload; violates the "less is more" principle. |

- The `metadata.*` storage shape (still snake_case: `metadata.observation_ids`, `metadata.files_touched`, etc. — that's the on-disk format inherited from SQLite + migration scripts and is correct as-is).
- `/api/v1/*` canonical endpoints (those are Zod-typed and live in km-core; out of scope here).
