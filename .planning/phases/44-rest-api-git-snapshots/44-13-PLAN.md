## New km-core APIs

```ts
// lib/km-core/src/store/GraphKMStore.ts

async findByContentHash(
  agent: string,
  contentHash: string,
  opts?: { ontologyClass?: string }
): Promise<Entity[]>;

async findRecentByAgent(
  agent: string,
  sinceISO: string,
  limit: number,
  opts?: { ontologyClass?: string }
): Promise<Entity[]>;
```

Both default `ontologyClass='Observation'` so writer call sites stay
terse. Both iterate the ontology-class node list (Graphology
node-attribute filter) and apply the per-call predicate (agent +
contentHash, or agent + createdAt > sinceISO). The semantic-dup helper
also caps to `limit` and sorts by `createdAt` DESC.

The cost model: per call O(N_observations_for_class) — at current
production scale (~3.9k observations) this is sub-millisecond.
Acceptance includes a perf assertion at 1k pre-seeded observations.
