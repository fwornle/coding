<objective>
Update `ObservationConsolidator._pushInsightToKG` (OC.js:654-665) to consume the new `{legacyId, mintedId}` return shape that Plan 59-01 added to `ObservationWriter.writeInsight`. The pre-edit code at OC.js:659-661 calls `await writer.writeInsight(...)` then re-derives the mintedId via `this._kmStore.findByLegacyId({system:'A', id: entry.topic})` — that lookup races against km-core's in-memory hydration and returns null at a ~1-in-100 rate (observed 2026-06-15: `Live Backfill Pre-flight Procedure and Wave-Analysis Routing` orphan). When the lookup returns null, the entire `has_insight` follower block at OC.js:679-705 silently short-circuits via `if (projectName && mintedId)`, producing an orphan Insight.

After this plan ships, the mintedId is read from the writer return directly — the race window is closed at its root. The has_insight block at OC.js:679-705 stays as-is (D-03.2: its role shifts to idempotent re-write protection).

Output: surgical 8-line rewrite of the OC.js:654-665 block to consume the new return shape; the has_insight block downstream is untouched.
