<objective>
Close the R2/R3/R4 Run-record persistence gaps diagnosed in RESEARCH.md so a single experiment's cells are distinguishable and every terminal outcome is recorded. Today `variant` is threaded into `span.meta` but dropped at Run-write, `repeat` is never threaded, and there is no `terminal_state` or `skip_reason` field — so `task_hash` (constant across the matrix) cannot identify a cell and timeouts/aborts/probe-skips have nowhere to land.

This plan is the foundational schema + measured-span-seam edit the runner engine (78-03) depends on. It resolves RESEARCH Open Question Q2 by threading the four fields through the SHIPPED seam (`measurement-start` → `span.meta` → `measurement-stop` tags → `writeRun`) — NOT via a runner-side raw metadata patch.

Purpose: make variant/repeat/terminal_state/skip_reason first-class, null-preserved Run.metadata tags.
Output: extended `writeRun` allow-list + two new `measurement-stop` flags + `--repeat` threading, all test-covered.
</objective>
