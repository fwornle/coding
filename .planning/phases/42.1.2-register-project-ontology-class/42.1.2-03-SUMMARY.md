## Decisions Made

- **Test file co-located with class under test (NOT appended to `OntologyConfigManager.layout.test.ts`).** CONTEXT.md `<decisions>` § "Claude's Discretion" left the choice open between (a) appending Test F to the existing layout test file in `src/ontology/` or (b) creating a new file under `src/agents/` alongside the wave-controller. Option (b) won because: the subject under test is `WaveController.ensureProjectAnchor` (an agent-layer concern), the test stubs a `kmCoreAdapter` (storage-layer surface), and `OntologyConfigManager.layout.test.ts` is now exclusively the registry-resolution-layer test. Mixing wave-controller integration tests into the layout test would muddy the file's purpose; Plan 02's SUMMARY explicitly called out that the layout test should stay focused on registry layer concerns.
- **`as unknown as { ... }` compile-time visibility hatch over a thin production-side internal accessor.** CONTEXT.md `<decisions>` § "Claude's Discretion" explicitly DISCOURAGES exposing a `__forTestEnsureProjectAnchor()` method on `WaveController` because that would be a production-side change just for testability — bigger blast radius than the test deserves. The cast is purely compile-time (TypeScript's visibility checker is structural; the field access is legal JS at runtime since `private` is not enforced by V8). Acceptable because the test sandbox's `kmCoreAdapter` field assignment is the EXACT same field assignment the production code would do at the bootstrap site when `KM_CORE_PERSISTENCE=km-core`.
- **Structural-typed local interfaces (`QueryFilter`, `StoredEntity`, `StoreEntityCall`, `QueryEntitiesCall`) instead of importing the real `KmCoreAdapter` and `Entity` types from `../storage/km-core-adapter.js`.** Real types pull `@fwornle/km-core` into the test surface, which (a) couples the test to km-core dependency evolution and (b) risks the test breaking when km-core changes its `Entity` shape for reasons unrelated to ensureProjectAnchor's contract. Local minimal interfaces are exactly what `ensureProjectAnchor` reads — name + entityType + the spy's own bookkeeping fields. The cast is structurally compatible with the real `KmCoreAdapter` at the two-method surface ensureProjectAnchor actually exercises.
- **`repositoryPath` set to a per-test `os.tmpdir()` subpath.** `WaveController`'s constructor instantiates `WorkflowReportAgent(repositoryPath)`, whose constructor synchronously calls `mkdirSync('${repositoryPath}/.data/workflow-reports', { recursive: true })` (workflow-report-agent.ts:62). If `repositoryPath` were `process.cwd()` or the real repo root, every test run would litter `.data/workflow-reports/` with test artifacts. Routing through `os.tmpdir() + per-test-suffix` puts those artifacts under `/tmp/wave-controller-ensure-project-anchor-test-<pid>-<timestamp>-<rand>/.data/workflow-reports/` — self-cleaning under OS-tmp eviction policy.
- **Mutable closure-captured `codingMinted` flag for the idempotency test.** The cleanest alternative would have been to re-architect `buildHarness` so the fake `storeEntity` callback could call back into a `setCodingMinted(true)` mutator. That would have required either (a) a circular reference between the harness and the queryReturn getter or (b) a `RefCell`-style wrapper object — both add complexity for a one-test concern. Capturing `codingMinted` as a `let` in the test's own scope and flipping it from the test body (after observing `harness.storeEntityCalls.length > originalLength`) keeps the harness simple and the idempotency contract explicit.
- **Two-commit topology: submodule commit `7f71c8f` + outer-repo pointer-bump `1577f1367`.** The execution prompt explicitly specified this topology after verifying via `ls -la integrations/mcp-server-semantic-analysis/src/agents` that the directory is real (`drwxr-xr-x`), not a symlink. Plan 02's single-commit topology applied only because `src/ontology/` is a symlink; `src/agents/` is the opposite case. The submodule was already on `main` (verified by orchestrator probe: "Your branch is ahead of 'origin/main' by 26 commits"), so no branch switch was needed.
