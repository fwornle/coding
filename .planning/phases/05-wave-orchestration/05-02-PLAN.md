**Key implementation details:**

1. **execute() method** -- the main entry point:
   - Initialize GraphDatabaseAdapter
   - Load component manifest via `loadComponentManifest()`
   - Load existing KG entities via `graphDB.getAllEntities()` (map GraphEntity to KGEntity)
   - Execute Wave 1 -> persist -> Execute Wave 2 -> persist -> Execute Wave 3 -> persist
   - If any wave fails and `failFast` is true, abort remaining waves but return partial results
   - Build and return WaveExecutionResult summary
   - Log a structured summary report at the end

2. **executeWave1()** -- single agent, produces L0+L1:
   - Instantiate `Wave1ProjectAgent` (static import -- same plan creates it)
   - Call `wave1Agent.execute(wave1Input)`
   - Wrap result into WaveResult

3. **executeWave2()** -- parallel agents, one per L1 component:
   - **Dynamic import**: `const { Wave2ComponentAgent } = await import('./wave2-component-agent.js');`
   - For each L1 entity from wave1Result, prepare Wave2Input
   - Get component files via code-graph-rag (try/catch -- if CGR unavailable, fall back to empty file list and let the agent use directory scanning)
   - Use `runWithConcurrency()` to limit parallel agents to `maxAgentsPerWave`
   - Collect all WaveAgentOutputs into WaveResult

4. **executeWave3()** -- parallel agents, one per L2 node:
   - **Dynamic import**: `const { Wave3DetailAgent } = await import('./wave3-detail-agent.js');`
   - Collect all L2 entities from wave2Result (both manifest-defined and discovered)
   - For each L2 entity, prepare Wave3Input
   - Use `runWithConcurrency()` for parallel execution
   - Collect results into WaveResult

5. **persistWaveResult()** -- incremental merge per CONTEXT.md decision:
   - Convert KGEntity[] to SharedMemoryEntity[] using `mapEntityToSharedMemory()`
   - Use PersistenceAgent to persist (it handles conflict resolution)
   - Also persist KGRelation[] as relationship edges via GraphDatabaseAdapter
   - Log entity count per wave

6. **mapEntityToSharedMemory()** -- CRITICAL mapping function (see RESEARCH.md Pitfall 1 and Pitfall 6):
   - `entityType: entity.type` (NOT entity.entityType -- KGEntity has `type`)
   - `hierarchyLevel: entity.level`
   - `parentEntityName: entity.parentId`
   - `isScaffoldNode: (entity.level ?? 3) < 3` (L0, L1, L2 are scaffold)
   - Attach ontology metadata to prevent redundant LLM re-classification:
     ```typescript
     metadata: {
       created_at: new Date().toISOString(),
       last_updated: new Date().toISOString(),
       team: this.team,
       source: 'wave-analysis',
       ontology: {
         ontologyClass: this.getOntologyClass(entity.level),
         ontologyVersion: '1.0',
         classificationConfidence: 1.0,
         classificationMethod: 'auto-assigned',
         ontologySource: 'lower' as const,
         classifiedAt: new Date().toISOString()
       }
     }
     ```
   - `getOntologyClass(level)`: 0 -> 'Project', 1 -> 'Component', 2 -> 'SubComponent', 3 -> 'Detail'

7. **runWithConcurrency()** -- bounded parallel execution (see RESEARCH.md Pattern 1):
   - Takes array of thunks `(() => Promise<T>)[]` and max concurrent count
   - Implements work-stealing pattern: starts `maxConcurrent` workers, each pulls next task when done
   - Returns results in order
   - If `failFast` and any task throws, cancel remaining (reject Promise.all)

8. **logWaveBanner()** -- per CONTEXT.md progress visibility decision:
   ```typescript
   const line = '='.repeat(60);
   log(line, 'info');
   log(`=== ${wave}: ${description} ===`, 'info');
   log(line, 'info');
   ```

9. **updateProgress()** -- writes wave progress to progressFile:
   - Read existing progress JSON, preserve debug state fields (singleStepMode, mockLLM, llmState)
   - Merge wave-specific data: currentWave, totalWaves, message
   - Write back to progressFile

10. **buildSummaryReport()** -- structured completion summary:
    - Calculate entitiesByLevel from all wave results
    - Count manifest vs discovered entities
    - Log structured summary: entities per level, total time, any failures
    - Return WaveExecutionResult

**execute() method flow:**

1. **Scan directory structure**: Walk the repo at depth 2-3 to get overall project shape. Produce a compact directory listing string.

2. **Read existing KG entities**: Use `input.existingEntities` as additional context for richer summaries. Format as a brief text block listing entity names, types, and first observation.

3. **For each L1 component** from `input.manifest.components`:
   a. Find representative files: Use component keywords/aliases to locate relevant directories. Read 3-5 source files (prioritize main entry points, config files, key source).
   b. Build LLM prompt asking for:
      - A comprehensive summary of this component (2-3 paragraphs)
      - 3-7 specific observations about architecture, patterns, key behaviors
      - Suggested L2 children (what sub-components exist within this component)
   c. Call LLM via `this.llmService.chat()` with tier 'mid' (not the expensive tier -- Wave 1 produces summaries, not deep analysis)
   d. Parse LLM response into entity + observations + child suggestions

4. **Build L0 Project entity**:
   - name: manifest.project.name ("Coding")
   - type: "Project"
   - level: 0
   - observations: Project-level summary from directory scan + component overview
   - hierarchyPath: "Coding"
   - parentId: undefined (root node)

5. **Build L1 Component entities** for each component:
   - name: component.name (e.g., "SemanticAnalysis")
   - type: "Component"
   - level: 1
   - parentId: manifest.project.name
   - hierarchyPath: `${manifest.project.name}/${component.name}`
   - observations: Component-specific observations from LLM analysis
   - significance: 8 (structural components are high significance)

6. **Build relationships**: parent-child edges from L0 to each L1.

7. **Build childManifest**: For each L1 component, include both:
   - Manifest-defined L2 children (discovered: false)
   - LLM-suggested additional L2 children (discovered: true)

8. **Return WaveAgentOutput** with all entities, relationships, childManifest.
