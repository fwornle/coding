## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dashboard → vkb-server API | Client-supplied runId, spec name, and override fields cross into path builds + a host spawn delegation |
| vkb-server → coordinator | The API forwards a launch/cancel request to the host executor |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-85-04-01 | Tampering / Info disclosure | run-status/:runId path build | mitigate | `_validRunId` charset+`..` guard BEFORE path.join; read confined to .data/experiments/runs/ |
| T-85-04-02 | Tampering (command injection via spec) | handleExperimentRun spec argument | mitigate | Spec must be a member of the server-listed config/experiments/*.yaml set; never a raw client path forwarded to spawn |
| T-85-04-03 | DoS (unbounded concurrent runs) | launch endpoint | mitigate | D-02 strict single-slot 409 (interactive-span OR live-run) — only one run/span ever live |
| T-85-04-04 | DoS (LevelDB lock contention during a live run) | live-run 409 check / run-status | mitigate | Do the live-run check + status via progress.json + pid liveness ONLY — never open the experiment store (Pitfall 6) |
| T-85-04-05 | EoP (invalid overrides drive a bad run) | override fields | mitigate | Validate repeats/timeout positive int + variant ∈ resolved names + variantOverrides keys ∈ resolved names (experiment-run.mjs:107-125 parity) before delegating |
| T-85-04-SC | Tampering | npm/pip/cargo installs | mitigate | No installs — N/A |
</threat_model>
