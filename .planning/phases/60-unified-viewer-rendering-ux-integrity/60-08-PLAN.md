needed (per CLAUDE.md `Rebuilding After Code Changes`).

60-01/56.1 lineage uses a `.selection-ring` circle (`D3GraphCanvas.tsx:836`) for the selection
visual. Options:
- (a) Keep ring as circle regardless of node shape — simpler, predictable selection UX, but visually

      work.

Recommend (a) — keep selection ring as circle, document the deliberate stylistic choice in the
SUMMARY. Selection ring is a hit-target affordance, not a class identity affordance.

- (a) Inline text after the row title: `· hidden by "Show debug entity types"` (operator-readable)
- (b) Icon + tooltip on the row (compact but less discoverable)

Recommend (a) — text is operator-friendly and matches the existing pattern in the Domain filter's
"Domain filter not applicable for this system" inline message. Confirm Tailwind class set during
execution to match existing micro-type tokens (CLAUDE.md UI-SPEC §3: `text-[10px]` / `text-[9px]`).

       ```ts
       import type { Selection } from 'd3-selection'
       import { SHAPE_PALETTE, shapeFallback, type ShapeKind } from './color-fallback'

       // Path-data generators. Sized so a node of radius `r` fits inside an
       // inscribed circle of the same radius (matches the existing layout
       // collision math — confirm during research, may need empirical tweak).
       const HEXAGON_PATH = (r: number): string => {
         // Pointy-top hexagon: 6 vertices at 60° increments starting from top
         const pts: string[] = []
         for (let i = 0; i < 6; i++) {
           const angle = (Math.PI / 3) * i - Math.PI / 2
           pts.push(`${r * Math.cos(angle)},${r * Math.sin(angle)}`)
         }
         return `M${pts.join('L')}Z`
       }

       const DIAMOND_PATH = (r: number): string => {
         return `M0,${-r} L${r},0 L0,${r} L${-r},0 Z`
       }

       export type ShapeAppender = (
         sel: Selection<SVGGElement, unknown, null, undefined>,
         r: number,
       ) => Selection<SVGElement, unknown, null, undefined>

       export const SHAPE_RENDERERS: Record<ShapeKind, ShapeAppender> = {
         circle: (sel, r) =>
           sel.append('circle').attr('r', r),
         square: (sel, r) =>
           sel.append('rect').attr('x', -r).attr('y', -r).attr('width', r * 2).attr('height', r * 2),
         hexagon: (sel, r) =>
           sel.append('path').attr('d', HEXAGON_PATH(r)),
         diamond: (sel, r) =>
           sel.append('path').attr('d', DIAMOND_PATH(r)),
       }

       export function renderNodeShape(
         entity: { ontologyClass?: string },
         sel: Selection<SVGGElement, unknown, null, undefined>,
         r: number,
       ): Selection<SVGElement, unknown, null, undefined> {
         const cls = entity.ontologyClass
         const kind: ShapeKind = (cls && SHAPE_PALETTE[cls]) ?? shapeFallback
         const renderer = SHAPE_RENDERERS[kind] ?? SHAPE_RENDERERS.circle
         return renderer(sel, r).attr('class', 'node-shape')
       }
       ```
