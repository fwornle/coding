**process.stderr.write rationale (per checker I-1):** The repair script writes status / progress
messages to `process.stderr` instead of `console.log` because stderr is the legitimate channel for
structured CLI logs in this codebase — `scripts/backfill-project-tag.mjs` (the template this script
mirrors) established the precedent. CLAUDE.md's `Constraint dodging is forbidden` clause targets
cases where an executor swaps to a different raw-write API to defeat a constraint regex that flagged
real concern (e.g., switching `console.log` to `process.stderr.write` purely to dodge `no-console-log`).
Here the channel choice is deliberate: stdout is reserved for machine-readable output (none in this
script), stderr carries operator-facing status text. The pattern matches the template script and
host-side CLI conventions; no constraint is being dodged.
