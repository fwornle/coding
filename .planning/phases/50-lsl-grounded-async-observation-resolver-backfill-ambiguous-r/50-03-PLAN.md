    ```
    #!/usr/bin/env bash
    set -euo pipefail

    REPO_ROOT="/Users/Q284340/Agentic/coding"
    STATE_FILE="${REPO_ROOT}/.data/lsl-resolver-state.json"
    LIMIT="${LSL_RESOLVER_LIMIT:-100}"
    PROXY_URL="${LLM_CLI_PROXY_URL:-http://localhost:12435}"
    NOW_ISO="$(date -u +%Y-%m-%dT%H:%M:%SZ)"

    log() { printf '[lsl-resolver][%s] %s\n' "$(date -u +%H:%M:%SZ)" "$*" >&2; }

    cd "${REPO_ROOT}"

    # Probe LLM proxy. If unreachable, skip this run (do NOT fail).
    PROXY_HTTP_CODE="$(curl -s --max-time 5 -o /dev/null -w '%{http_code}' "${PROXY_URL}/api/complete" -X POST -H 'Content-Type: application/json' -d '{}' || echo "000")"
    # The proxy returns 4xx for empty body — that's a SUCCESSFUL reachability probe.
    # 000 (curl error) or 5xx means unreachable / broken.
    case "${PROXY_HTTP_CODE}" in
      2*|4*) log "LLM proxy reachable (HTTP ${PROXY_HTTP_CODE}), proceeding" ;;
      *) log "LLM proxy unreachable (HTTP ${PROXY_HTTP_CODE}) — skipping this run"; exit 0 ;;
    esac

    # Compute --since from state file, or default to 7 days ago.
    if [[ -f "${STATE_FILE}" ]]; then
      SINCE="$(node -e "try { console.log(require('${STATE_FILE}').last_run_at || '') } catch { console.log('') }")"
    fi
    if [[ -z "${SINCE:-}" ]]; then
      SINCE="$(date -u -v-7d +%Y-%m-%dT%H:%M:%SZ 2>/dev/null || date -u -d '7 days ago' +%Y-%m-%dT%H:%M:%SZ)"
      log "no prior state — defaulting --since to ${SINCE}"
    else
      log "resuming from ${SINCE}"
    fi

    # Invoke the Plan 1 CLI.
    log "running: node scripts/resolve-observations-from-lsl.mjs --since ${SINCE} --limit ${LIMIT}"
    set +e
    node scripts/resolve-observations-from-lsl.mjs --since "${SINCE}" --limit "${LIMIT}"
    EXIT_CODE=$?
    set -e

    if [[ ${EXIT_CODE} -ne 0 ]]; then
      log "resolver returned exit ${EXIT_CODE} — state file NOT updated, next run will retry from ${SINCE}"
      exit ${EXIT_CODE}
    fi

    # Atomic state write.
    mkdir -p "$(dirname "${STATE_FILE}")"
    TMP_FILE="${STATE_FILE}.tmp"
    printf '{"last_run_at":"%s","last_run_limit":%s}\n' "${NOW_ISO}" "${LIMIT}" > "${TMP_FILE}"
    mv "${TMP_FILE}" "${STATE_FILE}"
    log "run complete — state updated to ${NOW_ISO}"
    ```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-50-03-CR | DoS | launchd KeepAlive tight loop | mitigate | Plist intentionally omits `KeepAlive` and sets `StartInterval=1800`. When the resolver returns non-zero, launchd waits 30 min before the next attempt — no tight loop. `ThrottleInterval=30` is a floor under that. Wrapper script exits 0 (not 1) on proxy-unreachable so even a broken proxy won't surface as repeated launchd errors. |
| T-50-03-SI | DoS | State file corruption | mitigate | Atomic `.tmp` + `mv` write pattern. If a write is interrupted, the previous state file remains intact. Wrapper script tolerates a missing state file by defaulting `--since` to 7 days ago. |
| T-50-03-CR2 | Tampering | Wrapper script arguments | mitigate | All flags (`--since`, `--limit`) are computed from local state — no user-supplied input. Plan 1's CLI validates these per its own contract. |
| T-50-03-PR | Information disclosure | Log file at `.logs/lsl-resolver.log` | accept | Logs project + timestamps + exit codes only. No user message content. Same surface as existing `.logs/health-coordinator.log`. |
| T-50-03-RL | DoS | LLM cost amplification | mitigate | Wrapper hard-caps `--limit 100` per invocation. Plan 1's CLI further caps via its own default. `taskType: 'observation-resolution'` routes to haiku tier per CLAUDE.md. Idempotency prevents re-processing already-resolved rows. |
| T-50-03-SC | Tampering | npm/launchctl installs | mitigate | No new npm packages. `launchctl bootstrap` only loads from the user's `~/Library/LaunchAgents/` — no privileged operations. Installer requires no `sudo`. |
| T-50-03-DC | Tampering | health-coordinator SQL read | accept | Read-only `COUNT(*)` queries against the existing coordinator DB handle. No new connection, no writes. Failure mode (try/catch) returns null counters instead of crashing the coordinator. |
