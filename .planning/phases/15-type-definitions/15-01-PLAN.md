  <name>Task 2: Create WorkflowState discriminated union and migration schemas</name>
  <files>shared/workflow-types/state.ts, shared/workflow-types/schemas.ts, shared/workflow-types/index.ts</files>
  <action>
1. Create `shared/workflow-types/state.ts` with Zod-first discriminated union:

**PauseReasonSchema**: `z.enum(['user-requested', 'single-step-boundary'])`

**IdleStateSchema**: `{ status: z.literal('idle') }` -- no other fields, idle carries no data

**RunningStateSchema**: Single schema with `status: z.literal('running')` and `subStatus: z.enum(['executing-step', 'between-steps'])`. Include:
- `workflowId: z.string()`
- `workflowName: z.string()`
- `config: RunConfigSchema` (imported from config.ts)
- `progress: RunProgressSchema` (imported from config.ts)

Use a single 'running' schema with `subStatus` enum inside rather than nested discriminatedUnion -- this avoids the Zod pitfall of non-unique discriminator values.

**PausedStateSchema**: `{ status: z.literal('paused'), workflowId, workflowName, config, progress, pausedAt: z.object({ step: z.string(), substep: z.string().optional() }), pauseReason: PauseReasonSchema, resumable: z.boolean() }`

**CompletedStateSchema**: `{ status: z.literal('completed'), workflowId, workflowName, config, completedSteps: z.number(), duration: z.number(), summary: z.record(z.unknown()).optional() }`

**FailedStateSchema**: `{ status: z.literal('failed'), workflowId, workflowName, config, failedStep: z.string(), error: z.string(), stackTrace: z.string().optional(), progress: RunProgressSchema }`

**CancelledStateSchema**: `{ status: z.literal('cancelled'), workflowId, workflowName, config, cancelledAt: z.string(), lastStep: z.string() }`

**WorkflowStateSchema**: `z.discriminatedUnion('status', [IdleStateSchema, RunningStateSchema, PausedStateSchema, CompletedStateSchema, FailedStateSchema, CancelledStateSchema])`

Export all individual state schemas, the union schema, and all inferred types (WorkflowState, IdleState, RunningState, PausedState, CompletedState, FailedState, CancelledState). Also export PauseReason type.

2. Create `shared/workflow-types/schemas.ts` with:

**StepStatusSchema**: `z.enum(['pending', 'running', 'completed', 'failed', 'skipped'])` -- migrated from workflow-events.ts StepStatus
**StepDefinitionSchema**: `z.object({ name: z.string(), description: z.string(), wave: z.number().optional(), substeps: z.array(z.object({ id: z.string(), label: z.string() })).optional() })` -- migrated from workflow-events.ts

**WorkflowStateWithMigrationSchema**: Wraps WorkflowStateSchema with `z.preprocess()` that handles:
- Old `status: 'starting'` -> maps to `{ status: 'running', subStatus: 'executing-step', ... }`
- Old flat progress format (where singleStepMode/mockLLM are top-level keys mixed with status) -> restructures into `{ status, config: { singleStepMode, mockLLM, ... }, progress: { ... } }`
- Already-new-format data passes through unchanged

Export StepStatus, StepDefinition types via z.infer. Export WorkflowStateWithMigrationSchema.

3. Create `shared/workflow-types/index.ts`:
- Re-export everything from config.ts, state.ts, schemas.ts
- This is the single import point for consumers
