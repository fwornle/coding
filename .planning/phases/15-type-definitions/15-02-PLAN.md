1. Create `shared/workflow-types/transitions.ts`:

**InvalidTransitionError class**: extends Error, carries `fromStatus: string` and `eventType: string`. Message format: `Invalid transition: cannot apply '${eventType}' in '${fromStatus}' state`

**WorkflowTransitionEventSchema**: Zod discriminated union on 'type' field with these event variants:
- `{ type: 'start', config: RunConfigSchema, workflowName: z.string(), firstStep: z.string() }` -- idle -> running
- `{ type: 'step-complete', stepName: z.string(), duration: z.number() }` -- running -> running (next step)
- `{ type: 'pause', reason: PauseReasonSchema }` -- running -> paused
- `{ type: 'resume' }` -- paused -> running
- `{ type: 'fail', error: z.string(), step: z.string(), stackTrace: z.string().optional() }` -- running -> failed
- `{ type: 'cancel', reason: z.string().optional() }` -- running|paused -> cancelled
- `{ type: 'complete', summary: z.record(z.unknown()).optional() }` -- running -> completed
- `{ type: 'reset' }` -- failed -> idle

Export `type WorkflowTransitionEvent = z.infer<typeof WorkflowTransitionEventSchema>`

**TransitionMap type**: Compile-time enforcement mapping each state to its valid events:
```typescript
type TransitionMap = {
  idle: Extract<WorkflowTransitionEvent, { type: 'start' }>;
  running: Extract<WorkflowTransitionEvent, { type: 'pause' | 'fail' | 'cancel' | 'complete' | 'step-complete' }>;
  paused: Extract<WorkflowTransitionEvent, { type: 'resume' | 'cancel' }>;
  failed: Extract<WorkflowTransitionEvent, { type: 'reset' }>;
  completed: never;
  cancelled: never;
};
```

**transition() function**: `(state: WorkflowState, event: WorkflowTransitionEvent) => WorkflowState`
- Uses exhaustive switch on `state.status`
- Each case validates event.type and throws InvalidTransitionError for invalid events
- `idle + start`: Creates running state with `Object.freeze(event.config)` for runtime immutability, initial RunProgress
- `running + step-complete`: Updates progress (increment currentStepIndex, add to completedSteps, update lastUpdate/elapsedSeconds)
- `running + pause`: Creates paused state with pausedAt from current progress position
- `running + fail`: Creates failed state with error and current progress
- `running + cancel`: Creates cancelled state with cancelledAt timestamp and lastStep
- `running + complete`: Creates completed state with completedSteps count and duration
- `paused + resume`: Returns to running state, preserving config and progress
- `paused + cancel`: Creates cancelled state
- `failed + reset`: Returns idle state
- `completed/cancelled + any`: Throws InvalidTransitionError (terminal states)

1. Create `shared/workflow-types/derived.ts`:

**deriveStepStatuses function**: `(state: WorkflowState, stepDefinitions: readonly StepDefinition[]) => Map<string, StepStatus>`

Logic by state.status:
- `idle`: All steps 'pending'
- `running`: Steps in completedSteps -> 'completed', currentStepName -> 'running', rest -> 'pending'
- `paused`: Steps in completedSteps -> 'completed', pausedAt.step -> 'paused' (use 'running' status since StepStatus has no 'paused' -- the parent state carries pause semantics), rest -> 'pending'
- `completed`: All steps 'completed'
- `failed`: Steps in completedSteps -> 'completed', failedStep -> 'failed', rest -> 'pending'
- `cancelled`: Steps in progress.completedSteps -> 'completed', rest -> 'pending'

**deriveSubstepStatuses function**: `(state: WorkflowState, stepName: string, substeps: readonly { id: string; label: string }[]) => Map<string, StepStatus>`

Logic: If step is not the current step, all substeps inherit step status. If step is current and state is running with substep tracking (currentSubstepIndex defined), substeps before index -> 'completed', at index -> 'running', after -> 'pending'.
