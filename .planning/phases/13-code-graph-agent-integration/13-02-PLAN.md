**1. Modify semantic-analysis-agent.ts `analyzeEntityCode()` (line ~1920):**
- The method already accepts `AnalyzeEntityCodeInput` which now has `cgrContext?: string` from Plan 01
- In the prompt construction, if `input.cgrContext` is provided (non-empty string), insert it as a new section BEFORE the existing file content section. The CGR context is already formatted as `<code_graph>...</code_graph>` XML by CgrObservationBuilder
- Add to the system prompt or user prompt instructions: "If <code_graph> data is provided, reference it in your observations. Prefix observations grounded in code graph data with [LLM+CGR]. Prefix observations from your own analysis with [LLM]."
- Do NOT change the method signature — `cgrContext` is already optional on the input type

**2. Add observation post-processing utility** (add to semantic-analysis-agent.ts as a private helper or to cgr-observation-builder.ts):
- `autoTagObservations(observations: string[], hadCgrContext: boolean): string[]`
- Per locked decision: don't rely solely on LLM self-tagging
- If `hadCgrContext` is true: scan each observation for code entity references (PascalCase identifiers, file paths like `*.ts`). If references found and no tag present, prepend `[LLM+CGR]`. If no references and no tag, prepend `[LLM]`.
- If `hadCgrContext` is false: prepend `[LLM]` to any untagged observations
- Skip observations already starting with `[CGR]`, `[LLM]`, or `[LLM+CGR]`

**3. Modify wave1-project-agent.ts:**
- Add constructor params or method params to accept `CgrQueryCache | null` and `CgrObservationBuilder | null`
- In the per-L1-entity analysis loop (where entities are built):
  - If cgrCache is available: `await cgrCache.ensureReady()` (first call only, subsequent calls are no-op since promise is resolved)
  - Query: `const cgrEntities = await cgrCache.queryComponentEntities(componentName, componentKeywords)`
  - Build observations: `const cgrObs = cgrBuilder.buildStructuralObservations(cgrEntities, componentName)`
  - Push `cgrObs` into the entity's observations array (these are `[CGR]`-tagged)
  - L0/L1 CGR is OPTIONAL per locked decision — no `_noCgrEvidence` flag needed here
  - Pass `cgrContext: cgrBuilder.formatForLLMPrompt(...)` in `AnalyzeEntityCodeInput` when calling SAA
- Wrap all CGR calls in try/catch — on failure, log warning and continue without CGR data

**4. Modify wave2-component-agent.ts:**
- Accept `CgrQueryCache | null` and `CgrObservationBuilder | null` (same pattern as Wave 1)
- In the per-L2-entity pipeline (before SAA call):
  - Query: `const details = await cgrCache.queryEntityDetails(entityName, componentFiles)`
  - Build structural obs: `cgrBuilder.buildStructuralObservations(details.entities, entityName)`
  - Build relationship obs: `cgrBuilder.buildRelationshipObservations(details.callees, details.imports)`
  - Push all `[CGR]` observations into entity.observations
  - Format for LLM: `const cgrPrompt = cgrBuilder.formatForLLMPrompt(details)`
  - Pass `cgrContext: cgrPrompt` to `analyzeEntityCode()` input
  - Post-process SAA observations with `autoTagObservations(result.observations, !!cgrPrompt)`
  - If `!cgrBuilder.hasEvidence(details.entities)` and entity level >= 2: set `entity._noCgrEvidence = true`
- Wrap in try/catch, continue on failure

**5. Modify wave3-detail-agent.ts:**
- Same CGR injection pattern as Wave 2, PLUS:
  - Additional query: `const callGraph = await cgrCache.queryCallGraph(entityName, 2)` (depth 2-3 per locked decision, use 2 as default)
  - Build call graph observations: `cgrBuilder.buildRelationshipObservations(callGraph.chains.map(c => c.callee), [])` and also a dedicated call chain observation like `[CGR] Call chain: X -> Y -> Z`
  - Pass both `details` and `callGraph` to `cgrBuilder.formatForLLMPrompt(details, callGraph)`
  - L3 entities REQUIRE 1+ [CGR] or [LLM+CGR] per locked decision — set `_noCgrEvidence` if no CGR evidence

**6. For all wave agents:**
- CGR observations are pushed to entity.observations BEFORE the SAA call — they count toward the 3+ minimum
- Post-process ALL SAA-returned observations through `autoTagObservations()`
- If `cgrCache?.isAvailable() === false` at the start, skip ALL CGR work and proceed with existing LLM-only pipeline (per locked decision: if CGR entirely unavailable, no flags)
