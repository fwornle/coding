**Function A — pollProxySemantic** (D-01, D-02; Pattern A fail-open):

```javascript
/**
 * Phase 34 R1 (D-01, D-02): tier-pinned semantic-work probe.
 * POSTs a single-token-elicit prompt every 60s; classifies the response per D-02:
 *   - HTTP 4xx/5xx                         -> reason='http_<code>',     semantic_ok=false
 *   - Network timeout (>10s)               -> reason='timeout',         semantic_ok=false
 *   - HTTP 200 missing/empty content       -> reason='empty_content',   semantic_ok=false
 *   - HTTP 200 content lacks 'OK' substring -> reason='oksub_missing',  semantic_ok=false
 *   - HTTP 200 + content contains 'OK'     -> reason=null,              semantic_ok=true
 * NEVER silently 'healthy' on error (Pattern A); always sets last_probe_end.
 */
async function pollProxySemantic() {
  const probeEndedAt = () => new Date().toISOString();
  const start = Date.now();
  const probeBody = {
    messages: [{ role: 'user', content: 'reply with the single token: OK' }],
    provider: 'copilot',
    tier: 'haiku',
    maxTokens: 5
  };
  const prevSemantic = currentState.proxy.semantic_ok;
  let r;
  try {
    r = await fetch(`${PROXY_URL}/api/complete`, {
      method: 'POST',
      headers: { 'content-type': 'application/json' },
      body: JSON.stringify(probeBody),
      signal: AbortSignal.timeout(PROXY_PROBE_TIMEOUT_MS)
    });
  } catch (err) {
    const elapsed = Date.now() - start;
    const reason = (err && (err.name === 'TimeoutError' || /aborted|timeout/i.test(err.message)))
      ? 'timeout' : `network_${err.message || 'error'}`;
    currentState.proxy.semantic_ok = false;
    currentState.proxy.last_round_trip_ms = elapsed;
    currentState.proxy.reason = reason;
    currentState.proxy.last_probe_end = probeEndedAt();
    if (prevSemantic !== false) log(`proxy semantic_ok flip -> false (${reason})`, 'INFO');
    return;
  }
  const elapsed = Date.now() - start;
  currentState.proxy.last_round_trip_ms = elapsed;
  currentState.proxy.last_probe_end = probeEndedAt();
  if (!r.ok) {
    currentState.proxy.semantic_ok = false;
    currentState.proxy.reason = `http_${r.status}`;
    if (prevSemantic !== false) log(`proxy semantic_ok flip -> false (http_${r.status})`, 'INFO');
    return;
  }
  let body;
  try { body = await r.json(); }
  catch (err) {
    currentState.proxy.semantic_ok = false;
    currentState.proxy.reason = 'empty_content';
    if (prevSemantic !== false) log(`proxy semantic_ok flip -> false (empty_content/json-parse: ${err.message})`, 'INFO');
    return;
  }
  // D-02: extract content. Shape per copilot proxy: { choices: [ { message: { content: '...' } } ] } OR { content: '...' } OR { text: '...' }
  const content = (body?.choices?.[0]?.message?.content)
    ?? body?.content ?? body?.text ?? '';
  if (!content || typeof content !== 'string') {
    currentState.proxy.semantic_ok = false;
    currentState.proxy.reason = 'empty_content';
    if (prevSemantic !== false) log(`proxy semantic_ok flip -> false (empty_content)`, 'INFO');
    return;
  }
  if (!content.includes('OK')) {
    currentState.proxy.semantic_ok = false;
    currentState.proxy.reason = 'oksub_missing';
    if (prevSemantic !== false) log(`proxy semantic_ok flip -> false (oksub_missing — got: ${String(content).slice(0, 80)})`, 'INFO');
    return;
  }
  // Success.
  currentState.proxy.semantic_ok = true;
  currentState.proxy.reason = null;
  if (prevSemantic !== true) log(`proxy semantic_ok flip -> true (${elapsed}ms)`, 'INFO');
  log(`proxy semantic probe ok (${elapsed}ms)`, 'DEBUG');
}
```

/**

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| coordinator -> proxy (POST /api/complete) | Localhost-only; coordinator and proxy both run as the same UID under launchd. The probe is unauthenticated — same posture as any localhost developer tool. |
| coordinator -> proxy (GET /health) | Localhost-only; read-only; returns networkMode + provider list. |
| proxy -> coordinator | Out of scope — coordinator does not accept callbacks. |
| /health/state -> any HTTP client on the host | Coordinator binds 0.0.0.0:3034 (Phase 33 D-X); state.proxy.reason can leak failure-mode strings. No secrets in those strings (no tokens, no provider responses past 80 chars logged truncated). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-34-02-01 | Spoofing | proxy POST /api/complete | accept | Localhost-only; no auth. Same trust model as the proxy itself. If a malicious local process MitMs port 12435, the whole CLI is already compromised. |
| T-34-02-02 | Tampering | probe response body | mitigate | Response is parsed as JSON in try/catch; on parse failure semantic_ok=false (fail-closed). Content-substring check ('OK') prevents prompt-injection of false 'healthy' state from the model itself — would require a model that returns the literal token 'OK', which is the intended behavior. |
| T-34-02-03 | Repudiation | log lines | accept | Every state transition logged at INFO; per-probe at DEBUG. Existing 10MB log rotation in coordinator preserves history. |
| T-34-02-04 | Information Disclosure | state.proxy.reason | mitigate | Reason values are class names (`http_401`, `timeout`, `oksub_missing`, etc.) — NOT response bodies. The DEBUG log line truncates content to 80 chars with `String(content).slice(0, 80)` — could leak a partial provider response. Acceptable: logs are local-only. If concerns escalate, add a redact step (deferred). |
| T-34-02-05 | Denial of Service | probe rate -> token cost | mitigate | 60s interval is locked (D-01); ~600 tokens/h to copilot is the documented quota. PROXY_PROBE_TIMEOUT_MS=10s prevents probe queueing. |
| T-34-02-06 | Elevation of Privilege | n/a | n/a | This plan only OBSERVES; no privileged actions taken. Plan 34-03 introduces launchctl kickstart, which is gated by `gui/$UID/...` — same UID as coordinator, never root. |
