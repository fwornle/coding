Implement the `claude` subcommand handler in `scripts/convert-transcripts.js`.

Per D-02: `node scripts/convert-transcripts.js claude <path-to-jsonl>`

The handler must:
1. Validate the file path exists and ends with `.jsonl`
2. Open the JSONL file as a read stream (reuse the readline + createReadStream pattern from StreamingTranscriptReader.js -- per D-06, reuse existing reader patterns)
3. For each line: call `parseClaude(line)` from TranscriptNormalizer
4. Collect messages into conversation exchanges (group consecutive user+assistant pairs)
5. For each exchange group: call `observationWriter.processMessages(messages, { agent: 'claude', sourceFile: path })`
6. Report progress to stderr every 100 lines: `[convert] Processed ${n} lines, ${obs} observations...`
7. On completion: print summary to stderr: `[convert] Done: ${totalLines} lines, ${totalObs} observations, ${errors} errors`

Also harden `parseClaude()` in TranscriptNormalizer.js if needed:
- Handle Claude JSONL content blocks (array of `{type: "text", text: "..."}` objects) by joining text blocks
- Handle `tool_use` and `tool_result` message types by mapping to role "tool"
- Skip `system` type messages (they contain prompt caching metadata, not conversation content)
- Handle malformed JSON gracefully (return null, increment error counter)

Test with a real Claude JSONL file if one exists at `~/.claude/projects/` or create a 5-line test fixture inline.

  <action>
Implement the `copilot` subcommand handler in `scripts/convert-transcripts.js`.

Per D-02: `node scripts/convert-transcripts.js copilot <path-to-events.jsonl>`

The handler must:
1. Validate the file path exists
2. Open the events file as a read stream (same readline pattern as claude handler)
3. For each line: call `parseCopilot(line)` from TranscriptNormalizer
4. parseCopilot returns null for non-conversation events -- skip those
5. Collect non-null messages into conversation exchanges (group user+assistant pairs)
6. For each exchange group: call `observationWriter.processMessages(messages, { agent: 'copilot', sourceFile: path })`
7. Progress reporting same pattern as claude handler

Also harden `parseCopilot()` in TranscriptNormalizer.js:
- Handle Copilot event variants: `conversation.message`, `conversation.turn`, `completion.response`
- For `completion.response` events: extract from `data.choices[0].message` shape
- Handle both `data.content` (string) and `data.message.content` (nested) shapes
- Skip events with no meaningful content (e.g., status events, heartbeats)
- Handle multipart content (Copilot sometimes sends incremental delta events -- accumulate into full messages)

Note: Copilot events.jsonl location varies. Common paths:
- `~/.copilot/session-state/events.jsonl`
- `/tmp/copilot-events-*.jsonl`
The CLI takes an explicit path argument, so the user provides the correct location.
