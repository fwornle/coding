Create `src/live-logging/TranscriptNormalizer.js` as an ESM module exporting three parser functions and the MastraDBMessage type shape as a JSDoc typedef.

**MastraDBMessage shape** (per D-05, matches mastra's internal format):
```javascript
/**
 * @typedef {Object} MastraDBMessage
 * @property {string} id - Unique message ID (uuidv4)
 * @property {string} role - 'user' | 'assistant' | 'system' | 'tool'
 * @property {string} content - Message text content
 * @property {string} createdAt - ISO 8601 timestamp
 * @property {Object} [metadata] - Optional metadata
 * @property {string} [metadata.agent] - Source agent: 'claude' | 'copilot' | 'mastra'
 * @property {string} [metadata.sessionId] - Original session identifier
 * @property {string} [metadata.sourceFile] - Path to source transcript file
 * @property {string} [metadata.format] - Original format: 'jsonl' | 'events' | 'specstory'
 */
```

**parseClaude(jsonlLine)**: Parse a single line from Claude JSONL transcript.
- Claude JSONL lines have shape: `{"type":"assistant"|"human"|"system","message":{"role":"...","content":"..."},"timestamp":"..."}`
- Reuse the JSON.parse pattern from StreamingTranscriptReader.js (line ~79) -- do NOT import StreamingTranscriptReader, just use the same parsing approach
- Map `type: "human"` to `role: "user"`, `type: "assistant"` to `role: "assistant"`
- Extract content from `message.content` (may be string or array of content blocks -- handle both)
- Generate deterministic ID from content hash + timestamp to avoid duplicates
- Return null for unparseable lines (skip gracefully)

**parseCopilot(eventLine)**: Parse a single line from Copilot events.jsonl.
- Copilot events have shape: `{"event":"conversation.message","data":{"role":"user"|"assistant","content":"...","timestamp":"..."}}`
- Filter to only `event: "conversation.message"` events, return null for others
- Map role directly (copilot uses standard role names)
- Generate deterministic ID from content hash + timestamp

**parseSpecstory(mdContent)**: Parse a full .specstory markdown file into an array of MastraDBMessages.
- .specstory files use `### Human:` and `### Assistant:` headers to delimit exchanges
- Split on `### Human:` and `### Assistant:` regex patterns
- Extract timestamps from the YAML frontmatter or filename (`YYYY-MM-DD_HHMM-HHMM_hash.md`)
- Return array of MastraDBMessage objects in chronological order
- Handle code blocks (``` fences) within messages without splitting on them

Use `process.stderr.write()` for any logging (per CLAUDE.md, no console.log).
Use `crypto.randomUUID()` for non-deterministic IDs, or `crypto.createHash('sha256')` for deterministic ones.

**ObservationWriter.js** -- ESM class that takes normalized MastraDBMessages and writes observations to LibSQL.

Per D-07/D-08/D-09: observe() LLM calls MUST route through localhost:8089 (the LLM proxy bridge), NOT directly to Google/Anthropic. This fixes the `GOOGLE_GENERATIVE_AI_API_KEY not found` error.

```javascript
class ObservationWriter {
  constructor(options = {}) {
    this.dbPath = options.dbPath || '.observations/observations.db';
    this.proxyUrl = options.proxyUrl || 'http://localhost:8089';
    this.model = options.model || 'google/gemini-2.5-flash'; // from config.json
    this.batchSize = options.batchSize || 10;
  }

  async init() { /* Create/open LibSQL DB, ensure observations table exists */ }
  async summarize(messages) { /* Call LLM proxy to generate observation summary */ }
  async writeObservation(summary, messages, metadata) { /* Write to LibSQL */ }
  async processMessages(messages, metadata) { /* Summarize + write pipeline */ }
  async close() { /* Close DB connection */ }
}
```

Key implementation details:
- `init()`: Use `better-sqlite3` (already in project) or raw SQLite via `node:child_process` calling `sqlite3` CLI to create the observations table: `CREATE TABLE IF NOT EXISTS observations (id TEXT PRIMARY KEY, summary TEXT, messages TEXT, agent TEXT, session_id TEXT, source_file TEXT, created_at TEXT, metadata TEXT)`
- `summarize()`: POST to `${this.proxyUrl}/api/complete` with `{ model: this.model, messages: [{ role: 'system', content: 'Summarize this coding session exchange into a concise observation...' }, { role: 'user', content: JSON.stringify(messages) }] }`. Handle proxy unavailable gracefully (store raw messages without summary).
- `writeObservation()`: INSERT into observations table
- `processMessages()`: Groups messages into chunks (per config messageTokens limit from `.observations/config.json`), summarizes each chunk, writes observations
- Load config from `.observations/config.json` for model and token limits
- All logging via `process.stderr.write()`
