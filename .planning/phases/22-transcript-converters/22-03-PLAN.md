Key implementation details:

**Directory scanning:**
- Use `fs.readdirSync(dirPath)` to list all `.md` files in the specstory history directory
- Sort by filename (filenames are `YYYY-MM-DD_HHMM-HHMM_hash.md` so alphabetical = chronological per D-12)
- Filter out non-.md files

**Manifest tracking (per D-10):**
- Manifest shape: `{ version: 1, files: { "relative/path.md": { hash: "sha256...", convertedAt: "ISO8601", observationCount: N } } }`
- On each file: compute SHA-256 hash, check manifest
- Skip if manifest has matching hash AND `force` is false
- Re-process if hash differs (file was updated) or `force` is true
- Update manifest entry after successful conversion
- Save manifest after each file (crash-safe incremental progress)

**Single file conversion:**
- Read entire file content (specstory files are typically 10-200KB, safe to read fully)
- Call `parseSpecstory(content)` from TranscriptNormalizer to get MastraDBMessage array
- Extract session metadata from filename: date from `YYYY-MM-DD` prefix, session hash from suffix
- Call `observationWriter.processMessages(messages, { agent: 'specstory', sourceFile: filePath, sessionDate: date })`
- Return observation count for manifest tracking

**Progress reporting:**
- After each file: `[specstory] Converted ${filename} (${msgCount} messages, ${obsCount} observations)`
- Skip reporting: `[specstory] Skipped ${filename} (already converted)`
- Final summary: `[specstory] Batch complete: ${converted}/${total} files, ${totalObs} observations, ${skipped} skipped`

All logging via `process.stderr.write()`.
