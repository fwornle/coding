## Trust Boundaries

| Boundary | Description |
|----------|-------------|

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-56-03-01 | Tampering | onTickClick 7-field setState | mitigate | Single atomic Zustand `set({...})` — same pattern as the existing 4-field write; no torn-state window |
| T-56-03-02 | Information Disclosure | timestamp labels render `new Date(ms)` output | accept | Timestamps come from the server-provided session boundaries; rendering them is the explicit user requirement (AC #1) |
| T-56-03-03 | XSS via label content | `toLocaleDateString` output rendered into `<span>` text | accept | The Intl.DateTimeFormat output is a vetted locale-controlled string, not user-supplied. React text rendering escapes content automatically |
| T-56-03-04 | DoS | scaleTicks memo recomputes on every windowKey change | accept | 7-element array build is O(1); recomputation cost negligible |
| T-56-03-05 | Repudiation | Logger.info on tick click | accept | Existing logging discipline preserved; no audit requirement |

All threats LOW. The hand-rolled formatter avoids introducing a fresh d3 sub-module dependency, which would be a (low-risk) supply-chain consideration per CLAUDE.md npm install legitimacy rules.
