```typescript
// In src/store/viewer-store.ts ViewerState interface, after pathToSelected (line 77)
export type SelectionSource = 'graph' | 'timeline' | 'history' | null

export interface ViewerState {
  // ... existing fields ...
  // ---------- Phase 56 — cross-pane selection sync ----------
  selectionSource: SelectionSource
  highlightedRowKey: string | null
  selectedSessionId: string | null

  setSelection: (args: {
    nodeId?: string | null
    sessionId?: string | null
    highlightedRowKey?: string | null
    source: SelectionSource
    pathToSelected?: ReadonlySet<string>
  }) => void
  clearSelection: () => void
}
```

Atomic-setState convention (existing in-tree pattern at LslTimelineStrip.tsx:302-307 and D3GraphCanvas.tsx:493-500):
```typescript
useViewerStore.setState({
  selectedNodeId: id,
  pathToSelected: new Set<string>(),
  highlightedRowKey: id,
  selectionSource: 'history',
})
```
Downstream plans MUST use `setState({...})` (multi-field atomic) rather than chained named setters when writing more than one selection field at once — otherwise subscribers see torn state. Single-source writes (e.g. `setSelectedNode(id)` solo) remain fine for BC.

clearSelection() contract:
- Nulls: selectedNodeId, selectedEdgeId, selectionSource, highlightedRowKey, selectedSessionId
- Resets: pathToSelected → new Set<string>()
- Also clears: lslSessionFilter=[], lslFilterEntityIds=null (cascade per CONTEXT.md "Esc + click-background clears in all three panes")
- Does NOT touch: searchQuery, selectedClasses, filter-related slices
