Centering effect (NEW â€” placed AFTER the existing applySelectionStyling effect at lines 213-219):
```typescript
useEffect(() => {
  if (!svgRef.current || !selectedNodeId) return
  if (selectionSource === 'graph') return  // self-originated; no centering
  const node = visibleEntities.find((e) => e.id === selectedNodeId)
  if (!node || node.x === undefined || node.y === undefined) return
  const svg = d3.select(svgRef.current)
  const { width, height } = svgRef.current.getBoundingClientRect()
  const transform = d3.zoomIdentity.translate(width / 2 - node.x, height / 2 - node.y).scale(1)
  if (zoomBehaviorRef.current) {
    svg.transition().duration(500).call(zoomBehaviorRef.current.transform, transform)
  }
}, [selectedNodeId, selectionSource, visibleEntities])
```
Note: `visibleEntities` is in the dep list because nodes need their x/y from the force simulation to be settled before centering â€” Picking a node before settle returns undefined coords; the effect bails. The bail-out is intentional; the user will see the highlight without centering on rare cold-start clicks.
