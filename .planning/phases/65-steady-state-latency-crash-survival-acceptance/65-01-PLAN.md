ClaudeWorker (extends EventEmitter):
  new ClaudeWorker(model, systemPrompt, deps={})  // deps.idleMs injectable (Plan 63-01); default LLM_PROXY_WORKER_IDLE_MS
  .pid                                            // the real `claude -p` subprocess pid; read DIRECTLY off the handle (T-63-12)

  dispose()                                       // stdin.end() + SIGTERM; clears idle timer
  emits ONLY 'exit' — there is NO 'result' event to subscribe to for the raw cache fields
WorkerPool:
  new WorkerPool(opts={})                         // opts.idleMs / opts.crashThreshold / opts.crashWindowMs injectable
  keyFor(body) -> key
  workersByKey: Map<key, ClaudeWorker[]>
  async complete(body, abortSignal, overflowFn)  // CLI-fallback path; lazily spawns; honors LLM_PROXY_DISABLE_WORKER_POOL=1 -> overflowFn
  disposeAll()                                    // dispose every worker + clear the map (teardown uses this)

COMPLETION TOKEN SHAPE (load-bearing for D-02): the completion returned by write()/complete() is
  { content, model, tokens: { input, output, total } }
where `tokens.input` is the SUMMED cache-inclusive figure
  inputTokens + cacheReadInputTokens + cacheCreationInputTokens
(see inputTokensFromEvent + toCompletion, worker-pool.mjs:191-231). The DEDICATED `cacheReadInputTokens`
split is NOT surfaced separately on the completion, and the worker emits no 'result' event carrying the
raw modelUsage. Therefore the in-scope cache-presence signal is the SUMMED `tokens.input` floor (a genuine
warm turn re-reads the cached prompt/context, so input is in the hundreds/thousands, not ~1) — NOT a
dedicated cacheRead assertion. Surfacing the split would be a production worker-pool.mjs change (out of
scope for this verify-only phase).
