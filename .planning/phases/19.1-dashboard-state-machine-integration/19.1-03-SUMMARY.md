# Phase 19.1 Plan 03: Enrich State Machine with Trace Data Summary

**Enriched step-complete events with trace fields (agentInstances, llmCallEvents, entityFlow, qaResult) accumulated in stepsDetail through state machine, preserved by bridge for dashboard trace modal**

## Performance

- **Duration:** 7 min
- **Started:** 2026-03-11T18:02:55Z
- **Completed:** 2026-03-12T00:10:00Z
- **Tasks:** 2
- **Files modified:** 4

## Accomplishments
- Expanded StepCompleteEventSchema with optional trace fields (wave, tokensUsed, llmCalls, agentInstances, llmCallEvents, entityFlow, qaResult)
- Added StepDetailSchema and stepsDetail array to RunningState and CompletedState types
- Created buildStepComplete helper in wave-controller that attaches stepMetrics to all 4 dispatch calls
- State machine transition handler accumulates enriched stepsDetail on each step-complete
- Bridge preserves enriched stepsDetail when converting state machine format to legacy format
- Fixed terminal state filter so completed/failed/cancelled workflows appear in API for 30 minutes

## Files Created/Modified
- `integrations/mcp-server-semantic-analysis/src/shared/workflow-types/transitions.ts` - Enriched StepCompleteEventSchema, stepsDetail accumulation in step-complete handler, stepsDetail carry-through to completed state
- `integrations/mcp-server-semantic-analysis/src/shared/workflow-types/state.ts` - Added StepDetailSchema, stepsDetail field on RunningState and CompletedState
- `integrations/mcp-server-semantic-analysis/src/agents/wave-controller.ts` - buildStepComplete helper, replaced 4 inline dispatches
- `integrations/system-health-dashboard/server.js` - Terminal state filter fix, bridge uses enriched stepsDetail

## Decisions Made
- Used z.record(z.string(), z.unknown()) for agentInstances, llmCallEvents, entityFlow, qaResult in Zod schemas to avoid coupling to TraceAgentInstance/TraceLLMCall interfaces (different field names like agentId vs agentName)
- Carried stepsDetail from RunningState through to CompletedState so terminal state bridge can access full trace data
- Terminal states included via isRecentTerminal (age < 30min) rather than removing the terminal filter entirely

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Schema type mismatch with TraceAgentInstance/TraceQAResult**
- **Found during:** Task 1 (schema expansion)
- **Issue:** Plan specified typed agentInstances schema with `agentName` field, but TraceAgentInstance uses `agentId`/`agentType`. Similarly TraceQAResult.passed is boolean, not number.
- **Fix:** Used z.record(z.string(), z.unknown()) for flexible pass-through schemas; cast in buildStepComplete helper
- **Files modified:** transitions.ts, state.ts, wave-controller.ts
- **Verification:** npm run build passes
- **Committed in:** 8acfc40 (Task 1 commit)
