<objective>
Land the two lifecycle FOUNDATIONS the rest of Phase 63 builds on: (1) D-08 — make pool-side disposal remove the worker from `workersByKey` SYNCHRONOUSLY (not only via the async `'exit'` event), closing the acquire-after-SIGTERM race; and (2) D-04 / WLIFE-02 — a per-worker `unref()`'d idle timer that evicts a quiet worker after `LLM_PROXY_WORKER_IDLE_MS` (default 30 min), freeing its RAM, with the next request lazily respawning.

Purpose: D-08 is the synchronous-drop invariant that cancel (Plan 04), crash-cooldown (Plan 03) and idle-evict (this plan) all reuse — it must exist first. Idle eviction is the simplest lifecycle behavior and rides directly on the existing `dispose()` -> `'exit'` -> `_dropWorker` reap path, so it pairs naturally with the D-08 fix.
Output: `ClaudeWorker` idle timer + a `WorkerPool` synchronous dispose-and-drop helper, plus unit coverage for both.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pool acquire ↔ worker dispose | `complete`'s `workers.find((w) => !w.busy)` runs concurrently with lifecycle disposal; a worker SIGTERMed but not yet spliced is an untrusted/zombie handle |
| background timer ↔ event loop | an idle `setTimeout` that is NOT unref'd would pin the proxy process alive at shutdown |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-63-01 | Tampering/DoS | acquire-after-SIGTERM race | mitigate | `_disposeAndDrop` splices the worker from `workersByKey` SYNCHRONOUSLY before returning, so a concurrent `acquire` cannot pick a disposed worker (D-08). Proven by the Task-1 synchronous-drop unit case. |
| T-63-02 | Denial of Service | dangling idle timer keeps proxy alive | mitigate | idle timer is `unref()`'d (never keeps the event loop alive) and `dispose()` clears it so shutdown/disposeAll leaves no live timer. Proven by the Task-2 cleared-on-dispose case. |
| T-63-03 | DoS | idle timer never fires / leaks RAM | mitigate | `_clearIdleTimer` on dispatch + re-arm on settle bound the worker's idle life to `LLM_PROXY_WORKER_IDLE_MS`; on fire it disposes and is reaped. Proven by the Task-2 idle-fire case. |
| T-63-SC | Tampering | npm/pip/cargo installs | accept | No new packages — Node builtins only (`node:child_process`, `node:crypto`, `node:events`). No install task in this plan; package legitimacy gate N/A. |
