<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| worker stdin write ↔ subprocess | the `claude` subprocess pipe can close mid-write (worker crashed); the write is untrusted to succeed |
| worker stdout event ↔ pending promise | a `result` event is untrusted to belong to the CURRENT request — a late/stray result from a prior turn must not satisfy the next caller |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-63-04 | DoS / Information Disclosure | unguarded EPIPE throw loses queued-job rejection (caller hangs forever) | mitigate | `_dispatch` write is guarded; any write error routes to `_onExit`, which rejects in-flight + drains queue RETRYABLE and emits 'exit'. Proven by the D-07 drains-queue unit case. |
| T-63-05 | Spoofing / Information Disclosure | a disposed/late worker leaks a stray `result` onto the NEXT caller (cross-request data bleed) | mitigate | monotonic generation guard drops any `result` not matching the live `_pending.generation`; combined with dispose-on-cancel (Plan 04) the disposed worker is also dropped from the pool. Proven by the D-02 "stray does not resolve next request" case. Residual severity LOW: concurrency-1 + `_pending=null`-on-settle already covers the common case; the generation guard closes the narrow in-flight window. |
| T-63-SC | Tampering | npm/pip/cargo installs | accept | No new packages — Node builtins only. No install task; package legitimacy gate N/A. |
