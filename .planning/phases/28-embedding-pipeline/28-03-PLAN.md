**Part A: Create `src/embedding/listener.ts`** (per D-05, D-06)

This is a long-running Redis pub/sub subscriber that embeds new items into Qdrant. No analog exists in the codebase -- use RESEARCH.md Pattern 4 and PATTERNS.md inferred structure.

```typescript
#!/usr/bin/env node
import Redis from "ioredis";
import { getEmbeddingService } from "./embedding-service.js";
import { getQdrantClient, ensureCollections } from "./qdrant-collections.js";
import { contentHash } from "./content-hash.js";
```

**Event message format** (published by ObservationWriter):
```typescript
interface EmbeddingEvent {
  type: "observation" | "digest" | "insight" | "kg_entity";
  id: string;           // UUID point ID for Qdrant
  content: string;      // Text to embed
  metadata: Record<string, string | number | boolean | null>;
  timestamp: string;    // ISO date
}
```

**Listener logic:**
1. Initialize: `const embedder = getEmbeddingService(); await embedder.initialize();`
2. Connect to Qdrant: `const qdrant = getQdrantClient(); await ensureCollections(qdrant);`
3. Connect to Redis: `const sub = new Redis(process.env.REDIS_URL || "redis://localhost:6379");`
4. Subscribe: `sub.subscribe("embedding:new")` with error handler that logs and exits
5. On message handler:
   ```typescript
   sub.on("message", async (channel, message) => {
     try {
       const event: EmbeddingEvent = JSON.parse(message);
       const collectionName = event.type === "kg_entity" ? "kg_entities" : `${event.type}s`;
       const hash = contentHash(event.content);
       const vector = await embedder.embedOne(event.content);
       await qdrant.upsert(collectionName, {
         wait: false,  // Don't block on Qdrant persistence for real-time path
         points: [{
           id: event.id,
           vector: Array.from(vector),
           payload: {
             ...event.metadata,
             content_hash: hash,
             model_version: embedder.getConfig().model,
             summary_preview: event.content.substring(0, 200),
           },
         }],
       });
       process.stderr.write(`[EmbeddingListener] Embedded ${event.type} ${event.id.substring(0, 8)}...\n`);
     } catch (err: unknown) {
       const msg = err instanceof Error ? err.message : String(err);
       process.stderr.write(`[EmbeddingListener] Failed to process event: ${msg}\n`);
       // Do NOT crash -- continue processing next events
     }
   });
   ```
6. Log startup: `process.stderr.write("[EmbeddingListener] Subscribed to embedding:new, waiting for events...\n");`
7. Graceful shutdown: handle SIGTERM/SIGINT to `sub.disconnect()` and exit cleanly

Add Redis publisher as a lazy-initialized, fire-and-forget optional dependency. Observations MUST never fail because Redis is down.

1. Add import at top of file (after existing imports):

   ```

2. In the constructor (near line 62 where `this._writeLock` is set), add:

   ```

3. Add a new method `_initRedis()` called lazily from `writeObservation`:
   ```javascript
   /**
    * Lazily initialize Redis publisher for embedding events.
    * Fire-and-forget: if Redis is unreachable, observations still work.
    */
   _initRedis() {
     if (this._redisInitAttempted) return;
     this._redisInitAttempted = true;
     try {
       this._redisPub = new Redis(process.env.REDIS_URL || "redis://localhost:6379", {
         maxRetriesPerRequest: 1,
         retryStrategy: () => null,  // Don't retry -- fail fast
         lazyConnect: true,
         connectTimeout: 2000,
       });
       this._redisPub.connect().catch((err) => {
         process.stderr.write(`[ObservationWriter] Redis connect failed (non-fatal): ${err.message}\n`);
         this._redisPub = null;
       });
     } catch (err) {
       process.stderr.write(`[ObservationWriter] Redis init failed (non-fatal): ${err.message}\n`);
       this._redisPub = null;
     }
   }
   ```

4. In `writeObservation()`, AFTER the `stmt.run(...)` call (line 430) and BEFORE `this._scheduleExport()` (line 433), add:
   ```javascript
   // Fire-and-forget: publish embedding event to Redis (per D-05)
   this._initRedis();
   if (this._redisPub) {
     this._redisPub.publish("embedding:new", JSON.stringify({
       type: "observation",
       id,
       content: redactedSummary,
       metadata: { agent, quality, date: nowISO.slice(0, 10), project: "coding" },
       timestamp: nowISO,
     })).catch((err) => {
       process.stderr.write(`[ObservationWriter] Redis publish failed (non-fatal): ${err.message}\n`);
     });
   }
   ```

**Key design decisions:**
- Redis connection uses `lazyConnect: true` + `connectTimeout: 2000` + `retryStrategy: () => null` to fail fast without blocking
- `_redisInitAttempted` flag prevents repeated init attempts if Redis is down
- The `publish().catch()` pattern ensures the promise rejection doesn't crash the process
- `redactedSummary` is used (not raw summary) to avoid leaking sensitive data to the embedding pipeline
