Create `src/embedding/backfill.ts` as a one-shot CLI script per D-07 and D-08. Structure follows `scripts/backfill-raw-observations.js` (batch loop with progress) and `scripts/sync-graph-to-qdrant.js` (init/teardown wrapper).

**CLI interface:**
```
npx tsx src/embedding/backfill.ts [--dry-run] [--batch-size N] [--tier observations|digests|insights|kg_entities]
```
- `--dry-run`: count items but don't embed or upsert
- `--batch-size N`: override default 64
- `--tier X`: only backfill one tier (default: all four)

**Script structure:**
1. Parse args (simple loop over `process.argv.slice(2)`, matching `--dry-run`, `--batch-size`, `--tier`)
2. Initialize services in `main()`:
   - `const embedder = getEmbeddingService(); await embedder.initialize();`
   - `const qdrant = getQdrantClient();`
   - `await ensureCollections(qdrant);` (idempotent collection creation)
   - `const db = new Database(".observations/observations.db", { readonly: true });`
3. Backfill each tier in sequence (or just one if `--tier` specified):

**Observations (EMBED-01):**
- Query: `SELECT id, summary, agent, created_at, quality FROM observations WHERE summary IS NOT NULL`
- Text to embed: `summary` field (the 4-line structured text, NOT raw messages)
- Qdrant payload: `{ agent, project: "coding", date: created_at?.split("T")[0], quality, content_hash, model_version: config.model, summary_preview: summary.substring(0, 200) }`
- Point ID: `id` (already UUID from `crypto.randomUUID()`)

**Digests (EMBED-02):**
- Query: `SELECT id, summary, created_at, quality FROM digests WHERE summary IS NOT NULL`
- Text to embed: `summary` field
- Qdrant payload: `{ date: created_at?.split("T")[0], quality, content_hash, model_version: config.model, summary_preview: summary.substring(0, 200) }`
- For `theme` and `agents`: extract from digest metadata if available in a `metadata` column, otherwise omit
- Point ID: `id` (UUID)

**Insights (EMBED-03):**
- Query: `SELECT id, summary, created_at FROM insights WHERE summary IS NOT NULL`
- Text to embed: `summary` field
- Qdrant payload: `{ content_hash, model_version: config.model, summary_preview: summary.substring(0, 200) }`
- For `topic`, `confidence`, `digestIds`: extract from insight metadata/content if columns exist, otherwise omit
- Point ID: `id` (UUID)

**KG Entities (EMBED-04):**
- Read from LevelDB at `.data/knowledge-graph/` using the `level` package
- The graph is stored as a Graphology serialized graph. Read the key that stores the full graph JSON (typically a single key containing the serialized graph). Parse it to extract nodes.
- ALTERNATIVE: If Graphology serialization is complex, use the `.data/observation-export/` JSON files or query the Qdrant `knowledge_patterns_small` collection for existing entity data. Check what's available.
- Filter: Skip nodes where `isScaffoldNode === true` or where observations count < 2 (per Pitfall 3 in RESEARCH.md)
- Text to embed: concatenate `name + "\n" + observations.join("\n")` (node name plus its observation strings)
- Qdrant payload: `{ entityType: node.type, hierarchyLevel: node.level, parentId: node.parentId, content_hash, model_version: config.model, summary_preview: text.substring(0, 200) }`
- Point ID: Generate deterministic UUID from node key using `crypto.createHash("md5").update(key).digest("hex")` formatted as UUID v4 shape (`xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx` with hex chars). This avoids Qdrant's UUID validation error on arbitrary key strings like `coding:CollectiveKnowledge` (per Pitfall 4 in RESEARCH.md).

**Batch embedding flow (for each tier):**
```typescript
const BATCH_SIZE = options.batchSize;
for (let i = 0; i < items.length; i += BATCH_SIZE) {
  const batch = items.slice(i, i + BATCH_SIZE);
  const texts = batch.map(item => item.text);
  const hashes = texts.map(t => contentHash(t));

  // Idempotency check: scroll Qdrant for existing content_hash matches
  const toEmbed: typeof batch = [];
  for (let j = 0; j < batch.length; j++) {
    const existing = await qdrant.scroll(collectionName, {
      filter: { must: [{ key: "content_hash", match: { value: hashes[j] } }] },
      limit: 1, with_payload: false, with_vector: false,
    });
    if (existing.points.length === 0) {
      toEmbed.push({ ...batch[j], hash: hashes[j] });
    } else {
      skipped++;
    }
  }

  if (toEmbed.length === 0) continue;
  if (options.dryRun) { dryRunCount += toEmbed.length; continue; }

  // Embed and upsert
  const embedTexts = toEmbed.map(item => item.text);
  const embeddings = embedder.embedBatch(embedTexts, BATCH_SIZE);
  for await (const vectors of embeddings) {
    const points = vectors.map((vec, j) => ({
      id: toEmbed[j].id,
      vector: Array.from(vec),
      payload: { ...toEmbed[j].payload, content_hash: toEmbed[j].hash, model_version: config.model },
    }));
    await qdrant.upsert(collectionName, { wait: true, points });
    embedded += points.length;
  }

  process.stderr.write(`  [${collectionName}] ${i + batch.length}/${items.length} processed (${embedded} embedded, ${skipped} skipped)\n`);
}
```

**Progress reporting:** Use `process.stderr.write` (never `console.log`). Report per-tier counts at end:
```
[Backfill] Observations: 558 embedded, 0 skipped
[Backfill] Digests: 132 embedded, 0 skipped
[Backfill] Insights: 12 embedded, 0 skipped
[Backfill] KG Entities: 145 embedded, 15 skipped (scaffold)
[Backfill] Total: 847 points across 4 collections
```

**Error handling:**
- Wrap `main()` in try/catch with `process.exit(1)` on fatal errors
- Per-item try/catch in the batch loop — log error, increment `failed` counter, continue
- Close `db` and LevelDB in `finally` block

**Logging:** All output via `process.stderr.write("[Backfill] ...")` — no `console.log`.
