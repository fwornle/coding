<objective>
Add the single per-request write hook at the proxy (`_work/rapid-llm-proxy/proxy-bridge/server.mjs`, runtime JS — NO build) that persists one context-turns JSONL line per measured request, plus the same-origin `GET /api/context-turns` read route. This is the core of the phase: the line assembles from data already in scope at both write sites (cache split, taxonomy, tool names, preview) and is appended plaintext during the span (D-03 append half). Observation correlation is NOT done here (Pitfall 1 — resolved at span close in Plan 05).

Purpose: Reveal every measured request's context honestly, reusing existing parsers/taxonomy (RESEARCH: ~90% wiring).
Output: `logContextTurn` + path helper + breakpoint-index emit + both call sites + the read route, all best-effort never-throw.

## Shared file contract (pinned — both this plan and Plan 05 depend on it)
- Dir: `<CODING_ROOT>/.data/measurements/<safeSanitizeTaskId(task_id)>/`
- Plaintext file during span: `context-turns.jsonl` (one JSON object per line).
- Line shape (field names Claude's Discretion but MUST include these keys):
  `{ ts, task_id, agent, wire:'anthropic'|'openai', request_id, model,
     usage:{ input, output, cache_read, cache_write }, cache_breakpoints:[msgIdx…],
     categories:[…], messages:[{ i, role, bytes, tool:{name,size}|null, preview }],
     observation_ref: null }`
- OpenAI wire: set `usage.cache_write` to `null` (parseOpenAICache always returns 0; null signals "provider reports none" so the UI renders N/A, not 0 — D-12).
</objective>
