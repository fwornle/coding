# Phase 87 — UI Design Contract

> Visual and interaction contract for the interactive-spans-and-branch-avenues dashboard surfaces. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **BROWNFIELD EXTENSION.** This phase adds surfaces to the existing `integrations/system-health-dashboard` React + Redux app (Performance tab / Phase 85 Control Center). It does NOT introduce a new design system. Every token, component, and colour below is **extracted from the existing dashboard**, not proposed. The new surfaces (Fork action, variant picker, sweep guardrail, origin-grouped N-way panel, merge-status badge, prune/promote actions) MUST reuse the components and tokens already in `src/components/ui/`, `src/index.css`, `tailwind.config.ts`, and `src/lib/colors.ts`. Do not invent a new palette, spacing scale, or typography.

## Spacing Scale

Declared values (must be multiples of 4). These are the **Tailwind default scale already in use across the dashboard** — `gap-2` (8), `gap-3` (12), `gap-4` (16), `px-2`/`py-1.5`, `space-y-3`, `p-2`, `mt-2` are the observed idioms in `difference-viewer.tsx` / `experiment-launcher.tsx`. Use these tokens, not new ones.

| Token | Value | Usage (observed in existing components) |
|-------|-------|-----------------------------------------|
| xs | 4px | `gap-1` icon↔label inside badges (`reconciliation-badge` `gap-1`); `gap-0.5` header sub-lines |
| sm | 8px | `gap-2` control rows (`flex flex-wrap items-center gap-2`); `px-2 py-1.5` cell padding |
| — | 12px | `gap-3` / `space-y-3` card-content section rhythm (`difference-viewer` `CardContent space-y-3`, `gap-x-3`) — Tailwind `3` unit, already pervasive |
| md | 16px | `gap-4` grid gaps between paired columns (`grid grid-cols-2 gap-4`) |
| lg | 24px | `p-6` default shadcn `Card` padding (card component) |
| xl | 32px | major section breaks between stacked `Card`s on the Performance tab |
| 2xl | 48px | page-level top rhythm (rare on this dense data surface) |

Exceptions: **12px (`gap-3`/`space-y-3`) is a load-bearing existing rhythm** in the Performance components (`difference-viewer.tsx` `space-y-3`, `gap-x-3 gap-y-2`) and is retained even though it is not a strict 8-point step. Control heights are the shadcn defaults: inputs/selects `h-9` (36px), `size="sm"` buttons `h-8` (32px). Do not introduce new spacing values.

## Typography

Extracted from the existing Performance components — the dashboard uses a compact, data-dense scale. Two weights only (`400` regular, `600` semibold via `font-semibold`/`font-medium`). `font-mono` (tabular) is reserved for machine values: task_ids, model names, token/cost counts, cell counts.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Card title | 16px (`text-base`, `CardTitle className="text-base"`) | 600 (`font-semibold`, shadcn CardTitle default) | 1.2 (`leading-none` per shadcn CardTitle) |
| Body / control label | 14px (`text-sm`) | 400 | 1.5 (`:root` default) |
| Secondary / muted caption | 12px (`text-xs`, `text-muted-foreground`) | 400 | 1.5 |
| Micro / dense metric | 11px (`text-[11px]` / `text-[10px]` badges) | 400 | 1.5 |

**Mono treatment (not a new size — a family switch):** `font-mono` on `text-sm`/`text-xs` for task_ids (`difference-viewer` `ModelHeader`), model names (`normalizeModel`), token counts (`CumulativeDelta` `font-mono`), and the sweep cell count (`matrix-cell-count` `font-mono font-semibold`). All numeric columns in the N-way panel MUST be `font-mono` for column alignment. Signed deltas use U+2212 (minus) not hyphen, matching `difference-viewer` `CumulativeDelta`.

## Color

The dashboard is **theme-aware (light + `.dark`)**. Colour comes from **three sanctioned sources only** — never hardcoded hex in TSX:

1. **shadcn CSS-var tokens** (`bg-background`, `bg-card`, `text-foreground`, `text-muted-foreground`, `bg-muted/40`, `border`, `ring-primary`, `bg-primary/10`, `text-destructive`, `bg-destructive/10`) — from `src/index.css`.
2. **Semantic status tokens** (`text-status-success` / `bg-status-success-subtle` / `border-status-success-line`, and `-warning` / `-error` / `-info` / `-neutral`) — from `tailwind.config.ts` `status.*` → `--st-*` CSS vars. **These are the ONLY correct source for merge-state and pass/fail colour.**
3. **`src/lib/colors.ts` JS palette** (`useWorkflowColors()`) — **only** for SVG `fill=`/`stroke=` presentation attributes (which cannot resolve `var()`). The N-way panel is HTML/CSS, so it uses tokens #1/#2; only if a merge-state indicator is drawn as an SVG glyph does it read from `colors.ts`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(var(--background))` / `hsl(var(--card))` (white light / slate-950 dark) | Page + card surfaces |
| Secondary (30%) | `hsl(var(--muted))` / `bg-muted/40` (slate-100 light / slate-800 dark) | Preview boxes, sub-panels, table zebra, disabled cells |
| Accent (10%) | `hsl(var(--primary))` (slate-900 light / near-white dark) | See reserved list below |
| Destructive | `hsl(var(--destructive))` (red-500 / dark red-800) | Prune button, delete/prune confirm bar only |

**Accent (`--primary`) reserved for — explicit list, never "all interactive elements":**
- The **primary CTA** of the fork flow: the "Fork into avenues" launch/confirm `Button` (default shadcn `Button` = `bg-primary`).
- The **transient pre-fill highlight ring** (`ring-2 ring-primary`) when the variant picker is populated from a fork (matches `experiment-launcher` `ring-2 ring-primary` DEFECT-B idiom).
- The **selected-for-compare** row emphasis in the N-way panel (2-of-N selection state).

All other buttons in these surfaces are `variant="ghost"` or `variant="outline"` (row actions: Fork, Promote, Prune-trigger), matching the runs-table row-action idiom.

**Merge-status badge colour mapping (D-04) — status tokens only, icon + label always (never colour alone):**

| Merge state | Token classes | lucide icon | Label |
|-------------|---------------|-------------|-------|
| merged | `border-status-success-line text-status-success` | `Check` | `merged` |
| unmerged | `text-muted-foreground` (neutral) | `GitBranch` | `unmerged` |
| conflicts | `border-status-warning-line text-status-warning` | `AlertTriangle` | `conflicts` |
| unknown / not-computed | (no badge rendered) | — | — (honesty: never fabricate a state, mirrors `reconciliation-badge` `null → no badge`) |

Ranking colour cue in the N-way panel: the best-ranked row (top outcome score) may carry a subtle `border-status-success-line` left accent; do NOT colour whole rows — keep the surface neutral and let the mono numbers rank. A worse-than-origin score uses `text-muted-foreground`, matching `difference-viewer`'s "increase → muted, decrease → success" honesty convention.

**Destructive actions in this phase (each + confirmation approach):**

1. **Prune avenue branch (D-05)** — removes the `avenue/<task_id>` git branch. Confirmation: inline `bg-destructive/10` confirm bar (the `runs-table.tsx:395` pattern), `role="alertdialog"`. Copy: **`Prune branch avenue/{task_id}? Measurement data stays in .data — only the branch is removed. This cannot be undone.`** Buttons: `Cancel` (`variant="ghost"`) + `Prune` (`variant="destructive"`). The "measurement data stays" reassurance is mandatory (D-05 guarantee).
2. **Promote to main (D-04)** — merges an avenue branch into main; state-changing, so it gets a confirm step (not destructive-red, use `variant="default"` confirm). Copy: **`Promote avenue/{task_id} to main? This merges the winning avenue's code changes into main.`** If git reports conflicts, block with `text-status-warning`: **`avenue/{task_id} has conflicts with main — resolve them before promoting.`**
3. **Launch a sweep (D-02)** — not destructive but cost-bearing; the count+cost preview + explicit `Launch {N} avenues` confirm IS the guardrail. Never auto-launch on toggle.

## Interaction Contract (surface-specific, for the executor)

These are prescriptive interaction contracts the executor implements against; they sit above the visual tokens.

1. **Fork trigger location:** a `Fork into avenues` ghost button appears **only on COMPLETED span rows** in `runs-table.tsx` (guard identical to the D-11 Re-run "only on completed experiment runs" guard). Clicking it opens/pre-fills the variant picker and scrolls it into view with the `ring-2 ring-primary` transient highlight.
2. **Variant picker axes (D-03):** four axes rendered as labelled groups — **Agent** (`claude` / `copilot` / `opencode` / `mastra`, multi-select), **Model** (`opus` / `sonnet` / `gpt-5` / `haiku` / …, multi-select), **SDD framework** (`gsd` / `spec-workflow` / `none`, multi-select, disambiguated label), **Knowledge injection** (on/off, prominent toggle). Curated-by-default: each picked combination = one avenue chip. A **Sweep** toggle expands chosen axes into their cross-product.
3. **Sweep guardrail (D-02, MANDATORY):** whenever a sweep would expand the matrix, the count+cost preview MUST render **before** any launch is possible, and the launch button stays disabled until it has. The count comes from the **server-resolved** cell count (never a client-side axes recompute — the `experiment-launcher` D-09 discipline). Over-threshold shows a `text-status-warning` caution line.
4. **Origin-grouped N-way panel (D-06/D-07):** a `Card` per origin span containing a sortable `Table`; **default sort = outcome score (Phase 73), best first**; secondary sortable columns = **tokens/cost, route quality, wall-clock** (all `font-mono`, right-aligned). Each row carries the **merge-status badge**, a **Promote** action, and a **Prune** action. Row selection is 2-of-N; enabling a `Compare selected (2)` CTA that calls `setCompareA/setCompareB` and switches to the Compare tab where the existing `DifferenceViewer` renders (the 86-05 wiring — do not fork it).
5. **Honesty carries forward (Phases 82/83/86):** null/unmeasured values render as em-dash or italic `unmeasured`, never `0` or a fabricated status. Merge state `unknown` → no badge. Reconciliation/score cells stay verbatim server values.
6. **Theme:** every new element must read correctly in BOTH light and `.dark` (use tokens, never raw hex). Verify visually in both themes at `localhost:3032` via `gsd-browser` (per CLAUDE.md — no hand-rolled Playwright scripts).
