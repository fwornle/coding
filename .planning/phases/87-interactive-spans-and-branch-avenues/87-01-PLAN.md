<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| task_id → git branch/worktree argv | task_id originates from a Run record / fork request; flows into `git worktree add -b avenue/<id>` and prune argv |
| host process → git working tree | these primitives mutate `.git/worktrees/` and create branches on the host repo |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-87-01-01 | Tampering | branch/worktree name from task_id | mitigate | `sanitizeTaskId` (`[A-Za-z0-9._-]` + path.basename) applied before any argv is built; branch name provably matches `avenue/[A-Za-z0-9._-]+`; unit test rejects metachar/`..` |
| T-87-01-02 | Tampering (cmd injection) | every git call | mitigate | fixed-argv `spawnSync('git',[...])` only, never `shell:true`, never string-concat into one arg; grep gate asserts no `shell: true` |
| T-87-01-03 | Tampering | prune deleting the wrong worktree/branch | mitigate | prune keyed on the sanitized `avenue/<task_id>` and the `.data/avenues/<id>` path only; `git branch -D` scoped to the exact `avenue/<id>` ref; idempotent no-op when absent |
| T-87-01-04 | Elevation | host git mutation from container | accept | these are host-only lib primitives; the container never imports them (enforced by Plan 05 routing all state-changing git through the host coordinator) — no container code path reaches here |
</threat_model>
