| Boundary | Description |
|----------|-------------|
| dashboard/container → vkb-server → coordinator | promote/prune requests cross from the untrusted UI to the host git executor |
| task_id in request → git branch argv | request body task_id flows into merge-status/promote/prune git ops |
| avenue branch → main | promote merges branch content into the main working tree |

## STRIDE Threat Register

| T-87-04-01 | Elevation | promote merging attacker-controlled branch into main | mitigate | host-only (coordinator seam, not container); conflict-blocked (promoteAvenue rejects when merge-tree reports conflicts); UI confirm gate (Plan 06) |
| T-87-04-02 | Tampering | prune deleting the wrong branch/worktree | mitigate | prune keyed on sanitized `avenue/<task_id>` (Plan 01); fixed-argv `git worktree remove`/`branch -D`; idempotent |
| T-87-04-03 | Tampering | main mutation during status compute | mitigate | merge-status uses read-only `git merge-tree`/`rev-list` only; unit test asserts main HEAD unchanged by any status call |
| T-87-04-04 | Access Control | external origin invoking host git ops | mitigate | coordinator endpoints reuse the `:2737` `req.socket.remoteAddress` localhost-only reject; vkb-server only proxies (no git argv in the container-adjacent layer) — Pitfall 6 |
| T-87-04-05 | Tampering (cmd injection) | task_id → git argv | mitigate | sanitizeTaskId at both the coordinator handler and the primitive; fixed-argv only, no shell:true |
