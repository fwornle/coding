## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → completeClaudeCode dispatcher | Untrusted `body.model` + messages decide pool key + routing; model is normalized via resolveClaudeModel. |
| dispatcher → WorkerPool → ClaudeWorker subprocess | Request crosses into a per-(model×prompt) worker; mis-routing across models would silently serve the wrong model (Q2). |
| operator env (`LLM_PROXY_DISABLE_WORKER_POOL`, `DISABLE_CLAUDE_DIRECT`) → dispatcher | Env flags compose the escape-hatch behavior (D-08). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-62-03-01 | Spoofing/Integrity | Wrong-model worker silently serves the wrong model | mitigate | Router keys STRICTLY by `resolveClaudeModel(body.model)`; a worker is never reused across models (Pitfall 2). Plan-01 POOL-02 test asserts a sonnet request never lands on a haiku worker. There is no protocol guard (Q2) — the router is the only line of defense. |
| T-62-03-02 | Tampering | Env-driven escape-hatch composition opens an unintended path | mitigate | Both flags are read via strict `=== '1'` exact match (no eval, no shell, no string coercion); D-08 truth table is the spec; the two flags are orthogonal and not coupled. Both-set degrades to today's pure-execFile baseline (safe). |
| T-62-03-03 | Denial of Service (RAM) | Unbounded prompt-pool fragmentation via many distinct system prompts | mitigate | LRU prompt-pool cap (`LLM_PROXY_WORKER_PROMPT_CAP`, D-02) drains the least-recently-used prompt-pool; per-key worker count capped at `LLM_PROXY_WORKER_POOL_SIZE` (D-05); overflow degrades to execFile rather than spawning unboundedly (D-06). |
| T-62-03-04 | Information Disclosure | Cross-call context leakage between callers sharing a worker | mitigate | Pool skips+replaces `needsRecycle`/`isStale` workers (Plan 02 threshold-driven recycle) before reuse; distinct system prompts already land in distinct pools (D-01). |
| T-62-03-05 | Denial of Service | Orphaned workers on proxy restart | mitigate | `workerPool.disposeAll()` wired into the proxy shutdown path reaps all workers; parent launchd job manages the proxy. |
| T-62-03-SC | Tampering | npm/pip/cargo installs | mitigate | Zero packages installed — all Node builtins. No install task; no legitimacy checkpoint required (RESEARCH § Package Legitimacy Audit: N/A). |
</threat_model>
