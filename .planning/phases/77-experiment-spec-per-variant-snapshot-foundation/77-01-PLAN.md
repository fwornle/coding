<objective>
Create `lib/experiments/experiment-spec.mjs` — the pure YAML spec resolver/validator that a
user declares an experiment with (SPEC-01) and that fails fast on invalid or unsupported
cells before any run starts (SPEC-02). It loads a human-authored YAML matrix, expands
cartesian axes into concrete variant cells, and validates every cell (agent enum D-05,
unsupported-combination gate D-07, test_command shell-safety D-08) with whole-run
all-or-nothing fail-fast (D-06).

Purpose: This module is the declaration + validation foundation the CLI (Plan 02) and the
per-cell restore orchestrator (Plan 03) both consume. It must be PURE and deterministic —
no LLM, no network, no fs beyond reading the spec file.
Output: `experiment-spec.mjs` + a one-line export addition to `evidence-harness.mjs` + tests
+ an example spec YAML documenting the schema.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-authored YAML → resolver | untrusted human-authored spec content crosses into the process |
| spec test_command → downstream span.meta → evidence-harness spawnSync | a user string becomes an executed argv (in a later phase) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-77-01 | Tampering/Elevation | test_command field (D-08) | mitigate | Reject at resolve time with the canonical `SHELL_META_RE` guard (imported, not re-invented); a `|`/`$(`/`;`/`&`/newline-bearing command throws before run 0. Tokenized fixed-argv only downstream. |
| T-77-02 | Tampering | YAML spec load (D-01) | mitigate | `js-yaml` `yaml.load` uses the DEFAULT_SCHEMA (no arbitrary JS type construction). Apply the taxonomy.mjs null/shape guard; reject non-object / missing-goal_sentence specs. |
| T-77-03 | Spoofing/Bypass | agent dimension (D-05) | mitigate | Enum-validate `agent` against the frozen KNOWN_AGENTS set; unknown agent aborts the whole matrix (D-06) so no unvalidated agent silently runs. |
| T-77-04 | Information Disclosure | unsupported combos (D-07) | mitigate | Frozen combination table fails `copilot + headless` fast with a Phase-78 pointer rather than pretending it works and producing a biased/mid-run failure. |
| T-77-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs in this phase — `js-yaml` and `node:crypto` are already project dependencies (used by taxonomy.mjs / measurement-stop.mjs). No new supply-chain surface. |
