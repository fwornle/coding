<objective>
Create `lib/experiments/experiment-restore.mjs` — the per-cell restore orchestrator that gives
RUN-01 its "every variant × repeat starts from the identical snapshot" guarantee, plus the
SC#4 byte-identical determinism PROOF (D-11/D-12). It wires (does NOT rebuild) the shipped
Phase-67 `restoreSnapshot` into an isolated sandbox per cell (D-10), computes a deterministic
digest over the restored git tree + KB + routing config, and asserts two repeats of a variant
are byte-identical — aborting the experiment on divergence.

Purpose: A cross-agent comparison is only trustworthy if every cell provably begins from the
same conditions. This module converts that from an assumption into a hard, digested assertion.
Output: `experiment-restore.mjs` + a thin operator CLI + tests proving determinism and the
divergence-abort.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| snapshot_id → sanitizeTaskId → restore sandbox path | a caller-supplied id becomes an fs path under .data/run-restores/ |
| captured env-allowlist → snapshot | environment values captured at declare-time land in the snapshot |
| live checkout/KB ↔ restore sandbox | the restore MUST NOT mutate the live tree/KB |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-77-08 | Tampering (blast radius) | restore isolation (D-10) | mitigate | Always call restoreSnapshot with `inPlace:false`; `grep` asserts no `inPlace:true` and the CLI exposes NO `--in-place` flag — the live checkout/KB is never touched, and a bad cell cannot corrupt real state. |
| T-77-09 | Tampering (path) | snapshot_id → sandbox path | mitigate | The Phase-67 rig's `sanitizeTaskId()` ([A-Za-z0-9._-]) already scopes the id before it reaches the run-restores/run-snapshots dir; the runner passes the id straight through — it does not build its own snapshot paths from unsanitized identifiers. |
| T-77-10 | Information Disclosure | env-allowlist in snapshot | accept | The digest reads only the rig's already-captured env-allowlist + processOverrides-only llm-settings.json; this phase adds NO new env into the snapshot beyond what the Phase-67 rig's allowlist already permits. |
| T-77-11 | Repudiation (false comparison) | determinism assert (D-11/D-12) | mitigate | assertRepeatsIdentical hard-throws on divergence with both digests printed — a comparison can never silently proceed from non-identical starting conditions. |
| T-77-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs — wires the shipped lib/repro/* rig and node:crypto builtin. No new supply-chain surface. |
</threat_model>
