<objective>
Extend `scripts/measurement-start.mjs` so an operator can declare a single variant's
executable config on the command line (SPEC-01's "CLI flags" surface) OR select a named cell
out of a validated spec file (SPEC-02's concrete-config resolution), threading the resolved
`{agent, model, framework, env, test_command}` into `span.meta` where `measurement-stop.mjs`
and `evidence-harness.mjs` already read them.

Purpose: This is the seam that turns a validated variant cell (Plan 01) into a measured span's
metadata. Without it, `measurement-stop.mjs` keeps writing `agent=canonicalAgent` fallbacks and
the harness never sees the fast per-variant `test_command`.
Output: new flags + a `--spec/--variant` mode on measurement-start.mjs, plus tests proving the
meta keys land and that CLI flags override the spec file.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator CLI argv → span.meta | operator-supplied flags/spec-path become measured-span metadata |
| spec-path → resolveExperimentSpec → fs read | a user-supplied file path is read from disk |
| test_command flag → span.meta.test_command → later spawnSync | a CLI string becomes an executed argv downstream |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-77-05 | Tampering/Elevation | --test-command flag | mitigate | Guard with the exported `SHELL_META_RE` and `process.exit(1)` before the span opens; snake_case `test_command` only, tokenized fixed-argv downstream. |
| T-77-06 | Spoofing/Bypass | --spec/--variant resolution | mitigate | resolveExperimentSpec runs its full whole-run validation (agent enum + combo gate + shell-safety) inside a try/catch; ANY invalid cell aborts the CLI before active-measurement.json is written — no partial/unvalidated span. |
| T-77-07 | Tampering | meta key names | mitigate | Only the exact downstream-read keys (`agent`/`model`/`framework`/`test_command`/`env`) are set via conditional spread; `grep` asserts no `testCommand` camelCase leak that would be silently dropped. |
| T-77-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs — reuses Plan-01 experiment-spec.mjs and existing measurement-start deps. No new supply-chain surface. |
</threat_model>
