Create directory `src/hooks/` and file `src/hooks/knowledge-injection-hook.js` (per D-01).

The script is a plain JS ESM module (NOT TypeScript -- matches STATE.md decision "Plain JS for src/retrieval/ modules"). Zero npm dependencies -- uses only Node.js built-in `http` module.

**Structure (top to bottom):**

1. **Shebang and import:**
   ```
   #!/usr/bin/env node
   import http from 'node:http';
   ```

2. **Constants:**
   - `SAFETY_TIMEOUT_MS = 5000` -- absolute process ceiling, `setTimeout(() => process.exit(0), SAFETY_TIMEOUT_MS).unref()` (prevents hook from ever hanging)
   - `MIN_TOKENS = 20` -- per D-03
   - `HTTP_TIMEOUT_MS = 2000` -- per D-09
   - `RETRIEVAL_PORT = 3033` -- retrieval service port
   - `MAX_QUERY_CHARS = 500` -- server rejects queries over 500 chars
   - `MAX_OUTPUT_CHARS = 9500` -- safety margin under 10K char limit (Pitfall 2 from RESEARCH.md)

3. **`async function main()`:**
   a. Read stdin using the established pattern:
      ```javascript
      const chunks = [];
      if (process.stdin.isTTY === false) {
        for await (const chunk of process.stdin) {
          chunks.push(chunk);
        }
      }
      const raw = Buffer.concat(chunks).toString('utf8').trim();
      ```
   b. Guard: if `!raw`, return (exit 0 no stdout -- per D-06)
   c. Parse JSON with try/catch. On parse failure, return (fail-open per D-09).
   d. Extract prompt: `const prompt = (input.prompt || '').trim();`
   e. Filter: empty prompt -> return (per D-05)
   f. Filter: `prompt.startsWith('/')` -> return (per D-04)
   g. Filter: `prompt.split(/\s+/).length < MIN_TOKENS` -> return (per D-03; whitespace split heuristic per Claude's Discretion)
   h. Truncate query for retrieval: `const query = prompt.slice(0, MAX_QUERY_CHARS);` (server rejects >500 chars)
   i. Call `callRetrieval(query)` (see below)
   j. Guard: if `!result || !result.markdown || result.meta?.results_count === 0`, return (per D-10)
   k. Safety truncation: if `result.markdown.length > MAX_OUTPUT_CHARS`, truncate to MAX_OUTPUT_CHARS + `\n\n[truncated]`
   l. Write JSON to stdout (per D-07, D-08):
      ```javascript
      process.stdout.write(JSON.stringify({
        hookSpecificOutput: {
          hookEventName: 'UserPromptSubmit',
          additionalContext: markdown,
        },
      }));
      ```

   Wrap entire main body in try/catch. Catch block: `process.stderr.write('[knowledge-hook] Error: ' + err.message + '\n');` then return (fail-open per D-09).

4. **`function callRetrieval(query)`:**
   Returns a Promise that ALWAYS resolves (never rejects -- fail-open per D-09).
   - POST to `127.0.0.1:3033/api/retrieve` with body `JSON.stringify({ query, budget: 1000, threshold: 0.75 })`
   - Headers: `Content-Type: application/json`, `Content-Length: Buffer.byteLength(body)`
   - `timeout: HTTP_TIMEOUT_MS` on the request options
   - On response: collect chunks, parse JSON, resolve with parsed data
   - On timeout: `req.destroy(); resolve(null);`
   - On error: `resolve(null);` (not reject)
   - On JSON parse failure in response: `resolve(null);`

5. **Invocation:**
   ```javascript
   main().then(() => process.exit(0));
   ```

**CRITICAL: No `console.log` or `console.error` anywhere.** Use `process.stderr.write()` for all diagnostics and `process.stdout.write()` for the single JSON output. Only ONE `process.stdout.write` call in the entire script (the final JSON output).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Hook stdin (Claude Code -> hook) | Claude Code pipes prompt JSON; trusted (same-user, same-machine process) |
| Hook -> retrieval service | localhost HTTP; trusted (same-machine, no network exposure) |
| Hook stdout -> Claude Code | Hook output injected as system-reminder; must be well-formed JSON |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-30-01 | Tampering | Hook stdout | mitigate | Single `process.stdout.write(JSON.stringify(...))` call -- no string concatenation, no console.log that could corrupt JSON output |
| T-30-02 | Denial of Service | Hook hanging | mitigate | 5-second safety setTimeout with unref() as absolute ceiling; 2-second HTTP timeout on retrieval call; hook always exits 0 |
| T-30-03 | Denial of Service | Retrieval service down | mitigate | Fail-open: HTTP errors and timeouts resolve to null, hook exits 0 with no stdout (per D-09) |
| T-30-04 | Information Disclosure | Large context injection | mitigate | 9500-char safety truncation before stdout write; retrieval service already caps at 1000 token budget (~4000 chars) |
| T-30-05 | Tampering | Oversized query to retrieval | mitigate | Client-side truncation to 500 chars before HTTP POST; server also validates and returns 400 if >500 |
| T-30-06 | Spoofing | Prompt injection via stdin | accept | Hook passes prompt as-is to retrieval -- retrieval service uses it as a search query only (no LLM processing of prompt in retrieval path); risk is low |
</threat_model>
