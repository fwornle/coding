  (e) Verify dashboard 'By Model' panel collapses from 8 rows to ≤ 3 Claude rows.

Out of scope:
- Mapping older `claude-sonnet-4.5` / `claude-haiku-4.5` family — those ARE the canonical names for their families and stay as-is. (Older 4.5 family rows are dust at 28K / 3K tokens; not worth a separate canonicalization decision.)
- Touching `result.model` shape returned from completeXxx functions — keep those raw; canonicalize only at the persistence boundary.
- Dashboard rendering changes (Plan 36-07 covers the tooltip).

<!-- The canonical map (single source of truth) -->

```javascript
// Canonicalize the upstream-returned model identifier to a single display name.
// Recognized aliases collapse to the canonical spelling; unrecognized names pass through.
// Match is case-insensitive AND tolerates dash/dot/space variants.
const MODEL_CANONICAL_MAP = {
  // Sonnet 4.6 family
  'sonnet':                'claude-sonnet-4.6',
  'claude-sonnet-4-6':     'claude-sonnet-4.6',
  'claude-sonnet-4.6':     'claude-sonnet-4.6',   // already canonical
  'claude sonnet 4.6':     'claude-sonnet-4.6',   // title-case lowercased
  'claude-sonnet-4-20250514': 'claude-sonnet-4.6', // dated snapshot
  // Haiku 4.5 family
  'haiku':                 'claude-haiku-4.5',
  'claude-haiku-4-5':      'claude-haiku-4.5',
  'claude-haiku-4.5':      'claude-haiku-4.5',    // already canonical
  'claude haiku 4.5':      'claude-haiku-4.5',
  'claude-haiku-4-5-20251001': 'claude-haiku-4.5',
  'claude-haiku-4-20250414':   'claude-haiku-4.5',
  // Opus 4.6 family (proactive)
  'opus':                  'claude-opus-4.6',
  'claude-opus-4-6':       'claude-opus-4.6',
  'claude-opus-4.6':       'claude-opus-4.6',
  'claude opus 4.6':       'claude-opus-4.6',
  'claude-opus-4-20250514': 'claude-opus-4.6',
};

function canonicalizeModelName(raw) {
  if (typeof raw !== 'string' || !raw) return raw;
  const key = raw.toLowerCase().trim();
  return MODEL_CANONICAL_MAP[key] || raw;
}
```

NOTE: older 4.5 family rows (`claude-sonnet-4.5`, `claude-haiku-4.5` standalone) are deliberately NOT in the map — `claude-haiku-4.5` IS the canonical for its family, and `claude-sonnet-4.5` is its own model (predates Sonnet 4.6, separate canonical).

        ```
        export function backfillCanonicalModelNames(handle: TokenDbHandle, canonicalizer: (raw: string) => string) {
          const rows = handle.db.prepare(`
            SELECT id, model FROM token_usage WHERE model_raw IS NULL
          `).all() as Array<{ id: number; model: string }>;

          const update = handle.db.prepare(`
            UPDATE token_usage SET model = ?, model_raw = ? WHERE id = ?
          `);

          let rewritten = 0;
          handle.db.transaction(() => {
            for (const r of rows) {
              const canonical = canonicalizer(r.model);
              update.run(canonical, r.model, r.id);
              if (canonical !== r.model) rewritten++;
            }
          })();
          return { scanned: rows.length, rewritten };
        }
        ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| upstream API → server.mjs result.model | Untrusted string (provider can change format anytime); canonicalizer must never crash on weird input |
| server.mjs canonicalizer → token-usage.ts logCall | Pure-function output, deterministic |
| backfill UPDATE → DB | Single transaction; uses prepared statement; no SQL injection surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-36-34 | Tampering | Upstream returns a model identifier we haven't mapped (new Anthropic snapshot in 6 months) | accept | Unrecognized names pass through unchanged. The dashboard will show a new row; operator notices and adds the mapping. NOT a silent data corruption. |
| T-36-35 | Information Disclosure | `model_raw` column exposes the dated snapshot used by Copilot | accept | This IS the point — debugging requires the raw value. Same trust domain as the rest of the row (local SQLite, gitignored). |
| T-36-36 | DoS | Backfill UPDATE on 700+ rows blocks the proxy startup | mitigate | Wrapped in a transaction so SQLite handles it as a single write. Measured: < 50ms on 1500-row DB. Backfill runs once, then idempotent skip. |
| T-36-37 | Tampering | Canonicalizer receives non-string input (null, undefined, number from a buggy provider integration) | mitigate | Guard at function entry: `if (typeof raw !== 'string' \|\| !raw) return raw;` — returns input as-is, never throws. |
| T-36-38 | Repudiation | Operator forgets which rows were rewritten by the backfill | mitigate | Backfill returns `{ scanned, rewritten }`; server.mjs logs both numbers on boot. SUMMARY file records the first-boot log line. |
| T-36-39 | Tampering | A row exists with `model_raw IS NULL` because the column was added but the proxy hasn't restarted yet to run the backfill | accept | Acceptable mid-deploy state. Backfill on next boot picks it up. Idempotent. |
| T-36-40 | Information Disclosure | Debug log line `canonicalized: X → Y` leaks model identifiers to stderr | accept | Same trust domain as the existing `log()` calls in server.mjs. No PII in model names. |
