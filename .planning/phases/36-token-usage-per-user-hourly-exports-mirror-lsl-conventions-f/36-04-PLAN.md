<objective>
Add the `user_hash` column + unique index to the SQLite schema, retag pre-existing rows on this machine with the real USER_HASH, rebuild the module-level `insertStmt` so new inserts bind user_hash, and replace cold-start `restoreFromJsonIfEmpty` with always-on `hydrateFromExports` that recursively merges all per-hour files. This unlocks cross-user merge: after `git pull` pulls a peer's `..._<other-hash>.json`, the next proxy boot ingests it via `ON CONFLICT(user_hash, id) DO NOTHING` and the dashboard shows the peer's rows alongside yours.

Purpose: Three engineering invariants get installed by this plan:
  (1) Schema invariant: dedup is on `(user_hash, id)` not `id` alone — so two users' `id=1` rows coexist.
  (2) Retag invariant: existing rows (pre-migration `user_hash='unknown'` defaulted) get UPDATEd to the real USER_HASH, so monolith-hydrate composite keys match DB composite keys.
  (3) Always-on hydration: every boot syncs ALL files in the export tree — so peer pushes are auto-merged without manual import. This is the semantic flip CONTEXT.md L72 calls out explicitly.

Output: Modified `_work/rapid-llm-proxy/src/token-usage.ts` with migration block + insertStmt rebuild + logCall update + new `hydrateFromExports`. Old `restoreFromJsonIfEmpty` deleted. Plan 36-03's `// TODO(36-04)` defensive fallback in exportToHourFile cleaned up.
