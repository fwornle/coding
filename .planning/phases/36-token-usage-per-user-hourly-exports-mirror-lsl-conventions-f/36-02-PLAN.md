<objective>
Compute the user-hash inside the launchd-invoked wrapper (`_work/rapid-llm-proxy/bin/start-llm-proxy.sh`) BEFORE `exec node`, so the proxy's TypeScript module receives `process.env.LLM_PROXY_USER_HASH` from any startup path (launchd, `bin/coding`, direct shell invocation).

Purpose: Plan 36-03 (writer rewrite) needs `process.env.LLM_PROXY_USER_HASH` to name per-hour files `YYYY-MM-DD_HHMM-HHMM_<hash6>.json`. Without this, every user would write into the same file and merge conflicts return. Per PATTERNS.md Section 4 anomaly + CONTEXT.md revised decision: `bin/coding` does NOT launch the proxy — launchd does — so env vars set by `bin/coding` never reach the proxy. The wrapper is the only ALWAYS-on-path point.

Output: Modified wrapper that exports the hash before exec, with safe fallback to `'unknown'` on any failure.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-36-06 | Information Disclosure | Hash leaks PII via filename | accept | Hash is SHA-256(salt + $USER) truncated to 6 hex chars — cryptographically irreversible at 24-bit truncation only by chance, and the input space is bounded (~10^6 typical $USER values; brute-force is feasible but the hash is NOT a secret — it's a peer-id token. CONTEXT.md L60-72 + LSL precedent accept this risk explicitly.) |
| T-36-07 | Tampering | Wrapper executes node script from hardcoded path | mitigate | Hardcoded absolute path `/Users/Q284340/Agentic/coding/scripts/user-hash-generator.js`; same trust domain as the wrapper itself (same machine, same $USER). No env-var-driven path = no PATH-injection risk. |
| T-36-08 | Tampering / Code injection | Stdout from user-hash-generator.js shell-substituted | mitigate | Validate output against `^[a-z][a-z0-9]{5}$` regex before export; non-matching output → set to 'unknown' and log. Prevents a malicious script from injecting shell metachars via stdout. |
| T-36-09 | DoS | Slow hash generation blocks proxy boot | accept | ESM dynamic import + SHA-256 of $USER ≈ 50ms cold-path (PATTERNS.md Section 4 measurement). Acceptable on proxy startup; happens once per launchd job lifetime. |
| T-36-10 | Repudiation | Hash mismatch between wrapper and tools | mitigate | Acceptance criterion enforces wrapper's hash equals direct `node -e import()` invocation. Any drift surfaces immediately on test. |
</threat_model>
