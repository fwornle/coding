# Plan 36-03 Summary

## Objective

Rewrite the JSON export side of `_work/rapid-llm-proxy/src/token-usage.ts`
to emit one file per `(time-window, user_hash)` pair under
`<baseDir>/YYYY/MM/`, mirroring the LSL filesystem convention. Source
the time-window from the coordinator (Plan 36-01) with local fallback;
source the user hash from `process.env.LLM_PROXY_USER_HASH`
(Plan 36-02) with `'unknown'` fallback.

## What changed in `_work/rapid-llm-proxy/src/token-usage.ts`

- **Module-level constants** read once at init:
  - `USER_HASH` â€” `process.env.LLM_PROXY_USER_HASH`, regex-validated, falls back to `'unknown'`.
  - `LSL_TIMEZONE` â€” `process.env.LSL_TIMEZONE`, default `Europe/Berlin`.
  - `COORDINATOR_URL`, `WINDOW_CACHE_TTL_MS=30_000`.
  - `_sessionDurationMs` â€” read once from `config/live-logging-config.json` via `CODING_REPO`, default 60 min.
  - `_intlFmt` â€” cached `Intl.DateTimeFormat` for `LSL_TIMEZONE`-aware year/month/day/hour extraction (constructor is the slow part).
- **New exports:**
  - `resolveTokenExportDir(overrideDataDir?)` â€” returns base dir for per-(window,user) files.
  - `currentWindow(now: Date)` â€” coordinator-curl with 30s cache + local-recompute fallback. Warmed via `setImmediate` at module init.
  - `exportToHourFile(handle, now: Date)` â€” the new writer. Right-exclusive `[windowStart, windowEnd)` SELECT, safety-merge by `(user_hash, id)`, dedup-on-write.
- **Debounce rewrite:** single `_exportTimer` replaced with `_exportTimers = new Map<windowKey, NodeJS.Timeout>()` keyed per window. Two timers coexist across an hour boundary so each window flushes to its own file. `scheduleExport(handle, now: Date = new Date())`.
- **Deprecations:** `resolveTokenExportPath` and `exportToJson` retained with `@deprecated` JSDoc for any external caller â€” Plan 36-05's migration removes the monolithic file.
- **TODO(36-04):** the `restoreFromJsonIfEmpty` block carries an explicit TODO comment; its replacement with `hydrateFromExports` is Plan 36-04's first task.
- **Defensive pre-36-04 fallback:** the SELECT first tries the full schema with `user_hash`; on `"no such column"` SqliteError it falls back to the legacy SELECT and synthesizes `user_hash` in-memory from the `USER_HASH` constant. Removed by Plan 36-04.
