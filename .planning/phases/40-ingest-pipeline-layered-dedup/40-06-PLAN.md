<objective>
Land the 4-stage `IngestPipeline` orchestrator that composes the extractor, deduplicator, store, and synthesizer into a single `ingest(text, opts)` call. Threads caller-supplied `ProvenanceStamp` through all stages without modification (CF-D30 — pipeline NEVER invents a stamp). Pre-loads same-class candidates via `store.findByOntologyClass(entity.ontologyClass)` before each dedup call (D-46), inheriting Phase 39 D-34's active-only-by-default filter. When dedup matches, writes the new entity with `supersedes: survivor.id` and lets Phase 39's `putEntity` handle the atomic predecessor-closure (D-33 + CR-01).

Also ships the integration tests that verify the boundary crossings: Phase 39 D-33 supersession path (works on active entities), Phase 39 CR-01 path (works on legacy non-v7 id predecessors without throwing), D-46 candidate-pool active-only filter, and the **ROADMAP SC#2 synthetic collision test** (one exact-name collision + one cosine collision + one LLM-semantic collision; all three caught at the correct layer in declared order; short-circuit proven by call-count assertions on the fake clients).

Output: 1 source file (~250 lines per RESEARCH size estimate) + 1 unit test file (~10 tests) + 3 integration test files (~7 total tests) + green vitest run + dist/ rebuilt.
