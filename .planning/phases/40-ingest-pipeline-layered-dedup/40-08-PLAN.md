**CR-01** (REVIEW.md offset 45-81, VERIFICATION.md gap #1):
`IngestPipeline.ingest()` at `IngestPipeline.ts:197-200` computes `ontologyClass = entity.ontologyClass ?? entity.entityType` and unconditionally calls `store.findByOntologyClass(ontologyClass)`. `entity.entityType` is `string` per the type but a buggy/legacy extractor can return an entity with both fields empty. `findByOntologyClass(undefined as any)` silently returns `[]`, every input becomes net-new, duplicates are silently written. LayeredDeduplicator's Pitfall-1 guard at `LayeredDeduplicator.ts:136` never runs because the pipeline never reaches `deduplicator.dedup()`. Fix: hoist the Pitfall-1 guard into the pipeline — gate the `findByOntologyClass` call behind `if (ontologyClass)`, fall through to the empty-candidates path with a stderr-warn.

**CR-04** (REVIEW.md offset 141-160, VERIFICATION.md gap #4):
`runStage('extract', ...)` overload at `IngestPipeline.ts:288` requires `opts: { provenance: ProvenanceStamp }` but the implementation at line 309 calls `this.extractor.extract(input as string, undefined)` — provenance is unused AND domain is hardcoded undefined. `ingest()` at line 181 honors `opts.domain`. Callers using `runStage('extract')` lose domain scoping. Fix: widen overload 1 to `runStage(name: 'extract', input: string, opts?: { domain?: string }): Promise<Entity[]>` (drop misleading provenance requirement; add optional domain) AND thread `opts?.domain` into the extractor call at line 309. Overloads 2, 3, 4 stay byte-identical (Q2 RESOLVED contract preserved).
