<objective>
Port OKM's `batchLLMDedup` prompt + 5-stage JSON-unwrap (`deduplicator.ts:421-475`) into `LLMSemanticMatcher` — a class implementing the `LLMSemanticLayer` interface. Caller-supplied `LLMClient` interface (`complete(req): Promise<{ content: string }>`) lets each system wire its own LLM provider (rapid-llm-proxy for OKM, groq/haiku for A, varies for B). Default threshold 0.70 (RESEARCH A3 — OKM's implicit threshold). Default error policy 'skip' returns `{ matched: false }` + stderr-warn so LLM failures don't block ingest (matches OKM `deduplicator.ts:213-218` pattern). The 5-stage JSON-unwrap is mandatory because Copilot CLI / claude-code responses arrive in unpredictable shapes (Pitfall 3).

Purpose: Layer 3 of the 3-layer dedup pipeline — the most expensive layer, runs last per D-44 declared order, catches semantic duplicates that pass exact-name and embedding-cosine layers (e.g., "OOM" vs "Out of Memory"). Phase 43 (INT-03) will wire OKM's existing LLM proxy as the concrete client; Phases 41+42 may wire other providers.

Output: 1 source file (~120 lines per RESEARCH.md size estimate) + 1 test file (~9 test cases including all 5 JSON-unwrap variants + onError policies) + green vitest run + dist/ rebuilt.
