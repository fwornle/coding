<objective>
Land the `LayeredDeduplicator` class — the orchestrator that wraps the 3 layer slots (exactName, embedding, llmSemantic) with the short-circuit-on-first-match logic locked by D-44. Iterates layers in declared cost-ascending order (Jaccard cheapest → cosine → LLM most expensive). When a layer reports `matched: true` with `confidence >= layer.threshold` AND `shortCircuit: true` (default), downstream layers are NOT invoked — this is the load-bearing optimization that keeps per-batch dedup fast even with the LLM layer wired. When `shortCircuit: false`, all layers run sequentially for offline analysis / calibration runs (D-44).

Purpose: The orchestrator decouples the 3 layer algorithms from the framework that composes them. Phase 41 (A) and Phase 42 (B) wire different concrete `EmbeddingClient` / `LLMClient` impls into the layers but reuse this orchestrator verbatim — no fork. ROADMAP SC#2 ("layered dedup on synthetic batch catches all three collisions at the correct layer in order") is verified by Plan 06's integration test, but the orchestrator's per-layer contract is verified here.

Output: 1 source file (~80 lines per RESEARCH.md size estimate) + 1 test file (~8 test cases using makeLayerStub from Plan 01) + green vitest run + dist/ rebuilt.
</objective>
