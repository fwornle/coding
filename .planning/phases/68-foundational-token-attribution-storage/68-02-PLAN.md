<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| task_id → filename | task_id (operator- or PLAN-derived) is interpolated into `.data/measurements/<task_id>.json` — path-traversal surface |
| active-measurement.json → proxy hot path | A corrupt/untrusted span file is parsed by getActiveMeasurement which the proxy write path (Plan 68-03) calls per request |
| concurrent start/stop | Two start/stop invocations race on the single active-measurement.json |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-68-02-01 | Tampering | archive filename from task_id | mitigate | sanitizeTaskId() restricts to `[A-Za-z0-9._-]`, rejects `.`/`..`/empty, caps length; path.basename() applied at the rename target so no separator escapes archiveDir; integration test attempts `../../etc/passwd` and asserts no out-of-dir write. |
| T-68-02-02 | Denial of Service | getActiveMeasurement parse | mitigate | JSON.parse wrapped in try/catch returning null + non-fatal stderr; never throws — a corrupt span cannot crash the proxy request path that consumes the reader. |
| T-68-02-03 | Tampering | concurrent start/stop race | mitigate | stopMeasurement archives via temp-write + fs.renameSync (atomic on same filesystem) then removes the active file; rename is the crash-safe primitive. Residual: a true concurrent start during stop is operator-driven and out-of-scope (single-operator workflow) — documented as accepted. |
| T-68-02-04 | Information Disclosure | span goal_sentence/meta contents | accept | Span files live under the operator's own .data (same trust zone as token-usage.db); no new exposure beyond existing local telemetry. |
| T-68-02-SC | Tampering | npm installs | accept | No new packages introduced (node:fs/path only). No package-legitimacy checkpoint required. |
</threat_model>
