| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-68-03-01 | Denial of Service | resolveActiveTaskId in hot path | mitigate | Wrapped in try/catch returning '' on any error; getActiveMeasurement is itself null-safe (Plan 68-02); a measurement-read failure logs the row with task_id '' rather than failing the LLM call. |
| T-68-03-02 | Tampering | backfill UPDATE | mitigate | UPDATE is parameterized (better-sqlite3 placeholders), gated on `task_id = ''` so it never clobbers a live value, and span task_ids were sanitized at write time (Plan 68-02) so they carry no SQL-meta risk; --dry-run path issues no writes. |
| T-68-03-03 | Repudiation | backfill attribution correctness | accept | Timestamp-join attribution is heuristic (a row at a span boundary could in principle belong to an overlapping span); single-operator sequential spans make overlap unlikely. Documented as accepted; overwrite-guard (`task_id=''` only) bounds the blast radius. |
| T-68-03-SC | Tampering | npm installs | accept | No new packages (better-sqlite3 + node builtins only). No package-legitimacy checkpoint required. |
