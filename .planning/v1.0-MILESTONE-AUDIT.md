## Regression Details

### 1. OBSV-01 / OBSV-02 — LLM Observation Synthesis Disabled

**Commit:** 442afc1 — "fix: garbage insight names and observation bold formatting"
**Root cause:** Per-item LLM calls (30s+ each) caused workflow hangs at scale
**What happened:** All 4 `synthesizedContent` variables in `observation-generation-agent.ts` replaced with `const synthesizedContent: any = null`

This was a deliberate user decision for performance. The fix worked correctly but was too slow for production use.

**Options:**
- A. Accept regression, revise OBSV-01/OBSV-02 to "observations contain structured raw data" (lower bar)
- B. Re-implement with batch LLM synthesis instead of per-item calls (maintain quality, fix performance)
- C. Defer to a future milestone focused on observation quality

### 2. DATA-03 — analysisDepth Reverted to 'surface'

**Commit:** 802a3b0 — "fix: reverted back from deep to surface analysis (took too long)"
**Root cause:** Deep analysis significantly increased pipeline runtime

**Options:**
- A. Accept regression, remove DATA-03 from v1.0 requirements
- B. Make analysisDepth configurable (user picks speed vs. depth per run)
- C. Defer to when pipeline performance is improved
